# F-001：组图 23 List 2 显著性增强

## 1. Requirements Summary

- `prompts/2.group-image/组图 23.md` 的 List 2 MUST 从“合规补点”升级为“第一眼可见的辅助差异”。
- List 2 SHOULD 临时从 16 条提升为 18 条，以增强冷区补点和显著性调度空间。
- 差异 MUST 保持单点、小区域、可单独圈选。
- 差异 MUST NOT 依赖颜色、材质、细纹、轻微位移或大面积背景变化成立。

## 2. Design Decisions

### 2.1 显著性准入

List 2 的候选点 MUST 先过“第一眼可见”闸门，再过类型、句式和配额检查。若一个候选点需要细看、回看、放大或逐格对照，直接淘汰。

### 2.2 差异类别改写

List 2 SHOULD 改写为抽象类别驱动，而不是少量固定例子驱动。推荐类别：
- 局部结构缺损
- 局部表面破坏
- 局部状态反转
- 局部异常附着 / 脱落
- 局部方向 / 连接异常

同一类别 MUST NOT 连续堆叠成模板；同一类别 SHOULD 通过不同视觉结果表达，而不是只换位置名。

### 2.3 小区域但强信号

差异 MUST 集中在单个局部块，不跨多个主体，不扩散到整片承载面。显著性应来自轮廓中断、缺口、开合、附着、断裂、局部存在感变化，而不是大面积改图案。

### 2.4 对大承载面的限制

地板、毛毯、整墙、整窗帘、大家具主体表面 MUST NOT 作为显著性放大的主要载体。若必须落在承载面上，MUST 仅为块状、边界清楚、局部独立的小区域变化，并补写“局部变化，不改整片主体图案”。

### 2.5 防模式坍缩

规则 SHOULD 使用“类别 + 准入条件 + 禁止项”的写法，MUST NOT 列出少数高频模板例子。应明确“同类机制允许多形态表达”，避免模型反复产出单一破损样式。

## 3. Interface Contract

建议后续编辑点：
- `List 2：辅助差异` 段落
- `固定输出格式` 中 List 2 数量说明
- `合规自检` 中显著性与临时 18 条校验项

## 4. Constraints & Risks

- 必须保留九宫格覆盖。
- 必须保留紧邻互斥。
- 必须保留禁颜色 / 材质前缀。
- 风险：显著性过强导致画面变脏；需用“局部、边界清楚、单点主信号”约束。

## 5. Acceptance Criteria

1. List 2 每个差异点 MUST 在首次扫视中可被定位。
2. List 2 MUST 不再接受细裂纹、小点、细线、轻微磨损等低可见差异。
3. List 2 SHOULD 采用抽象类别驱动，且同类不形成连续模板。
4. List 2 差异 MUST 保持单点、小区域、可单独圈选。
5. 大承载面变化 MUST 受控，且不得演变成整片改图案。
