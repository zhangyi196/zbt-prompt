## Current Ideas

1. **主题重写优先：从“风格杂货”改为“有场景的主题盲盒”**  
   每个盲盒要像一个小片场：有常见空间、有主物、有配套小物、有少量装饰，而不是大量同尾词收纳物。

2. **保留主题味道，但减少组合词堆叠**  
   主题辨识度来自“真实场景物品组合”，不主要依赖 `碎花/格纹/复古/藤编 + 盒/箱/篮/挂`。

3. **每个主题建立精品核心池**  
   每个主题先有 40-60 个高适配物品，再用扩展池补丰富度；不为凑满 100 而制造低真实感物品。

4. **把悬挂类改为条件类**  
   悬挂物只有在目标图有墙、门、窗、杆、椅背等承载面时才启用；默认不再和地面/桌面物品同权重。

5. **主题矩阵分为“保留改造组 + 新增补强组”**  
   旧主题保留其审美方向，新主题补足桌面、教室、商店、工具、运动、医疗、儿童等常见目标图。

### Round 1 - Codebase And Content Exploration

#### Findings

- 当前盲盒库共有 14 个主题：缝纫拼布、茶歇咖啡、烘焙厨房、梳妆护理、阅读休闲、编织手作、布草收纳、家居装饰、节庆装饰、宠物角、门廊庭院、雨天出行、海边度假、海底宝藏与海洋植物。
- 每个主题固定包含 `large / medium / small / hanging` 四类，每类 100 个物品，即总计约 5600 个物品。
- `large` 类约 93.4% 命中容器/收纳语义，例如盒、箱、篮、筐、匣、袋、包、罐、瓶、坛、盘、碟、杯、壶等。
- `small` 类约 49.9% 命中“迷你、小巧、小号、标签、样片、配色、签、卡、扣、夹”等小型或抽象语义。
- `hanging` 类约 60.4% 命中悬挂承载依赖，约 57.4% 命中软性可变形物，例如帘、巾、串、挂兜、流苏、披肩、搭巾。
- `medium` 相对最健康，因为它包含更多真实工具/器具，如茶壶、剪刀、刷子、铲子、帽子、杯子、托盘等。

#### Challenged Assumptions

- ~~问题主要来自几个坏物品~~ -> 更像是分类和生成策略导致的结构性偏差。  
  - Reason: 高风险模式在四类中呈系统性分布，尤其是容器泛滥、小物抽象、悬挂依赖。

#### Ideas Generated

1. **场景适配主题库**  
   不再只围绕温馨杂货主题，而是覆盖桌面、厨房、浴室、儿童、办公室、教室、户外、工具、商店、医院、运动等常见画面场景。

2. **物品适配等级**  
   每个物品标记为 `core / support / conditional / risky / blocked`，抽取默认优先 core 与 support。

3. **类别重构**  
   将 `large / medium / small / hanging` 改造成 `主体锚点 / 独立清晰物 / 小型清晰物 / 块状装饰 / 条件悬挂 / 风险装饰`。

4. **主题内物品配额重设**  
   每个主题不必四类等量，改为 40% 独立清晰物、25% 主体锚点、20% 小型清晰物、10% 块状装饰、5% 条件悬挂。

5. **命名规范重写**  
   减少“材质/纹理 + 容器尾词”的自动组合，要求物品名优先是现实中可识别的对象。

#### Analysis Results

现有 14 个主题的问题不是“没有主题味道”，而是主题味道主要靠装饰形容词和容器尾词维持。更稳的方式是让主题本身变成一个常见小场景：

- 好主题应该让人立刻知道物品会出现在哪里，例如“书桌学习角”“浴室洗漱台”“便利店零食架”。
- 好主题应该天然产生真实物品，而不是靠拼接出 100 个相似容器。
- 好主题应该有清晰的主物、配物、小物、装饰物和条件物，不需要每类均等。
- 主题之间应该覆盖不同图像空间：桌面、地面、墙面、柜架、户外、人物周边、店铺货架。

#### Recommended Theme Standard

每个主题应写一段维护标准：

- **场景锚点**：这个主题默认出现在哪类画面区域。
- **主物范围**：5-10 个最能代表主题的高稳定物。
- **禁用尾词**：该主题内尤其容易滥用的尾词。
- **条件物**：只有有承载面/特定空间时才抽的物品。
- **不适合项**：不真实、太细、太抽象、光影风险、承载不稳的物品。

#### Devil's Advocate Analysis

**Objection 1: 新主题数量可能过多，维护成本会上升。**  
第二轮从 14 个主题扩展到 27 个主题。如果每个主题仍追求大规模物品池，会让维护压力翻倍，并且很容易再次用组合词凑数。

Strengthened version: 新增主题不要一次性全上。先形成 `14 个改造主题 + 6 个高价值补强主题 + 3 个条件主题`。其余作为候选，不进入第一批。

**Objection 2: 部分新增主题会稀释原有温馨手作/家居气质。**  
便利店、医疗、修理工具台、运动装备角虽然适配常见图，但和现有库的温馨杂货审美有距离。如果全部加入，工具会变得像通用素材库，而不是“盲盒主题库”。

Strengthened version: 新增主题应优先选择能兼容现有气质的场景，例如文具学习桌、儿童玩具地毯、浴室洗漱台、美术画材桌、露营野餐地、办公室桌面。便利店、医疗、修理工具、运动可以作为第二批或“实用补充包”。

**Objection 3: 旧主题之间已有明显重叠。**  
缝纫拼布、编织手作、布草收纳都属于布料/手作/软物；茶歇咖啡、烘焙厨房也都属于餐食台面；家居装饰、节庆装饰、海底宝藏与海洋植物都容易变成摆件陈列。

Strengthened version: 旧主题不一定全部平权保留。可以合并或明确分工：
- 缝纫工作台：剪裁、针线、布片、纸样。
- 编织篮与毛线角：毛线、织针、织片、记号扣。
- 衣柜布草架：毛巾、床单、洗衣、衣架，不再承担手作气质。

**Objection 5: 医疗护理主题有叙事升级风险。**

**Objection 6: 海底宝藏与海洋植物本身偏幻想，适配面窄。**  
它主题味道强，但不是通用目标图常见场景。若抽到普通室内图，海星、珊瑚、藏宝箱容易不贴。

**Style A: 直白长名**

优点：最清楚。  
缺点：长，UI 中可能拥挤。

优点：清楚、紧凑、适合 UI。  
缺点：不像自然中文标题。  
示例：`手作台｜缝纫拼布`

**Style C: 自然短名 + 场景后缀**

优点：读起来顺，仍明确。  
缺点：有些名字会略长。  
示例：`缝纫手作台场景`

#### Recommended Naming Style

推荐使用 Style C 作为盲盒名称：自然短名 + 场景后缀。  
若未来 UI 需要更短，可在内部保留 Style B 作为短标签。

#### Decision Log

> **Decision**: 在 20 个场景类别下设计物品类型，并暂时映射到现有四类结构。

#### General Allocation Rules

现有四栏可以临时重新理解为：

- `large`: 场景锚点物。不是“越大越好”，而是能说明场景的主物、承载物、成组物。
- `medium`: 主力清晰物。最应该多放，要求真实、独立、边界清楚。
- `small`: 小型可见物。必须第一眼可见，不用微型标签、样片、细扣、细线凑数。
- `hanging`: 条件物。只有明确可挂、可贴、可悬挂时才适用；数量应少于其他类，不再硬凑 100。

推荐配额：

- 多数主题：`large 25% / medium 45% / small 25% / hanging 5%`
- 户外主题：`large 30% / medium 45% / small 20% / hanging 5%`
- 海底主题：`large 30% / medium 45% / small 20% / conditional 5%`
- 若仍维持每类 100 个，至少应在内容质量上遵守上述权重，不要四类等量同权抽取。

### Round 12 - Optimize Item Direction

#### User Input

用户反馈：想看看类别下的物品方向是否还有优化空间。

#### Decision Log

> **Decision**: 对物品方向做结构性优化，而不是继续堆具体物品。  
> - **Context**: 用户关心类别下物品是否仍有优化空间。  
> - **Options considered**: 继续写样例；直接收敛；审查物品方向。  
> - **Chosen**: 审查物品方向并提出更强规则。  
> - **Rejected**: 继续写样例可能重复潜在问题；直接收敛会遗漏物品层风险。  
> - **Impact**: 方案新增物品池分层、风险比例、命名模板和每类禁用规则。

#### Remaining Problems In Current Item Direction

1. **`large / medium / small / hanging` 仍然容易误导维护者。**  
   如果只按大小理解，`large` 会再次塞满盒、箱、篮；`small` 会再次塞满迷你标签、夹子、扣子；`hanging` 会再次塞满帘、串、流苏。

2. **小物仍需更强“可见性”标准。**  
   例如珍珠、回形针、金币、夹子这类词，如果没有成组、承载或放大语义，仍可能太小。应优先写成 `珍珠贝`、`金币小堆`、`回形针盒`、`彩色夹子组`。

3. **物品库应避免混入活体或动物本体。**  
   工具有独立动物抽取逻辑，因此盲盒物品库中应尽量避免动物本体。海底类如果写 `海星`，最好改为 `海星壳饰`、`海星摆件`、`海星形标记牌`，避免和动物池语义混淆。

4. **容器不是不能用，而是不能成为主导。**  
   每个主题需要少量真实承载物，但不应靠 `盒/箱/篮/袋/包/罐/瓶` 同义扩写到几十条。

5. **条件物需要从“类别”变成“低权重角色”。**  
   悬挂物、墙面物、软布、绳带、灯、镜、透明物都需要条件才能用，不适合与主力物同权重。

6. **主题味道应来自物品组合，不来自前缀堆叠。**  
   旧库常用 `碎花/格纹/藤编/复古/亚麻/黄铜/白瓷` 做风格前缀。可以保留少量，但不应让所有物品都靠风格词区分。

#### Improved Item Pool Model

每个盲盒类别下建议拆成 5 个池，而不是只依赖大中小悬挂：

1. **core_items / 核心物**
   - 每类 25-40 个。
   - 最能代表场景。
   - 必须真实、常见、边界清楚。
   - 抽取权重最高。

2. **support_items / 配套物**
   - 每类 30-50 个。
   - 丰富场景，但不抢主题。
   - 多数对应现在的 `medium`。

3. **visible_small_items / 可见小物**
   - 每类 15-25 个。
   - 必须成组、带承载、或本身足够清楚。
   - 避免单颗、单线、单点、细小标签。

4. **conditional_items / 条件物**
   - 每类 0-12 个。
   - 悬挂、墙面、软布、可穿戴、需承载物。
   - 默认低权重。

5. **blocked_or_risky / 禁用或风险词**
   - 不参与默认抽取。
   - 灯光、镜面、透明反光、流苏细线、微型痕迹、抽象样片等。

#### Recommended Ratios

默认：

- core_items: 35%
- support_items: 35%
- visible_small_items: 20%
- conditional_items: 10%
- risky: 0% 默认抽取

户外：

- core_items: 40%
- support_items: 35%
- visible_small_items: 15%
- conditional_items: 10%

特殊主题，例如 `海底+潜水`：

- core_items: 35%
- support_items: 40%
- visible_small_items: 15%
- conditional_items: 10%
- 避免活体动物语义，偏装备、遗物、贝壳、珊瑚块、潜水物。

#### Naming Template

推荐：

- 真实名词：`铅笔`、`水壶`、`潜水镜`
- 成组可见：`金币小堆`、`彩色夹子组`、`贝壳堆`
- 承载清楚：`回形针盒`、`糖果罐`、`纽扣盘`
- 场景明确：`潜水标记牌`、`雨伞桶`、`野餐餐盒`

少用：

- 纯风格前缀堆叠：`复古碎花亚麻...`
- 抽象尾词：`样片`、`配色`、`布标`、`纹样`
- 细长软物：`流苏`、`帘带`、`边巾`、`细绳`
- 光影风险：`灯串`、`镜盒`、`发光瓶`、`反光片`

#### Category-Specific Optimization Examples

**桌面+学习**

- 强化：本子、笔、尺、计算器、订书机、便签、文件夹。
- 限制：回形针、夹子、标签必须带盒/组/牌，不写单个微物。
- 禁用倾向：样片、配色卡、过小纸角。

- 调整：`海星` 改为 `海星壳饰` 或 `海星形标记牌`。
- 限制：珍珠写成 `珍珠贝` 或 `珍珠小堆`，不写单颗珍珠。
- 禁用倾向：发光水母、灯串、镜盒、透明香瓶、细海草线。

#### Stronger Acceptance Checklist

每个物品进入核心池前，至少满足 5/6：

1. 一眼能识别是什么。
2. 边界清楚，可单独圈选。
3. 不依赖光影、反射、透明效果。
4. 不需要特殊承载面，或承载关系已明确。
5. 不是单点、细线、微型痕迹。
6. 与类别场景强相关。

进入小物池还要额外满足：

- 成组、带盒、带牌、带盘、带束，或天然轮廓明显。

进入条件池还要额外满足：

- 必须能说明挂在哪里、贴在哪里、套在哪里、放在哪里。

- 动物本体移出物品池，避免和动物抽取逻辑冲突。
- 发光、反光、波纹、水光类变化进入风险池。

## Synthesis & Conclusions

### Executive Summary

本次头脑风暴从“盲盒物品约三分之一不适合”出发，最终把问题定位为三层结构性问题：

1. **主题层**：旧 14 类偏温馨杂货，场景覆盖不足，用户不容易判断适用图。
2. **类别层**：`large / medium / small / hanging` 容易诱导容器泛滥、小物过细、悬挂依赖过多。
3. **物品层**：旧库大量依赖 `风格前缀 + 容器/挂件尾词`，真实感和可圈选性不足。

最终推荐方案：

- 盲盒类别重构为 **20 个常见场景入口**。
- 名称使用 **`常见场景+活动/用途`**，例如 `桌面+学习`、`沙滩+度假`、`海底+潜水`。
- 每个类别下不再只靠大中小悬挂，而是使用 **五层物品池**：核心物、配套物、可见小物、条件物、风险物。
- 现有四类 UI 可暂时保留，但内容语义要重新定义。

### Item Pool Model

每个类别下建议维护：

| Pool | Purpose | Default Weight |
|---|---|---|
| core_items | 最能代表场景、边界清楚、优先抽取 | 35-40% |
| support_items | 丰富场景的清晰配套物 | 35-40% |
| visible_small_items | 小但可见，成组/块状/有承载 | 15-20% |
| conditional_items | 悬挂、墙面、软布、可穿戴等条件物 | 0-10% |
| blocked_or_risky | 默认不抽，作为禁用/风险清单 | 0% |

### Compatibility With Existing Tool

若短期不改 UI：

- `large` -> 场景锚点物，不能再等同于大容器。
- `medium` -> 主力清晰物，应该成为最健康的主池。
- `small` -> 可见小物，禁止微型痕迹和抽象标签凑数。
- `hanging` -> 条件物，低数量、低权重，不必硬凑 100。

### Quality Rules

每个核心物至少满足：

1. 一眼能识别。
2. 边界清楚，可单独圈选。
3. 不依赖光影、反射、透明效果。
4. 不需要特殊承载面，或承载关系明确。
5. 不是单点、细线、微型痕迹。
6. 与类别场景强相关。

### Main Rejections

- 不建议继续用 14 个旧主题原样扩写。
- 不建议每个主题四类各 100 个硬凑。
- 不建议小物大量使用 `迷你/小巧/标签/样片/配色/扣/夹`。
- 不建议悬挂类继续大量使用 `帘/串/流苏/边巾/细绳`。
- 不建议海底类混入水杯、糖碟、香罐、镜盒等生活杂货。
- 不建议物品池混入动物本体；动物应继续走动物抽取逻辑。

### Recommended Implementation Path

1. **第一步：建立新类别表**  
   先把 20 个类别作为目标内容结构，不急着改 UI。

2. **第二步：选 3 个样板类别重写**  
   推荐：`桌面+学习`、`海底+潜水`、`公园+野餐`。  
   分别代表日常桌面、特殊主题、户外场景。

3. **第三步：按五层物品池写样板数据**  
   每类先写 40-60 个精品物，不追求 100 个。

4. **第四步：回填到现有四栏结构**  
   为兼容工具，把五层池映射回 large/medium/small/hanging。

5. **第五步：试用并记录“不适合”原因**  
   判断问题来自主题、物品类型、具体词，还是目标图不匹配。

6. **第六步：再扩展剩余类别**  
   不要一次性重写 5600 个物品。
