# 主图提示词 Agent

本目录只负责主图 AI 提示词链路，不负责 `Game content extraction/` 桌面工具，也不负责组图规则。

## 入口

- `主图 第一步.md`：判断 Target 是否需要新增内容；需要时只输出盲盒类别编号、动物字段、禁用字段和补偿字段。
- `主图 第二步 .md`：审核和摆放候选物；用户给了候选就不得脑补新候选。若同时提供第一步结果，可作弱参考，但不得压过用户明确给出的候选；当前只使用 `核心物品 / 配套物品 / 散落小型物品 / 场景扩展物` 四类，按稳定承载面上的可见物体处理。

## 使用顺序

1. 通常先执行 `主图 第一步.md`，判断场景是否需要新增物品。
2. 若用户已经明确提供候选，可直接进入 `主图 第二步 .md`；第一步结果若同时提供，只作弱参考。
3. 第二步若收到用户候选，只能审核、轻改和摆放这些候选，不得补新物。

## 硬规则

- 禁止脑补不可见对象、画外空间、遮挡后区域或身份不稳定对象。
- 新增、异物植入或替换为更大物体前，必须确认 Target 有完整可见且未被占用的可放置空间。
- 不改人物骨架、姿态、头身朝向、四肢动作、支撑关系和贴邻热区。
- 不做灯光、发光、明暗、阴影、高光、反光、滤光、投影或镜面成像变化。
- 位置字段只定位，不写颜色、材质、纹理、风格、明暗等外观修饰。
- 第二步中，`摆放内容` 与 `修改` 可自然补少量颜色、材质、状态描述，用来增强生活感，但不要堆砌修饰。
- 第二步默认围绕稳定承载面做摆放设计，优先自然、清晰、可落地的可见物体。
- 主图第二步中，用户给了候选就不得脑补新候选；放不下就舍弃，不允许硬塞。

## 边界

- 主图提示词只管理主图新增与摆放，不维护盲盒数据事实源。
- 盲盒类别、四池写库和审查修改规则，统一以 `../../Game content extraction/data/agents.md` 及同目录规则文档为准。
- 组图差异、表情前置和表情库继续留在仓库根目录，对应入口见 `../../agents.md`。
