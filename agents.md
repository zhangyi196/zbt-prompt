# Agents Guide

本文件为进入该仓库工作的 AI 代理提供最小维护上下文。适用范围：仓库根目录下全部 Markdown 提示词文档。

## 项目概览

本仓库维护中文提示词规范，服务卡通休闲找不同流程：场景候选物审核、差异点生成、Ref 变化迁移、剧情与人物表情前置设计。当前不是代码项目，无构建脚本、测试框架或运行入口；主要工作是编辑 Markdown 中的规则、输出格式、自检和版本说明。

## 文件职责

- `主图 第二步 .md`：场景候选物审核与摆放设计。基于 Target 和 0-6 个候选内容完成扫描、审核、有限修改与摆放；只有用户完全未提供候选时才可自行补 6 个，只要提供至少 1 个候选就不得补全或猜测新候选。
- `组图 4.md`：找不同关卡润色。基于 Group 2 Image 设计 18 个 Group 3 差异点；九宫格每区 2 个，总数 18，类型配额 3 / 6 / 3 / 3 / 3，禁止灯光、发光、明暗、反光、高光、阴影等变化与词汇。
- `组图 23.md`：Ref-A / Ref-B / Target 差异迁移。只绑定 Ref 真实可见变化，迁移到 Target，输出固定六板块；List 1 固定 16 条，List 2 固定 16 条；同一人物面部表情区与衣领/领带/颈部、头发/头饰、耳饰/耳侧饰品互斥，且高于 Ref 复刻。
- `组图 23 表情前置.md`：`组图 23.md` 的上游辅助文档。只输出剧情预案、人物表情预案、交给组图 23 的摘要和自检；不得输出完整差异列表、List 1、List 2 或热区图；表情库保持类别表，正向 25 个、负向 25 个，不写详细眉/眼/嘴模板；多人场景优先错开情绪极性。

## 编辑原则

1. 保持中文提示词风格，优先局部修改，不做无关重排。
2. 保留 Markdown 层级、编号、粗体重点、`[CRITICAL]` 标记、固定输出格式和固定句式。
3. 不删除硬性禁令、自检清单、数量配额、输出格式示例，除非用户明确要求。
4. 调整行为边界时，同步检查输入规则、流程、输出格式和自检，避免前后矛盾。
5. 文档保存为 UTF-8；中文文件名命令行操作使用引号或 `-LiteralPath`。
6. 若用户要求发布新版、记录版本变化，或修改属于明显规则升级，应更新标题版本号；轻微错字和局部措辞可不改版本号。

## 关键不变量

- 禁止脑补不可见对象、画外空间、遮挡后区域或身份不稳定对象。
- 差异点遵守“一圈一物一变化”或“单一不同点”原则。
- 不修改人物骨架、动作姿态、头身朝向、四肢动作或支撑关系。
- 不做灯光、发光、明暗、阴影、高光、反光、滤光、投影、镜面成像等变化。
- 不整体修改大面积背景、大物体主体或大面积表层。
- 位置字段只用于定位，不写颜色、材质、纹理、风格、明暗等外观修饰。
- 不把小变化升级成灾难、阴谋、案件、长期危机或大型事故。
- 数量配额、固定板块、字段名和自检覆盖必须保持一致。

## `组图 23` 专项约束

- `组图 23.md` 只以 Ref-A → Ref-B 的真实可见变化绑定事件功能；前置预案可参考，但不得覆盖 Ref 真实性、Target 可见性、一圈一物一变化、人物区域互斥和禁改规则。
- 同一人物若使用面部表情区，不得再修改其衣领/领带/颈部、头发/头饰、耳饰/耳侧饰品；若不修改面部表情，才可选择这些附属区域。
- 表情嘴部可承载 Ref 明确可见且剧情需要的物体或状态，如嘴巴塞满食物、嘴里叼着物体、嘴角夹着纸条、嘴边露出碎屑；不得脑补来源、动作过程或额外剧情。
- `组图 23 表情前置.md` 的表情库必须保持精简类别表：正向 25 个、负向 25 个，只写表情类别和适合场景。
- 多人表情若有 2 个以上可安全落地人物，应优先使用不同情绪极性并避免重复类别；该策略不压过 Ref 和 Target 可见性。

## 工作流程

修改前：阅读目标文件版本号、角色、输入规则、核心约束和输出格式；明确本次属于补充规则、收紧禁令、调整格式、修正文案、消除矛盾或版本升级；搜索同类规则位置，避免只改一处造成冲突。

修改中：只编辑与需求直接相关的段落；保留用户提供内容边界；避免新增与现有规则冲突的术语；不要在最终输出要求中加入“思维链”“候选池”“内部推理过程”等内容，流程说明应明确为内部执行且不要输出。

修改后：通读受影响章节，检查数量、格式、术语、优先级、禁区、承载关系、修改边界和自检是否一致；必要时检查 `组图 23.md` 与 `组图 23 表情前置.md` 的接入关系是否同步。

## 提交前检查

- UTF-8 中文 Markdown 正常。
- 用户内容边界、禁止脑补、灯光/成像禁令、人物姿态限制、大面积背景稳定限制仍在。
- 固定板块、列表条数、类型配额、字段名和自检项未失配。
- `组图 23.md` 人物互斥规则、嘴部可见物规则和 Ref 绑定规则未被削弱。
- `组图 23 表情前置.md` 仍只输出四板块，表情库仍为正向 25 / 负向 25 的类别表，多人极性错开规则仍在。
- 位置字段没有被写成外观描述字段。
