# Agents Guide

本文件为仓库内 Markdown 提示词维护提供最小上下文。本仓库不是代码项目，无构建、测试或运行入口；主要工作是编辑中文提示词的规则、输出格式、自检与版本说明。

## 文件职责

- `主图 第二步 .md`：场景候选物审核与摆放。基于 Target 和 0-6 个候选内容扫描、审核、有限修改；用户只要提供至少 1 个候选，就不得补全或猜测新候选。
- `组图 4.md`：找不同关卡润色。基于 Group 2 Image 设计 18 个 Group 3 差异点；九宫格每区 2 个，类型配额 3 / 6 / 3 / 3 / 3；禁止灯光、发光、明暗、反光、高光、阴影等变化与词汇。
- `组图 23 表情库.md`：表情抽取 App 资料源。固定正向 25 个、负向 25 个；每个表情 8 个眉/眼/嘴短模板；1-4 条为单人模板，5-8 条为多人模板；只写面部，不改头身朝向、姿态、四肢、衣领、头发或耳饰；模板保持静态可见结果，避免过程词、光影词、夸张张口和越界到脸颊/头身附属物。
- `组图 23 表情前置.md`：`组图 23.md` 上游辅助文档。只输出负向剧情表情组、正向剧情表情组、交给组图 23 的摘要和自检；每组包含极性、剧情、单人/多人、具体表情类别、人物定位、表情功能和适配提示；内置正向 25 个、负向 25 个简洁编号表情类别，只从上方类别表选择；表情库引用处只写简洁类别，不引入或链接 `组图 23 表情库.md`；不输出 List 1 / List 2、完整差异列表、热区图或表情库完整眉/眼/嘴变体。
- `组图 23.md`：Ref-A / Ref-B / Target 差异迁移。绑定 Ref 真实可见变化并迁移到 Target，固定输出六板块；List 1 固定 16 条，List 2 固定 16 条；若用户提供剧情文字和人物表情描述，必须按剧情道具、证据、目标或对方人物改写泛化表情；当前版本为压缩版，维护时不得为压缩而削弱固定输出接口、冷区避让、人物互斥和禁改规则。

## 通用不变量

- 保持中文提示词风格，优先局部修改，不做无关重排。
- 保留 Markdown 层级、编号、粗体重点、`[CRITICAL]` 标记、固定输出格式、固定句式、数量配额和自检项。
- 文档保存为 UTF-8；中文文件名命令行操作使用引号或 `-LiteralPath`。
- 规则升级、发布新版或记录版本变化时更新标题版本号；轻微错字和局部措辞可不改版本号。
- 禁止脑补不可见对象、画外空间、遮挡后区域或身份不稳定对象。
- 差异点遵守“一圈一物一变化”或“单一不同点”。
- 不修改人物骨架、动作姿态、头身朝向、四肢动作或支撑关系。
- 不做灯光、发光、明暗、阴影、高光、反光、滤光、投影、镜面成像等变化。
- 不整体修改大面积背景、大物体主体或大面积表层。
- 位置字段只用于定位，不写颜色、材质、纹理、风格、明暗等外观修饰。
- 不把小变化升级成灾难、阴谋、案件、长期危机或大型事故。

## `组图 23` 专项

- `组图 23.md` 只以 Ref-A → Ref-B 的真实可见变化绑定事件功能；前置预案、剧情文字和人物表情描述都只能作为参考，不得覆盖 Ref 真实性、Target 可见性、一圈一物一变化、人物区域互斥和禁改规则。
- 同一人物若使用面部表情区，不得再修改其衣领/领带/颈部、头发/头饰、耳饰/耳侧饰品；若不修改面部表情，才可选择这些附属区域。
- `组图 23.md` 接收表情库结果时，单人表情眼部回指证据物/目标物；多人表情眼部可回指对方人物，`[对方人物]` 只用于眼部视线或眼神，不得作为嘴部承载物。
- 表情嘴部可承载 Ref 明确可见且剧情需要的物体或状态，如嘴巴塞满食物、嘴里叼着物体、嘴角夹着纸条、嘴边露出碎屑；不得脑补来源、动作过程或额外剧情。
- 三段流程：`组图 23 表情前置.md` 给负向/正向两组剧情表情预案与简洁表情类别 → App 按表情类别抽取具体眉/眼/嘴 → `组图 23.md` 按剧情适配并生成完整差异方案；前置文档和前置输出不得把表情库 md 文件作为引用入口。
- `组图 23 表情库.md` 必须保持正向 25 个、负向 25 个，且每个表情固定 8 个眉/眼/嘴短模板；1-4 条列为单人并看向或避开 `[证据物]` / `[目标物]`，5-8 条列为多人并看向或避开 `[对方人物]`；每条模板至少保留剧情回指占位符，嘴部只可回指 `[证据物]`、`[目标物]`、`[剧情食物]` 或 `[剧情小物]`。
- `[剧情食物]`、`[剧情小物]` 只允许作为嘴部承载物出现在适合剧情证据的表情类别中；其他类别嘴部只写抿住、收住、偏侧笑线、横向扁开、发颤等面部状态。
- 多人表情若有 2 个以上可安全落地人物，应优先使用不同情绪极性并避免重复类别；该策略不压过 Ref 和 Target 可见性。

## 修改检查

修改前：阅读目标文件版本号、角色、输入规则、核心约束和输出格式；搜索同类规则位置，避免只改一处造成冲突。

修改中：只编辑与需求直接相关的段落；保留用户内容边界；避免新增与现有规则冲突的术语；不要在最终输出要求中加入“思维链”“候选池”“内部推理过程”等内容。

修改后：通读受影响章节，检查数量、格式、术语、优先级、禁区、承载关系、修改边界和自检是否一致；涉及表情链路时，同步检查 `组图 23 表情库.md`、`组图 23 表情前置.md` 和 `组图 23.md` 的接入关系。

提交前确认：UTF-8 中文正常；禁止脑补、灯光/成像禁令、人物姿态限制、大面积背景限制仍在；固定板块、列表条数、类型配额、字段名和自检项未失配；`组图 23.md` 的人物互斥、嘴部可见物、Ref 绑定和用户表情剧情化适配规则未被削弱。
