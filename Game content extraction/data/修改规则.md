# 盲盒物品库修改规则

本文档用于在 `审查规则.md` 审查完成后，指导如何替换 `blind_boxes.py` 中的问题物品。本文档优先提供可跨类别复用的通用修改框架；类别差异只作为附加边界处理，不能反过来覆盖全局硬规则。

## 适用范围

- 本规则适用于全部盲盒类别，不只服务 `桌面+学习`、`餐桌+茶歇`、`厨房+烘焙`、`卧室+梳妆`。
- 修改阶段先走通用规则，再看目标类别的附加边界；不要一开始就按单个类别特例选物。
- 文中示例只用于说明问题模式，不是白名单，也不是只拦示例本身。

## 修改目标

- 替换后的物品必须真实、常见、可见、符合场景，并适合找不同游戏。
- 修改必须解决审查指出的根因，而不是只改表面词。
- 常见度优先于测试边界；某个名称即使能通过自动测试，只要普通用户不会自然觉得它属于该场景，也必须替换。
- 第四池 `scene_expansion_items` 必须先通过场景自然度、前台可见性、非后场设备和非同族扩写检查；能通过中型边界测试不等于能入库。
- 第四池定义重构为“场景补充中型物”，不是“辅助器具池”；不得为了把小物中型化而生成 `抽取器 / 托架 / 收纳座 / 保温座 / 分区盒 / 立筒 / 挂架 / 临放盘` 等二级器具。

## 通用修改框架

所有修改项都先按下面五层顺序判断：

1. `场景语境层`
   这个物品是不是当前场景前台自然出现的东西，而不是后场、维护区、仓储区、专业操作区或安装结构的一部分。
2. `视觉本体层`
   如果把名称遮掉只看画面，它和同池已有物是不是仍像同一种东西。
3. `载体模板层`
   去掉功能词、对象词、内容物词后，剩下的是不是只剩 `垫`、`卷`、`浅篮`、`立座`、`盒`、`册`、`盘` 这类通用载体。
4. `四池角色层`
   它是不是放错了池，或者把前三池已有基础对象换个后缀后又写进第四池；第四池必须是弱主角场景物，不是小物的支架、座、筒、抽取器。
5. `可玩性层`
   它能不能作为一个清晰、稳定、单点可圈选的差异物，而不是靠细节、文字或微小变化成立。

## 根因标签

审查文档和修改动作都优先落到下面这些通用根因标签，便于跨类别复用：

- `场景错位`：真实存在，但不属于当前场景常见语境。
- `前台 / 后场错位`：更像清洁、维护、冷却、回收、封口、晾放、仓储或专业制备阶段物。
- `视觉本体重复`：名称不同，但画出来仍主要是同一种物。
- `载体模板化`：主要靠 `盒 / 袋 / 筒 / 垫 / 卷 / 册 / 盘 / 架 / 座 / 篮` 之类通用载体撑住名称。
- `跨池扩写`：前三池已有基础对象，第四池只是加上 `架 / 座 / 盒 / 筒 / 托盘 / 器 / 机` 再写一次。
- `二级器具化`：物品本体不是场景物，而是为了装、托、抽、挂、固定或保护另一个小物而存在。
- `功能词造词`：先有功能，再硬套 `板 / 盘 / 垫 / 盒 / 包 / 册 / 卷`。
- `可玩性弱`：体积太小、太薄、太依赖文字，或变化不够清楚。
- `国际化不稳`：强地域、强课程体系、强文化背景绑定，默认国际化用户不自然。

## 修改流程

1. 先读取 `agents.md`、`审查规则.md`、本文件、`审查文件.md` 索引和 `审查文件/{类别} 审查文档.md`。
2. 确认对应类别审查文档的审查对象与本次要修改的类别一致；若为空、过期或类别不一致，先重新审查并更新该类别文档，不得直接修改。
3. 按审查文档逐项处理，并为每个问题补齐根因标签：至少明确 `视觉本体族`、`问题模式`、`场景风险` 三项。
4. 删除同族中的低价值变体，不得只在同一模板内横向换前缀。
5. 为候选替换项执行“五层复审”：场景语境、视觉本体、载体模板、四池角色、可玩性。
6. 从不同物品族补入替换项，优先选择轮廓、体积、功能和摆放方式明显不同的真实物品。
7. 替换完成后重新检查四池数量、唯一性、跨池重复、第四池边界、尾词密度、场景常识和自然度。
8. 修改后回写对应类别审查文档的“处理状态”和“修改后复审”，记录已替换项、保留风险和剩余问题。

## 审查文件读取规则

- `审查文件/{类别} 审查文档.md` 是修改阶段的任务单；不得只根据聊天上下文或个人印象修改。
- 每个替换动作必须能对应到审查文档中的问题编号，或能明确归到某个 `视觉本体族 + 问题模式`。
- 若修改过程中发现审查文档遗漏了明显问题，先补充审查文档，再继续修改。
- 如果某项决定保留，必须在审查文档中写明保留理由和风险，不得静默跳过。
- 修改完成后，审查文档应能回答：哪些问题已解决、哪些问题仍保留、复审是否通过。

## 通用替换判定

以下情况必须替换，不能只改名称。下面写的是问题模式，不限于示例：

- `同视觉本体换叫法`
  如 `螺旋笔记本`、`硬壳笔记本`、`横线练习簿`，画出来仍主要是一本本子。
- `同用途换前缀`
  如 `文件托盘`、`信件托盘`、`打印纸托盘`，本质仍是托盘。
- `同内容物换包装`
  如 `单词闪卡盒`、`公式闪卡盒`、`数学闪卡盒`，本质仍是闪卡盒。
- `同材质换外壳`
  如 `金属书立`、`木质书立`、`塑料书立`，若轮廓近似，只留一个。
- `同功能换对象`
  如 `模具清洁刷`、`烤盘清洁刷`、`粉筛清洁刷`，本质仍是清洁刷。
- `同处理阶段换对象`
  如 `糕点冷却架`、`饼干晾凉篮`、`模具晾放架`、`美妆蛋干燥架`，都属于冷却、晾放或后处理承托。
- `同专业设备换场景`
  如 `真空封口机`、`糖浆压泵器`、`杏仁研磨机`、`奶油起泡器`，若更像后厨、吧台、工坊或专业制备台设备，应替换。
- `同载体换功能`
  如 `水杯隔热垫`、`防滑书写垫`、`绘图切割垫`，去前缀后仍主要是垫子。
- `同卷材换内容`
  如 `护书膜卷`、`包书纸卷`、`擦拭布卷`，去前缀后仍主要是卷材。
- `同浅盘 / 浅篮换对象`
  如 `画材浅篮`、`文具浅篮`、`零食小篮`，或 `岩石样本盘`、`零食小盘`，去前缀后仍是同一浅层承托物。
- `同立座 / 展示座换对象`
  如 `照片展示座`、`奖牌展示座`、`眼镜立座`，去前缀后仍只是立座或展示座。
- `同二级器具换对象`
  如 `杯套抽取器`、`餐盘托架`、`果盘托架`、`奶壶保温座`，本体不是好玩的场景物，而是另一个小物的取用、支撑或保温配件。
- `同册状载体换内容`
  如 `植物标本册`、`配色样本册`、`剪贴素材册`，若差异主要依赖内容说明而不是外形，应视为同一载体族。
- `场景常识不匹配`
  真实存在，但普通用户不会自然把它归到当前场景。

## 五问复审

修改时必须问五个问题：

- 普通玩家看到它，会不会觉得“这就是这个场景里的东西”？
- 它会不会自然出现在当前画面的前台，而不是清洁、维修、储存、冷却、晾干、封口、回收或后场处理阶段？
- 如果把名字遮掉只看图，它和同池已有物是不是还是同一种东西？
- 去掉前缀后，它是不是其实只剩 `垫`、`卷`、`浅篮`、`立座`、`盒`、`册`、`盘` 这类通用载体名字？
- 它本身是不是一个自然物件，而不是用来装、托、抽、挂、固定另一个小物的二级器具？
- 它是否比被替换物更自然、更常见、更好画，而不是更专业、更小众、更别扭？

若答案不稳，优先替换。以下内容属于高风险：

- 为了过测试硬套后缀的物品，如 `书包挂放座`、`日程翻页架`、`夹板收纳篮`。
- 抽象学习管理物或难以直观看懂的物品，如 `索引分页册`、`复习卡翻页册`。
- 普通家庭桌面低频的技术设备，如 `USB转接器`、`USB集线器`、`桌面扩展坞`。
- 与当前用途弱相关的舒适 / 环境设备，如 `白噪音机`，除非类别明确需要睡眠或环境声。
- 过于办公化、教室化、工坊化或后场化的设备，即使体积合适也不优先写入。
- 使用后处理或后场维护物，如 `糕点冷却架`、`模具晾放架`、`刷具晾放架`、`美妆蛋干燥架`、`桌面废纸桶`，除非该类别明确以清洁维护为主。
- 专业化或低频加工设备，如 `真空封口机`、`糖浆压泵器`、`杏仁研磨机`、`咖啡敲渣桶`，即使真实存在，也不能作为普通桌面或家庭场景的默认替换项。
- 去掉前缀后只剩通用载体的命名，如 `xxx垫`、`xxx卷`、`xxx浅篮`、`xxx立座`、`xxx样本册`。
- 二级器具化命名，如 `杯套抽取器`、`餐盘托架`、`果盘托架`、`奶壶保温座`、`方糖夹托座`、`护肤勺立筒`。

## 裸名复审规则

“去前缀裸名复审”是本文件的通用规则，默认适用于全部类别。

- 去掉 `对象词`
  如 `画材`、`文具`、`零食`、`照片`、`奖牌`、`岩石样本`。
- 去掉 `功能词`
  如 `隔热`、`防滑`、`切割`、`展示`、`记录`、`练习`、`整理`、`分区`、`收纳`、`保温`。
- 去掉 `内容物词`
  如 `护书膜`、`包书纸`、`糖包`、`茶包`、`样本`、`素材`。

复审后若主要只剩：

- `垫`
- `卷`
- `浅篮` / `小篮`
- `浅盘` / `小盘`
- `立座` / `展示座`
- `盒`
- `册`
- `盘`
- `板`
- `筒`
- `袋`
- `抽取器`
- `托架`
- `收纳座`
- `保温座`
- `立筒`
- `挂架`
- `临放盘`

则默认视为模板化高风险项。只有在下面两种情况下才允许保留：

- 这个名称本身就是现实里高频、稳定、独立的常用物品名。
- 同池内没有同载体族堆叠，且它在当前场景里确实比其他候选项更自然。

## 视觉本体族控制

修改时必须先按视觉本体族分组，再决定保留数量。下面的配额是通用上限，不是必须写满：

- `书 / 本 / 册 / 簿 / 记录本`：每池最多 2-3 个代表项。
- `文件承载族`
  如夹、袋、包、文件盘、资料盘，每池最多 1-2 个代表项。
- `平面承托族`
  如垫、板、盘，每池最多 1-2 个真正轮廓差异明显的代表项；不能用 `隔热`、`书写`、`切割`、`练习`、`展示` 等功能词继续扩写。
- `卷材族`
  如膜卷、纸卷、布卷，每池最多 1-2 个代表项；不能靠内容物不同重复占位。
- `浅层容器族`
  如浅篮、浅盘、小篮、小盘，每池最多 1 个代表项；不能靠对象词换前缀批量补位。
- `立座 / 展示座族`
  每池最多 1 个代表项；若只是把已有物品放到一个座上，不优先写入。
- `册状载体族`
  如样本册、素材册、标本册，每池最多 1 个代表项；若主要差异依赖内容说明而非外形，优先换成其他物品族。
- `电子小设备族`
  每池最多 1 个代表项；非电脑桌、吧台、后厨或专业台面场景原则上不写。
- `二级器具族`
  如抽取器、托架、收纳座、保温座、立筒、挂架、临放盘，第四池原则上不写；只有当它本身就是该场景非常常见、玩家一眼能认出的独立物品时才可少量保留。
- `袋、盒、篮、架、座、刷等容器或辅助族`
  不得成片堆叠；同一池若某族已经超额，新增替换项必须来自其他物品族，而不是继续换前缀。

## 场景语境规则

修改时优先判断“它属于哪里”，而不是“它存在不存在”。

通用判断：

- `不是后场`
  默认不写冷却、晾放、沥水、封口、回收、废弃、维护、清洁后处理物。
- `不是专业台面`
  默认不写工业后厨、咖啡吧台、专业影棚、实验台、器材仓、教师讲台风格设备。
- `不是固定结构`
  默认不写需要安装、依附、钩挂、贴合到空间结构上才成立的东西。
- `不是纯收纳壳`
  如果物品主要价值只是“把别的东西装起来”，而不是自身能成为稳定单物，优先替换。
- `不是二级辅助器具`
  如果物品主要价值只是抽取、托住、挂住、固定、保温或保护另一个更小的对象，优先替换为场景中自然存在的普通物件。
- `不是文化门槛物`
  默认面向欧美 / 国际化用户，除非类别明确要求，不写强中式、强地域、强课程体系绑定物。

## 类别附加边界示例

下面这些是常用类别的附加边界，属于“在通用规则之上再加一层”，不是通用规则本体：

- `桌面+学习`
  不是教室、办公室或电脑外设桌；不写教师设备、专业文件设备、低频电脑外设、抽象学习管理物和奇怪页面固定件。
- `餐桌+茶歇`
  不是咖啡店吧台或清洁后场；不写敲渣、测温、压泵、刮水、茶渍清理、杯口擦拭等维护或制备物。
- `厨房+烘焙`
  不是工业后厨、储藏处理区或洗碗晾干区；不写真空封口机、成串冷却 / 晾放架、专业加工机、清洁废弃桶和硬套护具收纳。
- `卧室+梳妆`
  不是影棚、床头电器区或美妆工具仓；不写干燥晾放架、医疗 / 睡眠设备、衣物护理后场物和泛电子杂物。

## 替换方向

优先选择这些替换方向：

- `换本体`
  用球体、盒体、卷状、瓶罐、软包、厚实工具、小设备等不同轮廓补位。
- `换摆放方式`
  用平放、竖放、立放、卷放、靠放、筒装、篮装的差异拉开视觉距离。
- `换使用角色`
  从阅读、记录、测量、计时、取用、承托、保护、收纳、整理等不同角色里找替换项。
- `换阶段`
  从“后处理物”换成“使用中前台物”，通常更自然。
- `换成物件本身`
  从 `杯套抽取器`、`餐盘托架`、`奶壶保温座` 这类二级器具，换成小奶壶、糖罐、点心罩、食谱架、香水瓶、首饰盒这类自然场景物。

避免这些替换方式：

- 用另一个 `xxx盒` 替换一个 `xxx盒`。
- 用另一个 `xxx托盘` 替换一个 `xxx托盘`。
- 用另一个 `xxx垫` 替换一个 `xxx垫`，即使功能词不同。
- 用另一个 `xxx卷` 替换一个 `xxx卷`，即使内容物不同。
- 用另一个 `xxx浅篮`、`xxx小篮`、`xxx立座`、`xxx展示座` 替换一个同类载体。
- 用另一个 `xxx样本册`、`xxx素材册`、`xxx标本册` 替换一个同类册状载体。
- 用同一基础物的材质、颜色、尺寸、封面或用途变体替换。
- 用更专业、更怪、更难识别的设备补位。
- 用通过测试但不自然的 `xxx器`、`xxx座`、`xxx架`、`xxx篮` 补位。
- 用二级器具补位，例如 `xxx抽取器`、`xxx托架`、`xxx保温座`、`xxx收纳座`、`xxx立筒`、`xxx挂架`。
- 用后场处理物补位，例如冷却、晾干、封口、回收、废弃、维修、清洁维护类物品。

## 四池修改规则

### `core_items`

- 保留能代表该类别的主体物，但同族不应撑满核心池。
- 核心物必须是普通用户自然能想到的主物，不要引入过专业设备凑数。

### `support_items`

- 写服务核心物的辅助件，但不能把核心物换小名重写。
- 单个微小物必须升级为可见单位。
- 同类工具成组出现时，要控制族密度，不让某一种辅助件独占整池。

### `visible_small_items`

- 小物必须是一盒、一包、一筒、一套、一卷、一束、一叠、一排等可见单位。
- 单位要自然，例如 `一套圆规` 比 `一包圆规` 更合理。
- 不要让同一种包装单位或同一种小物族连续堆叠。

### `scene_expansion_items`

- 只写场景补充中型物：弱主角、前台自然、玩家一眼能认出的普通物件。
- 不能是组合，不能是核心物小变体，不能是大型家具或固定装置，也不能是把小物硬撑成中型的二级器具。
- 第四池最容易发生托盘、架、座、盒、垫、刷、筒、抽取器、托架、保温座伪重复；修改后必须按视觉本体族和二级器具风险复审。
- 若某项虽然通过测试但场景常识弱，也必须替换。
- 第四池新增物必须先通过四重准入：
  - `前台可见`：像会摆在当前画面里的东西，而不是后厨、仓储、清洁、维修、晾干或废弃处理物。
  - `当前阶段`：服务当前场景的使用或摆放状态，不写使用后处理阶段，如冷却、晾放、干燥、沥水、封口、回收。
  - `非专业设备`：普通家庭和国际化用户一眼能理解；偏吧台、工坊、工业后厨、影棚或专业护理的设备默认替换。
  - `非同族扩写`：若前三池已有同视觉本体或同功能物，第四池不得用 `架/座/盒/筒/托盘/抽取器/托架` 把它中型化后再写一次。
  - `非二级器具`：它本身必须是场景物，而不是另一个小物的取用器、支撑架、保温座、收纳壳或临放盘。
- 第四池新增物必须执行“去前缀裸名复审”：若去掉对象词和功能词后，剩下的主要只是 `垫`、`卷`、`浅篮`、`立座`、`盒`、`册`、`盘`、`抽取器`、`托架`、`保温座` 这类通用载体或二级器具，默认判为模板化，除非它本身就是该场景非常高频且不可替代的代表物。
- 对 `器/机/架/座/筒` 结尾的候选项执行反向举证：必须能说明它在该类别画面中足够常见、独立、自然，并且不是为了服务另一个更小物件而存在；无法说明则替换为更朴素的场景物。
- 第四池优先选“自然前台物件”，谨慎选“制备 / 加工 / 清洁 / 维护 / 收纳后场设备”，原则上不选“抽取 / 托架 / 保温 / 临放 / 挂放”类二级器具。

第四池替换时优先回到物品本身：

- 餐桌茶歇：用小奶壶、糖罐、蜂蜜罐、黄油碟、小冰桶、点心罩、饼干罐、面包篮、餐巾架，替代 `杯套抽取器`、`餐盘托架`、`果盘托架`、`奶壶保温座`。
- 厨房烘焙：用食谱架、厨房计时钟、撒粉器、刮板、擀面杖、面团转移板、蛋糕铲、饼干压模、烤箱手套，替代真空封口机、成串晾干架、清洁刷和硬套 `xxx座/xxx架/xxx盒`。
- 卧室梳妆：用化妆镜、香水瓶、首饰盒、发饰盒、化妆包、纸巾盒、棉签筒、扩香瓶、台面小钟，替代 `护肤勺立筒`、`美妆蛋托座`、`发带挂架`、`临放盘` 等二级器具。

## 审查文档回写模板

每个修改项都尽量回写下面五项，让规则能跨类别稳定复用：

- `问题本体族`
  例如本子族、浅盘族、卷材族、立座族、电子外设族。
- `问题模式`
  例如视觉本体重复、载体模板化、前台 / 后场错位、跨池扩写。
- `场景风险`
  例如后厨化、吧台化、教室化、床头电器化、文化门槛过高。
- `替换方向`
  说明要换到哪一种不同物品族，而不是只说“换一个更合适的”。
- `复审结论`
  记录是否解决根因，是否还有剩余风险。

## 修改后复审

替换完成后必须回答：

- 问题物品的根因是否真的被解决？
- 是否又产生新的同族堆叠？
- 是否引入了不符合场景常识的专业或怪异物品？
- 第四池新增项是否自然处于画面前台，而不是后场处理、冷却晾放、封口储存、清洁维护或专业制备阶段？
- 是否只是把旧的 `盒 / 袋 / 盘 / 板 / 架 / 座 / 垫 / 刷` 换成新的同类模板？
- 去掉前缀后，这批新增项是否仍然只剩 `垫`、`卷`、`浅篮`、`立座`、`盒`、`册`、`盘` 之类的通用载体？
- 是否只是把前三池已有物品加上 `架 / 座 / 盒 / 筒 / 托盘 / 器 / 机` 后重新放进第四池？
- 第四池是否仍有 `抽取器 / 托架 / 收纳座 / 保温座 / 分区盒 / 立筒 / 挂架 / 临放盘` 成片出现？
- 第四池每个新增项是否能作为“物件本身”成立，而不是依赖另一个小物才能解释用途？
- 每个新增物品是否有独立轮廓，能被玩家当作一个清晰差异点？
- 每池是否仍为 50 条，池内唯一，第四池无跨池精确重复？

若任一答案不合格，继续替换，不应进入最终写库。
