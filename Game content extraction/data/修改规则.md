# 盲盒物品库修改规则

本文档用于在 `审查规则.md` 审查完成后，指导如何替换 `blind_boxes.py` 中的问题物品。审查负责发现问题并写入 `审查文件.md`，修改负责读取 `审查文件.md` 后重新选物；不能只把不合格名称换成另一个同模板名称。

## 修改目标

- 替换后的物品必须真实、常见、可见、符合场景，并适合找不同游戏。
- 修改必须解决审查指出的根因，而不是只改表面词。
- 按“视觉本体”去重：画出来像同一种东西的物品，不应靠用途、材质、封面、内容物或前缀重复占位。
- 常见度优先于测试边界；某个名称即使能通过自动测试，只要普通用户不会自然觉得它属于该场景，也必须替换。
- 每次替换后必须二次审查，确认没有产生新的伪重复、场景错位或尾词堆叠。

## 修改流程

1. 先读取 `agents.md`、`审查规则.md`、本文件和 `审查文件.md`。
2. 确认 `审查文件.md` 的审查对象与本次要修改的类别一致；若为空、过期或类别不一致，先重新审查并更新审查文件，不得直接修改。
3. 按 `审查文件.md` 的问题清单逐项处理，优先处理 `待处理` 项。
4. 针对每个问题物品读取其“视觉本体族”和“修改方向”，删除或替换同族中低价值的变体。
5. 对每个候选替换项做“常识自然度”判断：它是否像普通用户会在该场景中看到的东西，而不是办公、专业、工具房或硬凑后缀的物品。
6. 从不同物品族补入替换项，优先选择轮廓、体积、功能和摆放方式明显不同的真实物品。
7. 替换完成后重新检查四池数量、唯一性、跨池重复、第四池边界、尾词密度、场景常识和自然度。
8. 修改后回写 `审查文件.md` 的“处理状态”和“修改后复审”，记录已替换项、保留风险和剩余问题。

## 审查文件读取规则

- `审查文件.md` 是修改阶段的任务单；不得只根据聊天上下文或个人印象修改。
- 每个替换动作必须能对应到审查文件中的问题编号或视觉本体族。
- 若修改过程中发现审查文件遗漏了明显问题，先补充审查文件，再继续修改。
- 如果某项决定保留，必须在审查文件中写明保留理由和风险，不得静默跳过。
- 修改完成后，审查文件应能回答：哪些问题已解决、哪些问题仍保留、复审是否通过。

## 替换判定

以下情况必须替换，不能只改名称：

- 同视觉本体重复：如 `螺旋笔记本`、`硬壳笔记本`、`横线练习簿` 画出来仍主要是一本本子。
- 同用途换前缀：如 `文件托盘`、`信件托盘`、`打印纸托盘` 本质仍是托盘。
- 同内容物换包装：如 `单词闪卡盒`、`公式闪卡盒`、`数学闪卡盒` 本质仍是闪卡盒。
- 同材质换外壳：如 `金属书立`、`木质书立`、`塑料书立` 若轮廓近似，只留一个。
- 同功能换对象：如 `模具清洁刷`、`烤盘清洁刷`、`粉筛清洁刷` 本质仍是清洁刷。
- 同功能换叫法：如 `书页固定夹`、`页面压纸夹`、`书本压夹`、`书本固定夹` 都是在压住或固定页面，本质重复。
- 同设备换接口：如 `USB转接器`、`USB集线器`、`桌面扩展坞` 都是电子转接设备，除非场景明确是电脑桌，否则不应成组出现。
- 同切割/分配功能：如 `纸胶带切割器`、`胶带切割座` 本质都是胶带切割器，只能保留一个，且需确认场景常见。
- 场景常识不匹配：真实存在但不属于当前常见场景的物品，应替换。

## 常识自然度

修改时必须问三个问题：

- 普通玩家看到它，会不会觉得“这就是这个场景里的东西”？
- 它是不是只因为名称里带 `器`、`座`、`架`、`盒`、`筒` 才通过规则？
- 它是否比被替换物更自然、更常见、更好画，而不是更专业、更小众、更别扭？

若答案不稳，优先替换。以下内容属于高风险：

- 为了过测试硬套后缀的物品，如 `书包挂放座`、`日程翻页架`、`夹板收纳篮`。
- 抽象学习管理物或难以直观看懂的物品，如 `索引分页册`、`复习卡翻页册`。
- 普通家庭桌面低频的技术设备，如 `USB转接器`、`USB集线器`、`桌面扩展坞`。
- 与当前用途弱相关的舒适/环境设备，如 `白噪音机`，除非类别明确需要睡眠或环境声。
- 过于办公化、教室化、工坊化或后场化的设备，即使体积合适也不优先写入。

## 视觉本体族控制

修改时必须先按视觉本体族分组，再决定保留数量。原则上同一池内：

- 笔记本 / 练习簿 / 作业本 / 记录本：最多保留 2-3 个代表项。
- 文件夹 / 资料夹 / 报告夹 / 讲义夹：最多保留 1-2 个代表项。
- 托盘 / 信件盘 / 文件盘 / 打印纸盘：最多保留 1 个代表项。
- 笔筒 / 收纳筒 / 彩铅筒：最多保留 1-2 个代表项。
- 计时钟 / 小钟 / 定时器：最多保留 1-2 个代表项。
- 护腕垫 / 鼠标垫 / 书写垫 / 防滑垫：最多保留 1-2 个代表项。
- 页面固定 / 书页夹 / 压纸夹 / 书本压夹：最多保留 1 个代表项，且不能显得像奇怪小夹具。
- 电子转接 / 线缆整理 / 集线器 / 扩展坞：最多保留 1 个代表项；非电脑桌场景原则上不写。
- 胶带切割 / 纸胶带切割 / 胶带座：最多保留 1 个代表项。
- 袋、盒、篮、架、座、刷等容器或辅助族，不得成片堆叠。

若某一族已经超额，新增替换项必须来自其他物品族，而不是继续换前缀。

## 场景常识

修改时要判断目标类别的真实使用语境，而不是只看物品是否存在。

以 `桌面+学习` 为例，默认是欧美家庭或学生个人学习桌，不是教师讲台、办公室、会议室或实验室。以下内容应谨慎使用或替换：

- 过办公化设备：如胶圈装订机、桌面碎纸机、专业文件设备。
- 过教师化设备：如文档摄像头、课堂投影类器材。
- 过实验室化设备：如专业实验仪器。
- 过中式或课程体系绑定物：如算盘、古诗词读本、成语词典。
- 过电脑外设化设备：如多个 USB 转接、集线、扩展坞、线缆管理器。
- 过奇怪的辅助件：如书页固定架、页面压纸夹、索引分页册、书包挂放座。
- 过工艺用途的工具：如纸胶带切割器、胶带切割座；若保留，只能作为少量通用文具辅助件。

同类判断应应用到其他场景：厨房不是工业后厨，餐桌茶歇不是咖啡店后场，卧室梳妆不是专业影棚，运动场装备不是器材仓库。

## 替换方向

优先选择这些替换方向：

- 轮廓不同：球体、盒体、卷状、瓶罐、软包、立架、小设备、厚实工具等混合。
- 功能不同：阅读、记录、测量、计时、整理、保护、清洁、收纳、练习、辅助听觉等分散。
- 摆放方式不同：平放、竖放、立放、卷放、靠放、筒装、篮装。
- 可玩性明确：适合缺失、替换、移动、朝向、数量变化。

避免这些替换方式：

- 用另一个 `xxx盒` 替换一个 `xxx盒`。
- 用另一个 `xxx托盘` 替换一个 `xxx托盘`。
- 用同一基础物的材质、颜色、尺寸、封面或用途变体替换。
- 用更专业、更怪、更难识别的设备补位。
- 用通过测试但不自然的 `xxx器`、`xxx座`、`xxx架`、`xxx篮` 补位。
- 用标签、卡片、薄片、小配件、清洁刷、袋子批量补位。

## 四池修改规则

### `core_items`

- 保留能代表该类别的主体物，但同族不应撑满核心池。
- 若核心池出现大量书、本、册、盒、夹、袋，应压缩到代表项，并用不同本体族补充。
- 核心物必须是普通用户自然能想到的主物，不要引入过专业设备凑数。

### `support_items`

- 写服务核心物的辅助件，但不能把核心物换小名重写。
- 单个微小物必须升级为可见单位。
- 同类工具成组出现时，要控制族密度，例如笔组、尺套、夹罐、盒罐不能成片。

### `visible_small_items`

- 小物必须是一盒、一包、一筒、一套、一卷、一束、一叠、一排等可见单位。
- 单位要自然：`一套圆规` 比 `一包圆规` 更合理。
- 不要让同一种包装单位或同一种小物族连续堆叠。

### `scene_expansion_items`

- 只写可放置中型单物。
- 不能是组合，不能是核心物小变体，不能是大型家具或固定装置。
- 第四池最容易发生托盘、架、盒、垫、刷、筒伪重复；修改后必须按视觉本体族复审。
- 若某项虽然通过测试但场景常识弱，也必须替换。

## 修改后复审

替换完成后必须回答：

- 问题物品的根因是否真的被解决？
- 是否又产生新的同族堆叠？
- 是否引入了不符合场景常识的专业或怪异物品？
- 是否只是把旧的 `盒/袋/盘/板/架/座/垫/刷` 换成新的同类模板？
- 每个新增物品是否有独立轮廓，能被玩家当作一个清晰差异点？
- 每池是否仍为 50 条，池内唯一，第四池无跨池精确重复？

若任一答案不合格，继续替换，不应进入最终写库。
