# 盲盒物品库修改规则

本文档不是 SOP，也不是逐步操作表。
它是修改 `blind_boxes.py` 时必须服从的“任务宪法”：规定目标、边界、优先级、否决条件和验收标准。

当自动测试通过但结果仍然别扭、机械、伪多样或场景不自然时，必须继续复查并继续修改，直到通过本文件的判断标准；不得把“已经过测试”当作停止条件。

## 1. 文档定位

- 本规则适用于全部盲盒类别。
- 本规则服务“审查后修改”阶段：先有 `审查规则.md` 的问题归因和 `审查文件/{类别} 审查文档.md` 的问题记录，再按本文件完成替换和复查。
- 本规则规定“应该达成什么结果”，不规定每一步必须怎么做；只要最终结果符合本文件，就允许自主选择修改路径。

## 2. 核心任务

修改的目的不是把一批不合格名称换成另一批更会过规则的名称，而是：

- 让物品更真实。
- 让场景更自然。
- 让四池分工更清楚。
- 让多样性来自“物本身不同”，而不是“前缀不同、壳体不同、后缀不同”。
- 让每个条目都适合找不同游戏中的单点差异。

## 3. 优先级宪法

当多个标准冲突时，按下面顺序裁决，低优先级不得推翻高优先级：

1. `真实与自然`
   物品必须是现实里常见、稳定、普通用户能自然理解的东西。
2. `场景语境`
   物品必须属于当前类别的前台语境，而不是后场、维护、安装、仓储、回收、冷却、晾放或专业操作阶段。
3. `物体本体`
   物品必须首先是一个“独立成物”的对象，而不是内容物、功能词或小物的承载壳。
4. `四池角色`
   条目必须符合所在池的职责，尤其第四池必须是“场景补充中型物”，不是“辅助器具池”。
5. `可玩性`
   条目必须边界清楚、第一眼可识别、适合作为单点差异。
6. `多样性`
   多样性必须来自物体角色、轮廓、使用场景、摆放形态和本体族差异，不得来自换前缀、换内容词或换通用壳体。
7. `测试与配额`
   自动测试、数量、唯一性、尾词配额和兼容字段必须通过，但它们只是底线，不是生成方向。

一句话原则：
`能过测试，不等于应该入库；像真实场景物，才配入库。`

## 4. 四池定义

### `core_items`

- 写该类别最能代表场景与用途的主体物、主容器、主设备、主承载物。
- 不写微小配件、后场辅助物、纯功能壳体。

### `support_items`

- 写服务主体物使用的常见辅助件、工具件、替换件、取用件。
- 不写单个极小配件，不把同一基础对象机械换名重写。

### `visible_small_items`

- 写原本偏小、但已升级成清晰可见单位的条目，如一盒、一包、一筒、一盘、一套、一卷、一束、一叠、一册、一排。
- 不写单粒、单张、单片、单根、碎屑、小点、细线、标签、价格签。

### `scene_expansion_items`

- 历史字段名保留，实际语义是“场景补充中型物”。
- 它应当是普通玩家一眼能认出的、能独立出现在画面前台的自然物件。
- 它是弱主角，不是小物的抽取器、支架、托架、收纳座、保护座、分区盒或临放盘。
- 它必须先是“物”，再是“功能”；先能独立成立，再谈中型边界。

一句话定义：
`第四池写的是前台自然单物，不是为了过边界而临时造出来的壳体。`

## 5. 准入标准

任一候选条目，只有同时满足下面标准，才允许保留：

### 5.1 现实自然

- 普通用户会自然承认“这就是一个真实存在的东西”。
- 不依赖专业知识、课程体系、吧台流程、工坊流程或维护流程才看得懂。

### 5.2 场景前台

- 它会自然出现在当前类别画面的前台。
- 它不是清洁、回收、冷却、封口、晾放、维护、储运、安装或后处理阶段物。

### 5.3 独立成物

- 它本身就是物件，而不是“内容物 + 一个通用壳体”。
- 它本身就是物件，而不是“功能词 + 一个通用板/盘/垫/盒/册/卷”。
- 它本身就是物件，而不是“另一个更小物品的附属取用/承托/固定器具”。

### 5.4 第一眼可识别

- 脱离文字说明也能被当成一个明确物件。
- 不靠文字、图标、刻度、标签、小细节或非常细的局部结构成立。
- 不需要额外解释“它到底属于什么类别的东西”。
- 如果必须补一句“它其实是某种教具 / 演示器材 / 手作工具 / 课程用品”，才勉强说得通，默认不够自然。

### 5.5 池内角色清楚

- 它没有偷跑成核心物，也没有退化成微小配件。
- 尤其第四池，不得只是把前三池已有物套进另一个壳。

## 6. 强否决模式

下列模式默认不通过。除非能证明它是现实里高频、稳定、独立的常用物品名，否则必须替换。

### 6.1 不同前缀 + 相同后缀

例如：

- `糖包抽取器 / 餐巾抽取器 / 湿巾抽取盒`
- `文件托盘 / 信件托盘 / 打印纸托盘`
- `模具清洁刷 / 烤盘清洁刷 / 粉筛清洁刷`

这类问题的根因不是“名字像”，而是物体本体没有真的拉开。

### 6.2 不同内容物 + 相同壳体

例如：

- `食材分区篮 / 糖珠分区盘 / 柠檬皮分区盘`
- `样本册 / 素材册 / 标本册`
- `糖果盒 / 饼干盒 / 点心盒`

这类命名本质上仍然是“内容物换了，壳没变”。

### 6.3 修饰前缀 + 通用载体

例如：

- `隔热垫`
- `分区盒`
- `记录板`
- `整理盘`
- `展示座`
- `电路实验板`
- `植物压花板`
- `甜点切分板`
- `面包切片板`

这类问题的通用根因不是某个具体前缀不对，而是：

- 前面换成了不同说明词；
- 后面仍然只是同一种通用载体；
- 名字的“具体感”主要来自说明词，而不是来自物件本体。

如果去掉前面的修饰词，最后只剩通用载体，默认视为模板化高风险。
如果一个词的可成立性主要来自“说明它拿来做什么、服务什么、属于什么流程/课程”，而不是来自玩家自然熟悉的物件本体，也默认高风险。

这里的“修饰前缀”不是特指某几个词，而是泛指一切让名字显得更具体、但没有改变物件本体的说明部分，常见包括：

- `功能前缀`：如 `隔热`、`防滑`、`展示`、`记录`
- `动作前缀`：如 `切分`、`切片`、`分装`、`转移`、`压平`
- `流程前缀`：如 `晾放`、`冷却`、`封口`、`回收`
- `对象前缀`：如 `甜点`、`面包`、`糖包`、`茶包`
- `学科/用途前缀`：如 `实验`、`演示`、`压花`、`观察`

如果去掉这些修饰部分后，只剩 `板 / 盘 / 盒 / 篮 / 筒 / 器 / 架 / 座 / 垫` 这类通用载体，就不应因为“看起来像个完整名词”而放行。

一句话判断：
`只要真正稳定不变的是壳体，前面只是换说明词，这就是同一类伪自然结构。`

### 6.4 二级器具化

例如：

- `抽取器`
- `托架`
- `挂架`
- `收纳座`
- `保温座`
- `立筒`
- `临放盘`

第四池原则上不以这类条目为主来源。
如果一个名称的价值主要在“给另一个更小的物件服务”，它就不是优先写入的第四池物。

### 6.5 内容承载壳化

例如：

- `植物标本夹`
- `作品集画夹`
- `乐谱夹`
- `海报筒`

这类名字把真正应该关注的内容本体，塞进了承载壳里；画面重点不稳定，命名也容易机械扩写。

### 6.6 后场与维护物

例如：

- 晾放、冷却、沥水、回收、封口、清洁、擦拭、维修、废弃处理相关成片出现
- 偏咖啡吧台、后厨、工具房、专业制备台、维护台的物件成片出现

少量真实条目可存在，但不得成为该池主要来源。

### 6.7 解释成本过高的说明型命名

例如：

- `电路实验板`
- `月相演示球`
- `DNA结构摆件`
- `植物压花板`

这类词最常见的问题不是“完全不存在”，而是：

- 它们更像说明词、用途词、课程词，而不是普通玩家自然会说出的前台物名。
- 它们往往需要先解释用途、学科、流程或场景，用户才知道它是什么。
- 它们很容易借 `板 / 球 / 器 / 摆件` 之类后缀通过边界，却没有真正通过自然度。

这里的关键不是它是不是“教具”，而是它是不是一种`解释型命名`：

- 如果名称本体主要靠说明成立，而不是靠物件本体成立，就属于高风险。
- 不论这是课程器材、厨房工序物、茶歇服务物还是手作流程物，都按同一条规则处理。

除非该类别本身就是明确的专业场景，且该物在该场景中属于普通人第一眼能叫得出的前台物，否则默认不写入第四池。

## 7. 裸名复审原则

修改时必须做“裸名复审”。

做法不是机械删词，而是判断：

- 去掉对象词后，它剩下的是什么？
- 去掉功能词后，它剩下的是什么？
- 去掉内容物词后，它剩下的是什么？
- 去掉动作词后，它剩下的是什么？

如果最后主要只剩下下面这些通用壳体，默认高风险：

- `板`
- `盘`
- `垫`
- `盒`
- `包`
- `册`
- `卷`
- `袋`
- `篮`
- `筒`
- `架`
- `座`
- `托盘`
- `抽取器`
- `分区盒`
- `收纳座`
- `挂架`
- `临放盘`

关键判断不是“这个后缀能不能出现”，而是：

`这个名字是不是只靠后缀活着。`

尤其要补做一次“修饰壳复审”：

- `甜点切分板 -> 板`
- `面包切片板 -> 板`
- `食材分装盒 -> 盒`
- `面团转移板 -> 板`
- `电路实验板 -> 板`

如果去掉修饰或说明部分后，名字立刻退化成通用载体，默认按伪自然条目处理，而不是按“已经具体到某个动作 / 对象 / 用途 / 学科”放行。

## 8. 多样性标准

多样性不是“把名字改得不一样”，而是让物体真的不一样。

### 8.1 多样性的正确来源

优先来自：

- 物体角色不同
- 视觉轮廓不同
- 摆放形态不同
- 使用阶段不同
- 物品家族不同

不应主要来自：

- 前缀不同
- 内容词不同
- 同一壳体不同
- 同一后缀不同近义前缀

### 8.2 多样性复审必须同时看三条线

- `视觉本体族分布`
- `高频尾词分布`
- `模板骨架密度`

只看尾词配额不够。
即使尾词没有超额，只要仍然是一串“内容物 + 通用壳体”或“对象词 + 同一器具”，也必须继续修改。
即使尾词和模板都没有超额，只要仍然出现一批“需要解释它是什么课上器材”的条目，也必须继续修改。
即使内容看似贴合场景，只要仍然成片出现“修饰前缀 + 通用载体”，也必须继续修改，因为这通常只是把说明词包在壳体外面。

### 8.3 第四池的多样性要求

第四池不应主要由下列单一来源构成：

- 一串盒、篮、盘、筒、架、座
- 一串抽取器、托架、分区盒、收纳座
- 一串图册、相册、练习本、挂图
- 一串清洁、晾放、隔热、保温、分装壳体

第四池应尽量分散到不同真实物族，例如：

- 普通容器本体
- 服务或取用物
- 轻量设备
- 个人前台物
- 软包或布类
- 成品物
- 参考物
- 厚实工具

## 9. 测试与边界的关系

自动测试、数量和字段兼容是必须满足的底线，但它们不能反过来支配命名方向。

特别强调：

- `test_scene_expansion_items_keep_medium_scene_fragment_boundary` 只是底线测试。
- 测试允许的后缀，不是推荐造词模板。
- “能过中型边界”不等于“自然”“真实”“应该入库”。

任何条目，只要出现下面情况之一，就算测试通过也必须继续修改：

- 读起来像临时拼出来的壳体词
- 去掉前缀后只剩通用载体
- 一眼看不出它到底是什么物
- 明显更像后场流程物
- 明显只是为了通过测试才带上某个后缀

## 10. 自主复查义务

当结果有问题时，AI 不能停在“已经修改完”。

必须继续复查、继续替换，直到下面几类风险都被压下去：

- 场景自然度仍不稳
- 第四池仍像辅助器具池
- 伪多样性仍明显
- 同模板条目仍成片
- 虽然过测试，但普通人不会自然这样命名

完成一轮内容书写替换后，不得直接把“已替换”视为结束。
必须立刻重新核查当前类别、当前池的多样性，至少同时复看：

- `视觉本体族分布`
- `高频尾词分布`
- `模板骨架密度`
- `解释成本是否仍然偏高`

如果复查后仍存在下面任何一种情况，就必须再次替换，而不是停在现状：

- 单一尾词仍明显过密
- 同一模板骨架仍成片出现
- 仍然存在“不同前缀 + 相同后缀”的伪多样性
- 仍然存在“不同内容物 + 相同壳体”的伪多样性
- 虽然名字变了，但视觉本体族没有真正拉开
- 仍然存在“需要靠学科 / 课程 / 功能解释才能成立”的条目

这是一条闭环义务，不是一次性检查：
`替换 -> 复查多样性 -> 发现问题 -> 再替换 -> 再复查`

只有当多样性问题不再成立，才能进入最终通过判断。

允许自主决定替换路径，但不允许把不自然结果包装成“规则允许”。

一句话要求：
`如果你自己读着都像是在硬造词，就还没有改完。`

## 11. 验收标准

修改完成后，至少必须同时满足下面条件，才算通过：

### 11.1 基础结构

- 每池正好 50 条。
- 池内唯一。
- 与其他池无机械精确重复。
- 兼容字段和运行时映射不被破坏。

### 11.2 真实性

- 条目是真实可见的物，不是抽象名、过程名、壳体名或临时功能名。
- 普通用户第一眼能把它当成一个物件。
- 普通用户不需要额外说明“这其实是某种教具 / 演示器材 / 压花工具”才知道它是什么。

### 11.3 场景自然度

- 条目属于当前类别前台。
- 不成片滑向后场、维护、清洁、冷却、封口、安装、仓储或专业流程。

### 11.4 四池角色

- 第四池不再像抽取器池、支架池、托盘池、分区盒池。
- 第四池的条目首先是“场景物”，不是“别的物的辅助壳”。

### 11.5 多样性

- 不存在明显的“不同前缀 + 相同后缀”成片堆叠。
- 不存在明显的“不同内容物 + 相同壳体”成片堆叠。
- 多样性主要来自物体本体差异，而不是壳体差异。
- 修改后的结果重新复查时，不再触发需要继续整改的多样性风险；若复查仍能明确指出高频尾词过密、模板骨架过密或视觉本体族未拉开，则不得判通过。

### 11.6 可玩性

- 适合缺失、替换、移动、朝向、数量等清晰差异。
- 不依赖文字、标签、刻度、细小结构、发光、反光、透明或阴影。

### 11.7 自动化底线

- 内容测试通过。
- 第四池中型边界通过。
- 配额与唯一性检查通过。

## 12. 复审记录要求

修改完成后，对应类别审查文档必须能回答：

- 本次改了什么。
- 本次主要压掉了哪类伪多样性。
- 本次替换方向是什么。
- 还有哪些剩余风险。
- 为什么这些剩余项暂时保留。

审查文档不是流水账，而是“本次判断为何成立”的依据。

## 13. 类别提醒

这些提醒不是白名单，只是常见偏差校正。

- `桌面+学习`
  不要把第四池做成书册、图册、挂图、夹壳的横向平移，也不要滑向“学科教具名 + 板 / 球 / 器 / 摆件”的伪自然写法。
- `餐桌+茶歇`
  不要把第四池做成抽取器、托座、杯托、纸品壳体和后场清洁物，也不要滑向 `甜点切分板`、`点心分隔盘` 这类“服务动作 + 载体”的伪自然写法。
- `厨房+烘焙`
  不要把第四池做成压泵器、挤压器、晾放架、清扫刷、分装壳体和后场加工物，也不要滑向 `面包切片板`、`面团转移板`、`甜点切分板` 这类“工序动作 + 板/盘/盒”的伪具体命名。
- `卧室+梳妆 / 浴室+洗护`
  不要把第四池做成连续的盒、筒、架、盘、托盘、分格壳、瓶罐壳或纯收纳壳。
- `商店+零食 / 节日+礼物`
  不要把第四池做成礼盒、零食盒、外带袋、分配器或包装壳体横向换词。
- `手作 / 园艺 / 运动 / 出行`
  不要把第四池做成夹、架、盒、筒、袋、护套和说明性壳体的机械扩写。

## 14. 最终判语

合格修改不是“看起来像做了很多替换”，而是：

- 用户一看就觉得更自然；
- 场景一看就更像真实前台；
- 条目一看就不是硬造词；
- 多样性一看就不是换壳体。

如果结果仍然让人产生“这名字像 AI 为了过规则临时拼出来的”感觉，就视为未通过，必须继续修改。
