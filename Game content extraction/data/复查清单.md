# 盲盒物品库复查清单

本清单是“内容修改完成一轮后”的通用复查入口，不替代 `审查规则.md` 或 `修改规则.md`。
它只负责帮助你快速确认：当前结果能不能收尾，还是必须继续改。

## 1. 使用时机

- 完成一轮条目替换后，立即读取本清单。
- 先运行必要测试，再按本清单做人审复查。
- 复查结论回写到同一份 `审查文件/{类别} 审查文档.md`，不要另起临时总结文件。

## 2. 快速通过线

如果下面 4 条里任一条答“否”，默认还不能收尾：

- 这些条目读起来像真实前台物，而不是为了过规则临时造词。
- 第四池整体看起来像“场景补充中型物”，而不是支架 / 托盘 / 收纳壳堆叠池。
- 多样性主要来自物体本体差异，而不是换前缀、换内容词、换后缀。
- 普通玩家第一眼能认出它是什么，并适合做找不同单点差异。

## 3. 必查项

### 3.1 结构底线

- 四池都为 50 条，且池内唯一。
- 兼容字段、运行时映射、字段名没有被破坏。
- 没有把第四池重新并回兼容空桶。
- 没有明显跨池机械精确重复。
- 没有引入 `组合`、禁词或明显命中 blocked patterns 的条目。

### 3.2 场景与本体

- 条目属于当前类别前台，而不是后场、维护、清洁、冷却、封口、仓储或专业流程。
- 条目先是“独立成物”，不是“内容物 + 壳体”或“功能词 + 通用载体”。
- 去掉修饰词后，不会大面积退化成 `板 / 垫 / 盘 / 盒 / 册 / 架 / 座 / 托盘`。
- 不存在大量“需要解释是什么教具 / 演示器材 / 课程用品”才能成立的命名。

### 3.3 第四池专项

- 第四池没有明显成片出现 `支架 / 底座 / 托架 / 展示架 / 挂架 / 分区盒 / 临放盘 / 收纳座`。
- 第四池没有明显成片出现 `说明词 + 板 / 垫 / 盘 / 盒 / 册`。
- 第四池不是核心物近义变体、材质变体、尺寸变体的回收池。
- 第四池里没有“一眼读成同后缀家族”的长串，如 `xx布包 / xx工具包 / xx防尘袋 / xx提篮 / xx收纳篮 / xx卷筒`。

### 3.4 多样性

- 没有明显的“不同前缀 + 相同后缀”成片堆叠。
- 没有明显的“不同内容物 + 相同壳体”成片堆叠。
- 高频尾词、模板骨架、视觉本体族分布没有继续向单一模板收缩。
- 即使测试通过，也没有“名字变了，但看起来还是同一种壳体矩阵”的情况。

### 3.5 可玩性与国际化

- 条目轮廓清楚，适合做缺失、替换、移动、朝向或数量变化。
- 不依赖文字、标签、价格签、细线、小点、发光、反光、透明或阴影变化。
- 默认符合国际化常识，没有强中式、强地域、强课程体系绑定物品。

## 4. 一票否决信号

只要出现下面任一情况，就不要判“已完成”：

- 你自己读着都像在硬造词。
- 去掉修饰后，稳定不变的主要还是同一批通用壳体。
- 第四池虽然过了中型边界，但整体仍像器具池或壳体池。
- 用户一眼会把它读成“很多个同后缀词”，而不是很多个不同物族。

## 5. 回写要求

复查后，把结果写回对应类别审查文档的这三段：

- `本次修改记录`
- `修改后复审`
- `剩余风险与暂留理由`

如果复查中发现问题，默认进入下一轮：

`替换 -> 复查 -> 发现问题 -> 再替换 -> 再复查`
