# 门口+雨具 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `门口+雨具` 的四池内容。
- 本轮结论：当前版本的第一池仍能读出“门口雨具区”的主体场景，但已经混入较多置物盘、收纳篮、购物袋、文件袋和季节性出门配件；第二池被 `套 / 袋 / 盒 / 包 / 板 / 垫 / 筒` 的补充壳和收纳壳拉散；第四池明显没有通过，已经被写成挂板、整理筒、记录本和清洁盆的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的雨伞、雨衣、雨靴、门垫和鞋具主场景还在，但“出门杂物区”和“临放收纳区”的读感已经开始抬头。
- `support_items` 的主要问题不是和门口场景无关，而是把真实辅助件、补充件、保护壳、收纳壳和整理壳混在了一池里。
- `visible_small_items` 格式合规，但小喷雾、除湿耗材、束带、封口带和补充瓶叠加后，已经很像“门口补给包”而不是纯粹的小可见单位池。
- `scene_expansion_items` 是当前最大问题：挂板、挂架、记录本、整理板、篮筒盘桶和清洁盆集中回流，明显偏离“前台自然中型物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池最大问题不是整体失场，而是 `桶 / 筒 / 盘 / 盒 / 袋 / 篮` 壳层和“出门杂配件”已经开始挤占主体位。
- 低优先级平衡项：`立式伞桶`、`陶瓷伞筒`、`雨水托盘`、`鞋托盘`、`鞋油罐`、`鞋蜡罐`、`擦鞋海绵盒`、`除湿桶`、`除湿盒`、`清洁湿巾盒`、`钥匙托盘`、`出门置物盘`、`便携暖手袋`、`小镜子`、`帆布手提袋`、`折叠购物袋`、`防水文件袋`、`便携雨具包`、`包袋临放篮`、`针织围巾`、`羊毛帽`、`耳罩`、`擦手毛巾`。
- 这些条目不一定要立刻整片清洗，但它们说明本池已经开始用“临放壳、补给壳和出门配套物”维持密度；后续若要腾位，可优先从这组压缩。
- 当前可作为锚点保留的方向：`长柄雨伞`、`折叠雨伞`、`高尔夫雨伞`、`儿童雨伞`、`成人雨衣`、`儿童雨衣`、`便携雨披`、`防水斗篷`、`雨帽`、`高筒雨靴`、`短筒雨靴`、`防水鞋套`、`防滑门垫`、`吸水地垫`、`鞋刷`、`长柄鞋拔`、`防水喷雾瓶`、`烘鞋器`、`暖鞋器`、`鞋底刮泥板`、`背包防雨罩`、`防水手套`。

### support_items

- 本池被补充壳、收纳壳和保护壳拉得过散，优先替换：`折叠伞套`、`雨衣折叠袋`、`雨披收纳袋`、`雨水盘吸水垫`、`伞筒吸水垫`、`钥匙收纳盒`、`零钱收纳盒`、`信件收纳盒`、`小镜托盘`、`包带整理带`、`购物袋束带`、`防水袋束带`、`背包防雨套袋`、`衣帽刷套`、`便携湿巾包`、`随身纸巾包`、`包袋防尘袋`、`围巾整理板`、`手套晾放垫`、`门口清洁盆`、`鞋套整理盒`、`伞套沥水袋`、`鞋拔挂筒`、`玄关整理盘`、`购物袋收纳筒`、`背包挂带`、`擦鞋海绵补充盒`、`鞋油补充瓶`、`防水手套包`。
- 本池里也存在“本体退化成替换件”的风险，优先复查：`鞋刷头`、`湿巾补充包`、`纸巾补充包`。
- 问题根因不是这些词完全不能出现，而是这一池已经同时承担了“真实辅助工具池”“补给补充件池”和“玄关收纳整理池”三种职责。
- 当前更稳的辅助物方向仍应放在真正服务雨伞、鞋具、门垫和雨天出门动作的工具本体，如 `雨伞束带`、`伞面擦拭布`、`伞尖保护套`、`伞柄防滑套`、`雨靴撑片`、`鞋撑`、`鞋垫`、`鞋油刷`、`擦鞋手套`、`擦鞋布卷`、`鞋面护理刷`、`门垫清洁刷`、`地垫刮水器`、`除湿袋`、`除味炭包`、`帽檐清洁刷`、`雨靴清洁刷`、`围巾软刷`。

### visible_small_items

- 整体格式合规，暂不建议整池推倒。
- 低优先级压缩项：`一盒除湿片`、`一盒防潮珠`、`一盒干燥片`、`一盒暖贴包`、`一包鞋底防滑贴`、`一盒鞋带`、`一盒防水喷雾小瓶`、`一排除味喷雾小瓶`、`一盒鞋油补充瓶`、`一排香氛喷雾小瓶`、`一叠除湿纸包`、`一盒钥匙扣`、`一筒购物袋束带`、`一筒包带整理带`、`一盒门垫吸水片`、`一盒伞筒吸水垫`、`一卷防水封口带`、`一盒鞋蜡块`、`一包帽子防尘套`。
- 这些条目本身未必违规，但它们与第二池、第四池的补充件和整理壳叠加后，会继续放大“门口补给与收纳耗材区”的读感。

### scene_expansion_items

- 挂板、挂架和提示板家族回流，必须优先替换：`钥匙挂板`、`信件挂板`、`围巾挂板`、`手套挂架`、`帽子挂架`、`天气提示板`、`出门清单板`、`雨具整理板`、`帽檐收纳板`。
- 记录本和提醒本家族回流，必须优先替换：`雨天记录本`、`玄关记录本`、`出门提醒本`、`访客留言本`、`快递记录本`、`购物清单本`。
- 篮、筒、盘、托盘、座和桶家族过密，必须优先替换：`拖鞋篮`、`雨靴篮`、`购物袋筒`、`鞋刷筒`、`鞋拔筒`、`擦鞋盘`、`湿鞋托盘`、`出门杂物篮`、`包袋篮`、`背包篮`、`防水手套筒`、`伞套沥水座`、`擦手毛巾筒`、`湿鞋隔离盘`、`雨靴刷筒`、`鞋面护理盘`、`门口香氛盘`、`湿伞隔离桶`。
- 后场清洁和临时处理物偏多，必须整体压缩：`雨靴清洁盆`、`雨伞清洁盆`、`雨天工具盒`、`鞋底清洁器`。
- 泛用家居盒体与类别绑定偏弱，建议列为中高优先级替换：`抽纸盒`、`湿巾盒`、`纸巾木盒`。
- 当前较接近第四池方向、可作为保留候选的条目相对有限：`除湿机`、`暖风机`、`暖鞋机`、`空气净化机`、`香薰机`、`门口小钟`、`沙漏计时器`、`翻页台历`、`桌面月历`。

## 4. 多样性与模板密度

- 第一池当前最明显的压力不是“雨具太多”，而是 `盘 / 桶 / 筒 / 盒 / 袋 / 篮` 与围巾、帽子、购物袋这类出门杂配件一起抬高了场景杂音。
- 第二池最拥挤的尾词是 `套 / 袋 / 盒 / 包 / 板 / 垫 / 筒`，主要问题是“不同被服务对象 + 相同保护壳/收纳壳”的伪多样性。
- 第三池格式没错，但 `小喷雾 / 除湿耗材 / 束带 / 补充瓶 / 小整理件` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `挂板 / 挂架 / 提示板 / 整理板`、`记录本 / 留言本 / 清单本`、`篮 / 筒 / 盘 / 托盘 / 座 / 桶`、`清洁盆 / 清洁器 / 工具盒 / 隔离盘` 这几类玄关服务壳体链。

## 5. 第四池专项

- 第四池当前的根因不是个别坏词，而是整池写法站在“玄关收纳与提醒系统”的视角，而不是“门口前台场景补充物”的视角。
- 结果是四类问题同时叠加：挂板挂架成串、记录本成串、篮筒盘桶成串、清洁盆和临时处理器具混入。
- 本类第四池应优先补“普通玩家一眼就能认出的门垫区、伞架区、换鞋区、雨天出门角”的自然中型单物，而不是继续补挂板、提醒本、整理筒、临时隔离盘和清洁盆。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留雨伞、雨衣、雨靴、门垫、鞋具和烘干保暖装置这些稳定主体，必要时压缩临放盘、临放篮、购物袋、文件袋和季节性出门配件的横向平移。
- 第二池：从“补充件与收纳壳混池”拉回“真正辅助工具池”，优先保留擦拭、防滑、除湿、鞋具护理和雨具护理动作里会直接上手的物件本体。
- 第三池：继续保持 `一盒 / 一包 / 一排 / 一叠 / 一筒 / 一卷` 的可见单位格式，但避免除湿耗材、小喷雾、束带和整理件继续膨胀。
- 第四池：按“门口前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用挂板、挂架、记录本、整理筒、整理篮、清洁盆和隔离盘作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `板 / 架 / 本 / 篮 / 筒 / 盘 / 盒 / 袋 / 桶` 是否还成片出现。

## 7. 本次修改记录

- 本轮实际修改了 `blind_boxes.py` 中 `门口+雨具` 的 `support_items` 和 `scene_expansion_items`，`core_items` 与 `visible_small_items` 暂不大动。
- 第二池的修改方向是把“补充件、收纳壳、整理筒、整理盘、补充盒混池”拉回“真实雨具护理、鞋具护理、门垫清洁和雨天出门辅助工具池”。
- 具体清掉了 `折叠伞套 / 雨衣折叠袋 / 雨披收纳袋 / 吸水垫 / 补充包 / 收纳盒 / 托盘 / 整理带 / 束带 / 套袋 / 防尘袋 / 整理板 / 清洁盆 / 整理盒 / 挂筒 / 整理盘 / 收纳筒 / 补充盒 / 补充瓶 / 手套包` 这类弱本体壳层。
- 回填时改用 `伞骨清洁刷 / 伞柄清洁刷 / 伞布补漏贴 / 伞骨固定夹 / 靴筒撑板 / 鞋边清洁刷 / 鞋底清洁刷 / 刮泥铲 / 门缝挡水条 / 拉链润滑棒 / 门垫拍打器 / 鞋带穿引器` 等更直接服务雨天出门动作的工具本体。
- 第四池本轮按高风险家族整段清理，移除了 `挂板 / 挂架 / 提示板 / 清单板 / 记录本 / 留言本 / 篮 / 筒 / 盘 / 托盘 / 清洁盆 / 隔离桶 / 工具盒` 这类玄关整理系统写法。

- 后续复看时又发现 `xx卷 / xx画册 / xx图册` 仍然偏密，因此继续按“每种后缀最多保留两个”的口径微调，把 `条纹地毯卷 / 防水地毯卷 / 编织地毯卷 / 旅行画册 / 城市画册 / 家居画册 / 摄影画册 / 服饰图册 / 自然图册` 替换为 `菱纹毯面 / 几何席面 / 雨滴毯面 / 雨具组合 / 鞋靴组合 / 围巾组合 / 手套组合 / 门厅书立 / 防滑席面`，把第四池从“卷物和读物串联”再拉回一点到门口前台毯面物、席面物和出门组合物的混合结构。
- 本轮又继续压缩了 `毯面 / 席面` 家族，把 `菱纹毯面 / 欢迎毯面 / 脚印毯面 / 条纹席面 / 防滑席面` 改写为 `雨伞阵列 / 鞋靴阵列 / 换鞋分区 / 快递分区 / 出门分区`，避免第四池再次被同后缀软装词接管读感。

## 8. 修改后复审

- 当前四池数量均为 50，且池内唯一。
- 第四池与其余三池无重叠。
- 第四池本轮本地复查未命中禁词，且全部通过 `_is_medium_scene_fragment_item` 的中型场景边界校验。
- 这说明本轮已经先把“结构性错误”修掉了，尤其是第四池不再被 `挂板 / 挂架 / 记录本 / 篮 / 筒 / 盘 / 清洁盆 / 隔离桶` 这类玄关服务壳主导。
- 从读感上看，第二池已经明显回到雨具和鞋具护理工具池；第四池也从“玄关整理与提醒系统”转回“门口前台场景补充物”。
- 进一步按后缀统计复查后，第四池里的 `卷 / 画册 / 图册` 已分别压到 2 条，满足“每种后缀最多保留两个”的补充要求。
- 本轮继续复查后，第四池里的 `毯面 / 席面` 也已分别压到 `2 / 2`，当前仅保留 `雨滴毯面 / 刮泥毯面` 和 `几何席面 / 编织席面`。
- 本节结论仅针对 `门口+雨具` 的定点复查，不代表全仓所有类别已全量通过。

## 9. 剩余风险与暂留理由

- 第一池里仍保留了少量临放壳和出门配件，例如 `立式伞桶`、`陶瓷伞筒`、`雨水托盘`、`鞋托盘`、`除湿桶`、`除湿盒`、`钥匙托盘`、`出门置物盘`、`帆布手提袋`、`折叠购物袋`、`防水文件袋`、`便携雨具包`、`包袋临放篮`；这轮暂不动，是为了先稳住主体池，不把问题从第四池转移到第一池。
- 第三池仍保留了一些除湿耗材、小喷雾和整理配件单位，例如 `一盒除湿片`、`一盒防潮珠`、`一盒干燥片`、`一盒防水喷雾小瓶`、`一排除味喷雾小瓶`、`一筒购物袋束带`、`一筒包带整理带`；它们当前格式合规，但后续如果继续做第三池精修，可以再轻压一次。
- 第四池虽然已经脱离“整理壳链”，但现在仍有 `钟 / 历 / 机 / 卷 / 面 / 盆 / 册 / 图` 这些自然家族；后续若继续微调，重点应放在压模板密度，而不是重新引入 `板 / 架 / 本 / 篮 / 筒 / 盘 / 盒 / 桶`。
- 这轮已经把 `卷 / 画册 / 图册 / 毯面 / 席面` 压到较合理密度；后续若继续微调，优先关注 `钟 / 历 / 机 / 花盆` 这类相邻家族，而不是重新把卷物、读物或软垫面词补回来。
- 当前版本已经可以作为后续同类修改的合格基线，下一步若继续优化，建议优先做“第一池临放壳轻压”和“第三池除湿喷雾配件轻压”，而不是再大改第二池和第四池。
