# 商店+零食 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `商店+零食` 的四池内容。
- 本轮结论：当前版本的第一池仍能读出“商店零食货架区”的主体场景，但已经高度依赖袋、盒、桶、罐、杯、筒这类包装壳维持密度；第二池被 `盒 / 包 / 袋 / 杯 / 套 / 碟 / 卷` 的试吃、封口和收银服务壳拉散；第四池明显没有通过，已经被写成收银、打包和试吃服务系统的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的零食、饮料和商店货架主场景还在，但“不同零食内容物 + 相同包装壳”的伪多样性已经非常明显。
- `support_items` 的主要问题不是与零食店无关，而是把真实取物件、试吃件、包装件、收银件和一次性服务件混在了一池里。
- `visible_small_items` 格式合规，但迷你零食、饮料小罐和一次性小件成串后，已经很像“试吃补给包”。
- `scene_expansion_items` 是当前最大问题：收银设备、打包容器、托盘、纸袋、笔筒和包裹类条目集中回流，明显偏离“前台自然中型物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池最大问题不是整体失场，而是 `袋 / 盒 / 桶 / 罐 / 杯 / 筒 / 瓶` 壳层过密，已经明显形成“换内容物、不换包装本体”的伪多样性。

- 这些条目不一定要立刻整片清洗，但它们说明本池已经开始主要依赖“包装壳差异”而不是“零食本体差异”维持密度；后续若要腾位，需优先按包装家族压缩。
- 当前可作为锚点保留的方向相对有限，更多应作为“零食店货架真实存在”保留，而不是继续横向扩写同一包装骨架。

### support_items

- 本池被试吃壳、打包壳和收银服务壳拉得过散，优先替换：`试吃纸杯`、`试吃纸碟`、`纸餐巾包`、`湿巾包`、`封口夹盒`、`牛皮纸袋`、`手提纸袋`、`分食纸袋`、`瓶盖收集杯`、`纸杯盖`、`纸吸管包`、`杯托`、`饮料提手`、`冷饮杯盖`、`冷饮杯套`、`罐饮保温套`、`饼干隔纸`、`防油衬纸包`、`分食碟`、`试吃托碟`、`手套盒`、`一次性围裙包`、`纸巾补充包`、`杯口封膜`、`木勺包`、`硬币夹`、`纸杯套`、`密封贴纸卷`、`顾客纸巾盒`、`零食量杯`、`封箱胶带卷`。
- 本池里也存在“同流程不同对象 + 相同壳体”的伪多样性，优先复查：`价签夹`、`陈列挂钩`、`收银笔`、`桌面小铃铛`、`桌面记号笔`。
- 问题根因不是这些词完全不能出现，而是这一池已经同时承担了“真实取物工具池”“试吃服务池”和“收银打包辅助池”三种职责。
- 当前更稳的辅助物方向仍应放在真正参与夹取、分食、开瓶、开盖和封口动作的工具本体，如 `糖果夹`、`长柄食品钳`、`爆米花铲`、`坚果勺`、`分食勺`、`试吃小叉`、`饮料开瓶器`、`易拉罐开盖器`、`试吃牙签`、`货架挡片`、`保鲜封条`、`零食封口器`、`手持封口夹`、`开袋剪`。

### scene_expansion_items

- 收银设备和台面服务设备回流，必须优先替换：`点钞机`、`验钞器`、`扫码器`、`刷卡器`、`收银计算器`、`对讲机`、`便携收音机`、`台面小钟`、`翻页台历`、`音乐盒`。
- 打包容器和保温冷藏壳过密，必须优先替换：`抽纸盒`、`湿巾盒`、`曲奇铁盒`、`糖果铁盒`、`果汁冷饮桶`、`咖啡保温桶`、`外带餐盒`、`购物布袋`、`帆布提袋`、`保冷袋`、`手提保温袋`、`礼品纸袋`、`便携冷藏包`、`迷你冰桶`、`饮料冰桶`。

- 当前较接近第四池方向、可作为保留候选的条目相对有限：`购物篮`、`手提篮`、`爆米花机`、`口香糖机`、`咖啡机`、`热饮机`、`奶昔机`、`冰沙机`、`冰淇淋机`、`热狗机`。

## 4. 多样性与模板密度

- 第一池当前最明显的压力不是“零食主体物太多”，而是 `袋 / 盒 / 桶 / 罐 / 杯 / 筒 / 瓶` 与内容物替换一起抬高了包装壳噪音。
- 第二池最拥挤的尾词是 `盒 / 包 / 袋 / 杯 / 套 / 碟 / 卷`，主要问题是“不同服务步骤 + 相同试吃/打包壳”的伪多样性。
- 第三池格式没错，但 `迷你零食 / 小罐饮料 / 试吃小件 / 一次性消耗品` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `收银设备`、`袋 / 桶 / 盒 / 保温包 / 冷藏包`、`托盘 / 夹 / 笔筒 / 勺筒`、`包裹 / 花盆 / 花篮 / 点唱机` 这几类店务服务壳体链。

## 5. 第四池专项

- 第四池当前的根因不是个别坏词，而是整池写法站在“收银、打包和试吃服务系统”的视角，而不是“零食店前台场景补充物”的视角。
- 结果是四类问题同时叠加：收银设备成串、打包和保温冷藏壳成串、托盘和筒类成串、赠品包裹和无关装饰物混入。
- 本类第四池应优先补“普通玩家一眼就能认出的零食货架区、收银前台、饮料机旁或店内陈列角”里的自然中型单物，而不是继续补托盘、纸袋、收据夹、笔筒和包裹。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留典型零食、饮料和少量稳定零售主物，必要时压缩同包装壳家族的横向平移，避免继续靠“换内容物”扩写。
- 第二池：从“试吃壳与收银打包壳混池”拉回“真正辅助工具池”，优先保留夹取、分食、开瓶、开盖和封口动作里会直接上手的物件本体。
- 第三池：继续保持 `一盒 / 一包 / 一筒 / 一排 / 一盘 / 一袋` 的可见单位格式，但避免迷你零食、饮料小罐和一次性服务件继续膨胀。
- 第四池：按“零食店前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用收银设备、打包壳、托盘、笔筒、勺筒和包裹作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `袋 / 盒 / 桶 / 罐 / 杯 / 筒 / 瓶 / 包` 是否还成片出现。

## 7. 本次修改记录

- 本轮实际修改了 `blind_boxes.py` 中 `商店+零食` 的 `support_items` 和 `scene_expansion_items`，`core_items` 与 `visible_small_items` 暂不大动。
- 第二池的修改方向是把“试吃壳、打包壳和收银服务壳混池”拉回“真实零食店辅助工具池”。
- 具体清掉了 `试吃纸杯 / 试吃纸碟 / 纸餐巾包 / 湿巾包 / 牛皮纸袋 / 手提纸袋 / 分食纸袋 / 瓶盖收集杯 / 纸杯盖 / 纸吸管包 / 冷饮杯套 / 防油衬纸包 / 分食碟 / 手套盒 / 一次性围裙包 / 纸巾补充包 / 木勺包 / 纸杯套 / 密封贴纸卷 / 顾客纸巾盒 / 封箱胶带卷` 这类一次性服务壳和打包壳写法。
- 回填时改用 `零食夹 / 饼干夹 / 热狗夹 / 冰铲 / 糖铲 / 奶酪勺 / 调味撒粉器 / 饼干铲 / 曲奇夹 / 甜品铲 / 打价器 / 贴标器 / 补货理货棒 / 取物镊子 / 杯口压圈器` 等更直接参与夹取、分食、开瓶、封口和标价动作的工具本体。
- 第四池本轮按高风险家族整段清理，移除了 `点钞机 / 验钞器 / 扫码器 / 刷卡器 / 收银计算器 / 纸袋 / 保冷袋 / 冷藏包 / 托盘 / 笔筒 / 勺筒 / 包裹 / 花篮 / 点唱机` 这类收银、打包和试吃服务系统写法，并额外清掉了 `地毯卷 / 花盆 / 图册 / 画册` 这几组最容易模板化、也最容易把零食店前台写回旧词链的后缀家族。
- 第四池回填改成“零食机台 + 时间物 + 少量环境设备 + 地面物 + 零食阵列 + 陈列分区与少量展示面”这一路线，尽量让它更像零食店前台、饮料机旁或陈列角里自然可见的中型场景补充物。
- 本轮明确不再沿用 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 画册图册` 这类容易跨类别复用的旧生成习惯，改为直接补 `糖果阵列 / 饼干阵列 / 汽水阵列 / 冷饮展示面 / 收银前台分区 / 饮料机旁分区 / 糖果机旁分区` 这类只属于零食店语境的中型物。
- 本轮继续清理全局 `1 次组` 时，又把 `零食展示面 / 冷饮展示面` 收成 `点心套组 / 冷饮杯套组`，让它们更像前台可见商品组合，而不是陈列说明面。

## 8. 修改后复审

- 当前四池数量均为 50，且池内唯一。
- 第四池与其余三池无重叠。
- 第四池本轮本地复查未命中禁词，且全部通过 `_is_medium_scene_fragment_item` 的中型场景边界校验。
- 第四池中的 `地毯卷 / 毯面 / 席面 / 花盆 / 图册 / 画册` 当前实值分别为 `2 / 2 / 2 / 0 / 0 / 0`，已经不再出现同后缀连续堆叠压读感的问题。
- 这说明本轮已经先把“结构性错误”修掉了，尤其是第四池不再被 `袋 / 桶 / 盒 / 包 / 托盘 / 笔筒 / 勺筒 / 包裹` 或 `地毯卷 / 花盆 / 图册 / 画册` 这类店务服务和旧模板壳连片主导。
- 从读感上看，第二池已经明显回到零食店辅助工具池；第四池也从“收银、打包和试吃服务系统”转回“零食店前台场景补充物”。
- 本节结论仅针对 `商店+零食` 的定点复查，不代表全仓所有类别已全量通过。

## 9. 剩余风险与暂留理由

- 第一池里仍保留了大量 `袋 / 盒 / 桶 / 罐 / 杯 / 筒 / 瓶` 家族，例如 `薯片袋`、`玉米片袋`、`椒盐脆饼袋`、`奶酪脆球桶`、`爆米花桶`、`土豆棒筒`、`咸味饼干盒`、`黄油曲奇罐`、`水果软糖盒`、`果冻杯盒`、`混合果仁杯`、`香蕉片筒`、`燕麦棒筒`、`果汁瓶`、`汽水罐`；这轮暂不动，是为了先稳住零食货架主体池，不把问题从第二池和第四池转移到第一池。
- 第三池仍保留了较多迷你零食、饮料小罐和包装单位，例如 `一盒迷你巧克力豆`、`一包迷你薯片`、`一包焦糖爆米花`、`一筒威化卷`、`一排汽水罐`、`一排苏打水罐`、`一盒谷物棒`、`一盘果仁小包`、`一袋苹果干片`；它们当前格式合规，但后续如果继续做第三池精修，可以再轻压一次。
- 第四池虽然已经脱离“收银和打包壳链”，也清掉了 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 图册画册` 这条跨类别复用链，但当前仍有 `机 / 钟 / 历 / 地毯卷 / 毯面 / 席面 / 阵列 / 分区 / 展示面 / 组合` 这些自然家族；后续若继续微调，重点应放在继续压模板密度，而不是重新引入 `袋 / 桶 / 盒 / 包 / 托盘 / 笔筒 / 包裹`。
- 当前版本已经可以作为后续同类修改的合格基线，下一步若继续优化，建议优先做“第一池包装壳轻压”和“第三池迷你零食单位轻压”，而不是再把第二池和第四池拉回服务壳补位。
