# 手作+缝纫 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `手作+缝纫` 的四池内容。
- 本轮结论：当前版本的第一池仍能读出“手作缝纫工位”的主体场景，但已经混入较多纸样册、布料包、半成品和辅料卷；第二池被 `盒 / 筒 / 套 / 罩 / 包 / 册` 的配件壳和收纳壳拉散；第四池明显没有通过，已经被写成收纳、资料管理和清洁维护区的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的缝纫机、裁剪工具、测量工具和布艺主体场景还在，但“材料目录”和“半成品目录”的读感已经开始抬头。
- `support_items` 的主要问题不是与缝纫无关，而是把真实辅助工具、机件配件壳、收纳盒筒和标记辅材混在了一池里。
- `visible_small_items` 格式合规，但针线辅料、扣件、拉链件和小配件成串后，已经很像“缝纫辅料目录”。
- `scene_expansion_items` 是当前最大问题：提篮、工具包、布包、收纳袋、防尘袋、图册、记录本和卷筒集中回流，明显偏离“前台自然中型物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池最大问题不是整体失场，而是 `册 / 包 / 半成品 / 布料 / 卷 / 盒` 家族已经开始挤占主体位。
- 低优先级平衡项：`纸样册`、`拼布图样册`、`服装纸样包`、`绣花图案册`、`棉布匹`、`亚麻布匹`、`牛仔布料`、`毛毡布料`、`帆布布料`、`衬布料`、`热熔衬布`、`刺绣布料`、`拼布布片包`、`绗缝棉絮包`、`贴布绣布片`、`围裙半成品`、`托特包半成品`、`靠垫套半成品`、`布偶半成品`、`拼布被面`、`绣花布面`、`贴布绣布面`、`绣线卡册`、`木制针线盒`、`铁皮针线盒`、`缝纫线筒`、`绣线束`、`包边带卷`、`斜纹带卷`、`花边带卷`、`拉链卷`、`纽扣铁盒`。
- 这些条目不一定要立刻整片清洗，但它们说明本池已经开始用“材料壳、半成品壳和辅料壳”维持密度；后续若要腾位，可优先从这组压缩。
- 当前可作为锚点保留的方向：`桌面缝纫机`、`便携缝纫机`、`手摇缝纫机`、`家用锁边机`、`桌面绣花机`、`小型绗缝机`、`裁缝剪`、`布料剪`、`滚轮裁刀`、`裁剪垫`、`裁缝软尺`、`拼布方尺`、`曲线制图尺`、`纸样描线轮`、`拼布工作台`、`裁缝人台`、`熨衣板`、`刺绣绷架`。

### support_items

- 本池被配件壳、收纳壳和机件盒族拉得过散，优先替换：`手缝针册`、`机针替换包`、`绣花针套筒`、`珠针收纳筒`、`安全别针盒`、`滚刀替换片筒`、`描图纸卷`、`划粉轮盘`、`划粉笔筒`、`布用铅笔筒`、`梭芯盒`、`梭壳盒`、`压脚盒`、`拉链压脚盒`、`包边压脚盒`、`扣眼压脚盒`、`滚边压脚盒`、`机针盒`、`送布牙盒`、`缝纫机防尘罩`。
- 本池里也存在“同对象不同补位 + 相同壳体”的伪多样性，优先复查：`护指套`、`剪刀护套`、`纽扣定位尺`、`包边器`、`斜纹带成型器`、`脚踏控制器`。
- 问题根因不是这些词完全不能出现，而是这一池已经同时承担了“真实辅助工具池”“机件配件池”和“收纳保护池”三种职责。
- 当前更稳的辅助物方向仍应放在真正参与穿针、拆线、打孔、测量、整形、压缝和扣件处理动作的工具本体，如 `穿针器`、`自动穿针器`、`拆线器`、`拆线刀`、`打孔器`、`锥子`、`顶针`、`剪刀磨刀器`、`裁尺`、`水消笔`、`热消笔`、`缝份尺`、`缝边尺`、`熨烫尺`、`熨烫布`、`袖口整形器`、`裁缝火腿`、`压缝木块`、`翻带器`、`扣眼器`、`铆扣压器`、`四合扣压器`、`穿孔器`。

### visible_small_items

- 整体格式合规，暂不建议整池推倒。
- 低优先级压缩项：`一排梭芯`、`一盒线轴帽`、`一包压脚螺丝`、`一盒裁刀刀片`、`一包布夹`、`一盒缝纫夹`、`一盘纸样压块`、`一册布样册`、`一排短拉链`、`一包拉链头`、`一盒纽扣`、`一包按扣`、`一包四合扣`、`一包铆扣`、`一包鸡眼扣`、`一包钩扣`、`一包裤钩`、`一包珠片`、`一筒串珠`、`一束绣线`、`一排彩色线轴`、`一套缝纫线团`、`一盒缝线蜡`、`一卷熨烫胶带`。
- 这些条目本身未必违规，但它们和前两池、第四池的辅料壳与资料壳叠加后，会继续放大“缝纫辅料目录”的读感。

### scene_expansion_items

- 清洁维护物和日常杂电器回流，必须优先替换：`粘毛滚筒`、`硬毛除尘刷`、`手持吸尘器`、`电动起球器`、`手持除毛器`、`蒸汽挂烫机`、`便携收音机`、`防水计时钟`、`手持计时器`、`翻页月历`、`缝纫录音笔`。
- 提篮、工具包、布包和收纳袋家族过密，必须优先替换：`针线篮`、`布艺收纳篮`、`布料提篮`、`花边提篮`、`棉布提篮`、`裁片提篮`、`缝纫工具包`、`裁缝工具包`、`刺绣工具包`、`拼布工具包`、`剪裁工具包`、`熨烫工具包`、`针线包`、`拉链包`、`成品布袋`、`围裙布包`、`托特布包`、`抱枕布包`、`布偶布包`、`裁片收纳袋`、`布样收纳袋`、`样衣防尘袋`、`成品防尘袋`。
- 图册、册类和记录本家族回流，必须优先替换：`纸样文件夹`、`布样册`、`配色图册`、`花样图册`、`拼布图册`、`刺绣图册`、`布艺灵感册`、`量体记录本`、`缝纫记录本`、`版型记录本`。
- 卷筒和辅料卷家族过密，必须优先替换：`布料卷筒`、`松紧带卷`、`蕾丝带卷`、`棉线卷`。
- 当前较接近第四池方向、可作为保留候选的条目相对有限：`桌面废线桶`、`布屑收集桶`、`手持吸尘器`、`电动起球器`、`蒸汽挂烫机`。

## 4. 多样性与模板密度

- 第一池当前最明显的压力不是“缝纫主体物太多”，而是 `册 / 包 / 半成品 / 布料 / 卷 / 盒` 与材料目录家族一起抬高了场景杂音。
- 第二池最拥挤的尾词是 `盒 / 筒 / 套 / 罩 / 包 / 册`，主要问题是“不同机件或辅料 + 相同收纳壳/保护壳”的伪多样性。
- 第三池格式没错，但 `针线辅料 / 扣件 / 拉链件 / 小配件 / 线材小件` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `提篮 / 收纳篮`、`工具包 / 布包 / 防尘袋 / 收纳袋`、`布样册 / 图册 / 记录本 / 文件夹`、`布料卷筒 / 带卷 / 线卷` 这几类工位服务壳体链。

## 5. 第四池专项

- 第四池当前的根因不是个别坏词，而是整池写法站在“收纳、资料管理和清洁维护系统”的视角，而不是“手作缝纫前台工位”的视角。
- 结果是四类问题同时叠加：清洁维护器具成串、提篮工具包成串、图册和记录本成串、卷筒和辅料卷成串。
- 本类第四池应优先补“普通玩家一眼就能认出的缝纫桌面、裁剪区、熨烫区或成品展示位”里的自然中型单物，而不是继续补工具包、收纳袋、记录本和卷筒。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留缝纫机、裁剪工具、测量工具、熨烫工具和少量稳定材料主物，必要时压缩纸样册、布料包、半成品和辅料卷的横向平移。
- 第二池：从“机件配件壳与收纳保护壳混池”拉回“真正辅助工具池”，优先保留穿针、拆线、打孔、测量、整形、压缝和扣件处理动作里会直接上手的物件本体。
- 第三池：继续保持 `一盒 / 一包 / 一套 / 一组 / 一排 / 一叠 / 一卷 / 一筒 / 一束 / 一盘 / 一册` 的可见单位格式，但避免针线辅料、扣件和小配件继续膨胀。
- 第四池：按“手作缝纫前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用提篮、工具包、收纳袋、图册、记录本和卷筒作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `篮 / 包 / 袋 / 册 / 本 / 卷 / 筒 / 盒` 是否还成片出现。

## 7. 本次修改记录

- 本轮实际修改了 `blind_boxes.py` 中 `手作+缝纫` 的 `support_items` 和 `scene_expansion_items`，`core_items` 与 `visible_small_items` 暂不大动。
- 第二池的修改方向是把“机件配件壳、收纳壳和标记辅材壳混池”拉回“真实穿针、拆线、打孔、测量、整形、压缝和扣件处理工具池”。
- 具体清掉了 `手缝针册 / 机针替换包 / 绣花针套筒 / 珠针收纳筒 / 安全别针盒 / 滚刀替换片筒 / 描图纸卷 / 划粉轮盘 / 划粉笔筒 / 布用铅笔筒 / 梭芯盒 / 梭壳盒 / 压脚盒 / 拉链压脚盒 / 包边压脚盒 / 扣眼压脚盒 / 滚边压脚盒 / 机针盒 / 送布牙盒 / 缝纫机防尘罩` 这类弱本体壳层。
- 回填时改用 `滚刀换片器 / 纸样锥 / 描线尺 / 描图笔 / 划粉轮 / 领口整形器 / 包扣工具 / 抽褶器 / 拉链定位尺 / 珠针夹 / 线头剪 / 压线轮 / 压角棒 / 止口夹 / 压褶夹 / 梭芯绕线器 / 针板清洁刷 / 压脚清洁刷 / 布样剪 / 绣绷调节器` 等更直接参与动作的工具本体。
- 第四池本轮按高风险家族整段清理，移除了 `提篮 / 工具包 / 布包 / 收纳袋 / 防尘袋 / 图册 / 记录本 / 卷筒` 这类收纳、资料管理和清洁维护系统写法，并额外清掉了 `地毯卷 / 花盆 / 图册 / 画册` 这几组最容易模板化、也最容易把缝纫工位写回旧词链的后缀家族。
- 第四池回填改成“时间物 + 小型环境设备 + 布艺地面物 + 工位分区 + 纸样拼贴与工艺样本 + 成品展示面”这一路线，尽量让它更像缝纫前台工位自然可见的中型场景补充物，而不是工作室后场收纳区。
- 本轮明确不再沿用 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 画册图册` 这类容易跨类别复用的旧生成习惯，改为直接补 `布料分区 / 裁片分区 / 线轴阵列 / 纸样拼贴 / 绗缝样本 / 包边样本 / 围裙展示面 / 拼布展示面` 这类只属于缝纫工位语境的中型物。
- 后续微调又继续压缩了 `样本` 与 `展示面` 家族，把 `压褶样本 / 包边样本 / 锁边样本 / 袖口样本 / 领口样本 / 口袋样本 / 抽绳样本` 改写为 `操作面 / 半成品` 结构，并把 `围裙展示面 / 托特展示面 / 抱枕展示面` 打散成 `围裙套组 / 托特阵列 / 抱枕布面`，避免第四池再次被单一后缀接管读感。

## 8. 修改后复审

- 当前四池数量均为 50，且池内唯一。
- 第四池与其余三池无重叠。
- 第四池本轮本地复查未命中禁词，且全部通过 `_is_medium_scene_fragment_item` 的中型场景边界校验。
- 第四池中的 `地毯卷 / 毯面 / 席面 / 花盆 / 图册 / 画册 / 样本 / 展示面` 当前实值分别为 `2 / 2 / 2 / 0 / 0 / 0 / 2 / 2`，已经不再出现同后缀连续堆叠压读感的问题。
- 这说明本轮已经先把“结构性错误”修掉了，尤其是第四池不再被 `篮 / 包 / 袋 / 册 / 本 / 卷 / 筒 / 盒` 或 `地毯卷 / 花盆 / 图册 / 画册` 这类工位服务和旧模板壳主导。
- 从读感上看，第二池已经明显回到缝纫辅助工具池；第四池也从“收纳、资料管理和清洁维护系统”转回“手作缝纫前台场景补充物”。
- 本节结论仅针对 `手作+缝纫` 的定点复查，不代表全仓所有类别已全量通过。

## 9. 剩余风险与暂留理由

- 第一池里仍保留了少量材料壳、半成品壳和辅料壳，例如 `纸样册`、`拼布图样册`、`服装纸样包`、`绣花图案册`、`棉布匹`、`亚麻布匹`、`牛仔布料`、`毛毡布料`、`帆布布料`、`衬布料`、`热熔衬布`、`拼布布片包`、`绗缝棉絮包`、`围裙半成品`、`托特包半成品`、`靠垫套半成品`、`布偶半成品`、`绣线卡册`、`缝纫线筒`、`包边带卷`、`斜纹带卷`、`花边带卷`、`拉链卷`、`纽扣铁盒`；这轮暂不动，是为了先稳住主体池，不把问题从第四池转移到第一池。
- 第三池仍保留了较多针线辅料、扣件、拉链件和线材小件，例如 `一排梭芯`、`一盒线轴帽`、`一包压脚螺丝`、`一盘纸样压块`、`一册布样册`、`一排短拉链`、`一包拉链头`、`一盒纽扣`、`一包按扣`、`一包四合扣`、`一包铆扣`、`一包鸡眼扣`、`一包珠片`、`一筒串珠`、`一束绣线`、`一排彩色线轴`；它们当前格式合规，但后续如果继续做第三池精修，可以再轻压一次。
- 第四池虽然已经脱离“收纳和资料壳链”，也清掉了 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 画册图册` 这条跨类别复用链，但现在仍有 `钟 / 历 / 机 / 卷 / 面 / 分区 / 阵列 / 拼贴 / 样本 / 展示面` 这些自然家族；后续若继续微调，重点应放在继续压模板密度，避免再把 `样本 / 展示面` 重新堆高，而不是重新引入 `篮 / 包 / 袋 / 册 / 本 / 卷 / 筒 / 盒`。
- 当前版本已经可以作为后续同类修改的合格基线，下一步若继续优化，建议优先做“第一池材料半成品轻压”和“第三池针线扣件单位轻压”，而不是再大改第二池和第四池。
