# 海底+潜水 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `海底+潜水` 的四池内容。
- 本轮结论：当前版本的第一池仍能读出“潜水与海底探索”的主体场景，但已经混入较多采样容器、测绘仪器和沉船遗物簇；第二池被 `带 / 绳 / 套 / 罩 / 盒 / 册 / 袋 / 篮` 的服务壳和记录壳拉散；第四池明显没有通过，已经被写成甲板准备区、救援服务区和采样管理区的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的潜水装备主场景还在，但后半段已经明显抬高“测绘取样区”和“沉船考古区”的比重。
- `support_items` 的主要问题不是与潜水无关，而是把真实辅助件、绑带护套、记录册、样本壳和收纳壳混在了一池里。
- `visible_small_items` 格式合规，但标签片、样本件、滤筒和封袋叠加后，已经很像“采样标记补给包”。
- `scene_expansion_items` 是当前最大问题：桶、盆、袋、册、卷、包、篮、定位器和应急设备集中回流，明显偏离“前台自然中型物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池最大问题不是整体失场，而是 `瓶 / 管 / 箱 / 仪` 与沉船遗物家族已经开始挤占潜水主体位。
- 低优先级平衡项：`防水样本瓶`、`水样采集瓶`、`岩芯样本管`、`海底测量箱`、`海底记录仪`、`海底探钩`、`手持取样泵`、`水下抓取夹`、`沉船船锚`、`沉船舵轮`、`船体舷窗`、`船钟`、`木质货箱`、`陶瓷双耳罐`、`石质压舱块`。
- 这些条目不一定要立刻整片清洗，但它们说明本池已经开始用“取样器材、测绘器材和沉船遗物”维持密度；后续若要腾位，可优先从这组压缩。
- 当前可作为锚点保留的方向：`潜水面罩`、`全脸潜水面罩`、`呼吸管`、`潜水脚蹼`、`潜水湿衣`、`潜水干衣`、`潜水头套`、`潜水靴`、`潜水手套`、`浮力控制背心`、`潜水气瓶`、`双瓶背架`、`呼吸调节器`、`备用呼吸器`、`潜水配重带`、`潜水配重块`、`潜水电脑表`、`指针深度计`、`潜水压力表`、`潜水罗盘`、`水下写字板`、`导航线卷轴`、`潜水浮标`、`延迟水面浮标`、`救援浮袋`、`水下手电筒`、`水下探照灯`、`水下推进器`、`水下相机`、`相机防水壳`、`水下摄像机`、`防水定位信标`、`声呐测绘仪`、`手持金属探测器`、`水下测距尺`。

### support_items

- 本池被记录壳、样本壳和服务壳拉得过散，优先替换：`湿衣修补贴包`、`调节器防尘罩`、`配重袋`、`深度计护套`、`压力表护壳`、`防水铅笔筒`、`浮标卷盘`、`导航线卷盘`、`测量绳卷`、`推进器电池盒`、`样本记录册`、`样本封袋`、`采样铲护套`、`取样泵滤筒`、`海底测绘册`、`潜水记录册`、`潜点计划册`、`装备网篮`、`防水急救包`、`备用电池盒`、`防水文件夹`、`潜水检查册`。

### scene_expansion_items

## 4. 多样性与模板密度

- 第一池当前最明显的压力不是“潜水主体物太多”，而是 `瓶 / 管 / 箱 / 仪` 与沉船遗物家族一起抬高了场景杂音。
- 第二池最拥挤的尾词是 `带 / 绳 / 套 / 罩 / 盒 / 册 / 袋 / 篮`，主要问题是“不同被服务对象 + 相同记录壳/保护壳”的伪多样性。
- 第三池格式没错，但 `标签件 / 样本件 / 标记件 / 电池盒 / 封袋` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `桶 / 盆 / 冲淋袋`、`地图册 / 图鉴 / 日志册 / 记录本`、`提篮 / 布袋 / 工具包 / 急救包`、`绳卷 / 胶带卷 / 测线卷 / 浮绳卷` 这几类甲板服务壳体链。

## 5. 第四池专项

- 第四池当前的根因不是个别坏词，而是整池写法站在“甲板准备、救援服务和采样管理系统”的视角，而不是“水下前台场景补充物”的视角。
- 结果是四类问题同时叠加：清洗浸泡物成串、地图日志和记录册成串、提篮布袋和急救包成串、绳卷胶带卷和应急设备成串。
- 本类第四池应优先补“普通玩家一眼就能认出的水下作业区、沉船附近、潜水下潜点或海底探查区”里的自然中型单物，而不是继续补冲洗盆、地图册、提篮、绳卷和救援包。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留潜水装备、照明、推进、定位和典型海底探索物这些稳定主体，必要时压缩采样容器、测绘器材和沉船遗物的横向平移。
- 第二池：从“采样记录壳与装备保护壳混池”拉回“真正辅助工具池”，优先保留除雾、清洁、固定、信号和简单维护动作里会直接上手的物件本体。
- 第三池：继续保持 `一盒 / 一包 / 一组 / 一套 / 一排 / 一卷 / 一筒 / 一叠 / 一盘 / 一袋` 的可见单位格式，但避免样本件、标签件、封袋件和标记件继续膨胀。
- 第四池：按“水下前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用清洗盆、地图册、急救包、提篮、绳卷和甲板服务设备作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `桶 / 盆 / 袋 / 包 / 册 / 本 / 卷 / 器` 是否还成片出现。

## 7. 本次修改记录

- 本轮实际修改了 `blind_boxes.py` 中 `海底+潜水` 的 `support_items` 和 `scene_expansion_items`，`core_items` 与 `visible_small_items` 暂不大动。
- 第二池的修改方向是把“采样记录壳、装备保护壳和收纳壳混池”拉回“真实除雾、清洁、固定、信号和简单维护工具池”。

- 本轮明确不再沿用 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 画册图册` 这类容易跨类别复用的旧生成习惯，改为直接补 `珊瑚礁座 / 海扇珊瑚座 / 鹿角珊瑚座 / 礁石洞口座 / 沉船舷窗阵列 / 压舱石阵列 / 海胆阵列` 这类只属于海底潜水语境的中型物。

## 8. 修改后复审

- 当前四池数量均为 50，且池内唯一。
- 第四池与其余三池无重叠。
- 第四池本轮本地复查未命中禁词，且全部通过 `_is_medium_scene_fragment_item` 的中型场景边界校验。
- 第四池中的 `地毯卷 / 毯面 / 席面 / 花盆 / 图册 / 画册` 当前实值分别为 `0 / 2 / 2 / 0 / 0 / 0`，已经不再出现同后缀连续堆叠压读感的问题。
- 这说明本轮已经先把“结构性错误”修掉了，尤其是第四池不再被 `桶 / 盆 / 袋 / 包 / 册 / 本 / 卷 / 器` 或 `地毯卷 / 花盆 / 图册 / 画册` 这类甲板后勤、陆地绿植和阅读记录壳主导。
- 从读感上看，第二池已经明显回到潜水辅助工具池；第四池也从“甲板准备、救援服务和采样管理系统”转回“潜水前台场景补充物”。
- 本节结论仅针对 `海底+潜水` 的定点复查，不代表全仓所有类别已全量通过。

## 9. 剩余风险与暂留理由

- 第一池里仍保留了少量取样器材、测绘器材和沉船遗物，例如 `防水样本瓶`、`水样采集瓶`、`岩芯样本管`、`海底测量箱`、`海底记录仪`、`海底探钩`、`手持取样泵`、`水下抓取夹`、`沉船船锚`、`沉船舵轮`、`船体舷窗`、`船钟`、`木质货箱`、`陶瓷双耳罐`、`石质压舱块`；这轮暂不动，是为了先稳住主体池，不把问题从第四池转移到第一池。
- 第三池仍保留了较多样本件、标签件、封袋件和标记件，例如 `一盒防水标签片`、`一叠潜水记录卡`、`一套减压停留牌`、`一盒样本标签牌`、`一组防水样本瓶`、`一排岩芯样本管`、`一盒水样瓶塞`、`一包样本封袋`、`一包防水封口带`、`一排装备号码牌`；它们当前格式合规，但后续如果继续做第三池精修，可以再轻压一次。
- 第四池虽然已经脱离“甲板后勤壳链”，也清掉了 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 画册图册` 这条跨类别复用链，但现在仍有 `钟 / 历 / 机 / 面 / 座 / 分区 / 组合 / 阵列` 这些自然家族；后续若继续微调，重点应放在继续压模板密度，而不是重新引入 `桶 / 盆 / 袋 / 包 / 册 / 本 / 卷 / 器`。
- 当前版本已经可以作为后续同类修改的合格基线，下一步若继续优化，建议优先做“第一池取样遗物轻压”和“第三池样本标签单位轻压”，而不是再大改第二池和第四池。
