# 浴室+洗护 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `浴室+洗护` 的四池内容。
- 本轮结论：当前版本的第一池仍在场景内，瓶罐密度虽然高但大体可接受；第二池已经明显滑向 `盒 / 筒 / 架 / 垫 / 托 / 分装壳` 堆叠，第四池仍然没有通过，核心问题是它被写成了纸品盒、垃圾桶、托盘、布包、收纳壳和小型卫浴设备的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `visible_small_items` > `core_items`。

## 2. 问题总览

- `core_items` 的浴室台面感仍然成立，但 `瓶 / 罐 / 管 / 卷 / 巾` 家族已经偏密，部分条目更像洗护包装壳或成套软物横向平移。
- `support_items` 的主要问题不是不属于洗护，而是过度依赖 `盒 / 筒 / 架 / 垫 / 托 / 座 / 分装壳` 这类承载壳和保护壳来制造多样性。
- `visible_small_items` 格式合规，但旅行装、替换件和一次性洗护耗材已经成串，读感开始偏向“洗护补给包”。
- `scene_expansion_items` 是当前最大问题：纸品盒、垃圾桶、收纳篮、托盘、布包、香薰摆件和小型卫浴设备同时回流，已经偏离“浴室洗护前台自然单物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池没有出现必须整片清洗的问题，但瓶罐壳和平行软物已经过密。
- 低优先级平衡项：`漱口水瓶`、`洗手液瓶`、`卸妆膏罐`、`爽肤水瓶`、`乳液泵瓶`、`面霜罐`、`精华滴管瓶`、`洗发水瓶`、`护发素瓶`、`发膜罐`、`沐浴露瓶`、`身体乳瓶`、`沐浴油瓶`、`浴盐罐`、`浴巾卷`、`手巾卷`、`干发帽`、`干发巾`。
- 这些条目不是立刻必换，而是说明本池已经在用“不同内容物 + 相似瓶罐壳”以及“不同软物 + 相近形态”维持密度；后续若要腾位，可优先从这组下手。
- 当前可作为锚点保留的方向：`电动牙刷`、`冲牙器`、`香皂碟`、`洁面刷`、`洁面仪`、`面膜碗`、`长柄身体刷`、`洗脸盆`、`头皮按摩刷`、`剃须刀`、`剃须刷`、`浴室体重秤`。

### support_items

- 承载壳、分装壳和保护壳过密，优先替换：`冲牙喷嘴筒`、`牙套清洁盒`、`牙刷沥水架`、`漱口杯垫`、`香皂沥水垫`、`洁面海绵托`、`护肤刮勺盒`、`化妆棉盒`、`棉签盒`、`发卷筒`、`发夹盘`、`剃须刷架`、`剃须刀座`、`修甲工具盒`、`化妆刷晾干架`、`隐形眼镜盒`、`旅行分装瓶`、`旅行皂盒`、`干湿分装袋`、`排水口过滤篮`、`小号收纳盘`、`洗脸腕带包`、`发圈收纳筒`、`电动牙刷充电座`、`护肤分装罐`、`浴球挂绳`。
- 问题根因不是这些词完全不能出现，而是这一池已经过度依赖“工具 + 壳体”或“耗材 + 分装壳”写法来撑多样性。
- 当前更稳的辅助物方向仍应放在真正参与洗护、刮水、起泡、染发、修脚、清洁和整理动作的工具本体，如 `牙膏挤压器`、`洁面起泡器`、`面膜调棒`、`染发梳`、`浴盐量勺`、`硅胶搓澡刷`、`修脚磨板`、`指甲刷`、`梳子清洁刷`、`化妆刷清洁垫`、`化妆刷清洁杯`、`除雾擦`、`台面刮水器`、`玻璃刮水器`。

### visible_small_items

- 整体格式合规，暂不建议整池推倒。
- 低优先级压缩项：`一袋漱口片`、`一瓶漱口水旅行装`、`一排精华安瓶`、`一组乳液旅行瓶`、`一罐面霜小样`、`一排防晒乳小管`、`一组护手霜小管`、`一袋固体洗发皂`、`一排洗发水旅行装`、`一排护发素旅行装`、`一排沐浴露旅行装`、`一组身体乳小瓶`。
- 这些条目本身不违规，但它们和前两池、第四池的分装壳条目叠加后，会进一步放大“浴室补给包”的读感。

### scene_expansion_items

- 纸品壳、收纳壳和桶篮回流，必须优先替换：`抽纸盒`、`湿巾盒`、`棉柔巾盒`、`卷纸筒`、`备用卷纸筒`、`浴室垃圾桶`、`脚踏垃圾桶`、`毛巾篮`、`脏衣篮`、`泡脚桶`。
- 垫、托盘和承载壳回流，必须优先替换：`防滑踏垫`、`吸水脚垫`、`台面防水垫`、`台面托盘`、`木质托盘`、`玻璃托盘`、`香薰石盘`、`首饰盘`、`发饰盘`、`香皂木盒`、`化妆棉木盒`、`棉签玻璃筒`、`梳子筒`。
- 布包、洗漱包和泛收纳物回流，必须优先替换：`旅行化妆包`、`防水洗漱包`、`毛巾布包`、`浴帽布包`。
- 小型卫浴设备和泛家居摆件偏多，必须整体压缩：`毛巾加热器`、`除湿机`、`暖风机`、`小型消毒机`、`手持按摩器`、`足浴按摩器`、`翻页台历`、`桌面月历`、`桌面收音机`、`陶瓷小花盆`、`苔藓玻璃盆`、`浴室绿植盆`、`玻璃烛台座`、`陶瓷烛台座`。
- 当前较接近第四池方向、可作为保留候选的仅有少数：`吹风机`、`香薰机`、`蒸脸器`、`美容喷雾器`、`台面小钟`、`沙漏计时器`、`白噪音机`、`眼部按摩器`。

## 4. 多样性与模板密度

- 第一池最拥挤的家族是 `瓶 / 罐 / 管 / 卷 / 巾`，说明主体池已经出现洗护包装壳和软物成套平移。
- 第二池最拥挤的尾词是 `盒 / 筒 / 架 / 垫 / 托 / 座 / 瓶 / 罐`，主要问题是“不同小工具 + 相同承载壳”的伪多样性。
- 第三池格式没错，但 `旅行装 / 小样 / 替换件 / 一次性耗材` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `纸品盒 / 垃圾桶 / 收纳篮`、`托盘 / 盘 / 垫`、`洗漱包 / 布包 / 分装壳`、`花盆 / 烛台 / 台历 / 收音机` 这类氛围摆件链。

## 5. 第四池专项

- 第四池当前的根因不是单个坏词，而是整池写法站在“浴室收纳与清洁备品”视角，而不是“浴室洗护前台场景”视角。
- 结果是四类问题同时叠加：纸品壳堆叠、桶篮和托盘堆叠、洗漱包和分装壳堆叠、小型设备和氛围摆件混入。
- 本类第四池应优先补“普通玩家一眼就能认出的浴室台面或洗护前台中型物、自然外露的稳定设备或场景补充物”，而不是继续补纸巾盒、托盘、桶篮、布包和分装壳。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留稳定的洗护主体物、刷具、盆具、剃须物和少量高频软物，必要时压缩不同内容物共用相似瓶罐壳的条目。
- 第二池：从“承载壳池”拉回“辅助工具池”，优先补真正参与洗护、清洁、起泡、刮水、整理和护理动作的工具本体。
- 第三池：继续保持 `一盒 / 一卷 / 一筒 / 一组 / 一排 / 一包` 的可见单位格式，但避免旅行装、小样和替换件继续膨胀。
- 第四池：按“洗护前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用纸品盒、托盘、桶篮、布包和分装壳作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `盒 / 筒 / 盘 / 垫 / 桶 / 篮 / 包` 是否还成片出现。

## 7. 本次修改记录

- 已按本单完成第二轮落库修改，修改文件：`Game content extraction/data/blind_boxes.py`。
- `core_items`：
  这轮不再把第一池完全放着不动，而是轻压几处最明显的瓶罐 / 软物平移，把 `洗手液瓶`、`沐浴油瓶`、`干发巾`、`须后水瓶` 调整为更像真实浴室主体物的 `感应皂液器`、`浴室拖鞋`、`浴袍`、`剃须镜`，把上一轮仍偏“不同内容物 + 相似瓶壳”和“不同软物 + 相近卷巾”的读感再往下压一层。
- `support_items`：
  本轮未继续改第二池，沿用上一轮已经完成的工具本体化结果，不再在这一池追加新的壳体替换，避免把本轮重点打散。
- `visible_small_items`：
  本轮仍未改第三池；旅行装、小样和替换件偏密的问题继续记账，但不是这次“假修复重做”的主战场。
- `scene_expansion_items`：
  本轮把第四池按“二次重做”处理，针对上一轮新长出的 `机 / 器 / 钟 / 盆` 卫浴设备矩阵再次拆解，不再继续往 `护肤冷藏机`、`化妆刷清洗机`、`毛巾烘干机`、`电动除毛器`、`冷热美容器`、`洁面导入器` 这类同族设备上横向扩写。
  回填方向改成真正属于浴室洗护前台的不同物族混合：保留少量必要的吹风、香薰、蒸脸、除湿、暖风和计时设备，同时补入 `防水洗衣袋`、`网眼洗衣袋`、`热水袋`、`冰敷袋` 这类真实袋包，`地巾卷`、`防水浴帘卷` 这类稳定织物卷，`软毛清洁刷`、`玻璃清洁刷`、`浴缸清洁刷`、`马桶清洁刷`、`镜面清洁擦` 这类浴室里一眼能认出的独立工具，以及 `浴帘挂环阵列`、`牙杯阵列`、`香皂阵列`、`护肤分区`、`沐浴分区` 这类更贴近洗护前台的中型结构物，让第四池从“卫浴小设备陈列串”退回到更像真实浴室前台会自然出现的中型单物混合池。
  本轮额外把 `xxx卷` 链压到只保留两个，不再拿 `浴袍卷 / 擦手巾卷 / 棉布浴帘卷 / 竹纤维毛巾卷 / 棉布方巾卷` 这种平行软物继续堆密度；同时没有再用 `绿植盆 / 玻璃盆` 这类旧补位习惯去填空。

## 8. 修改后复审

- 已完成本类第二轮结构复查。
- `浴室+洗护` 四池当前均为 50 条，且池内唯一。
- `scene_expansion_items` 与其他三池无机械精确重复。
- 按 `test_blind_box_content_model.py` 的同口径做了本类专项校验：
  `scene_expansion_items` 本类 50 条全部通过中型边界检查；
  本类四池均未命中 blocked patterns；
  第四池与前三池无机械精确重叠。
- 第四池里当前以 `卷` 结尾的条目已经收敛到 `2` 个，仅保留 `地巾卷 / 防水浴帘卷`，不再出现同后缀成串压读感的问题。
- 共享规则测试目前仍由其他尚未改完的类别拖失败，这一轮不再把“全量红灯”误判成本类失败信号；本类新增内容已经过专项脚本复查。
- 人审结论：
  相比上一轮，这次真正压掉的是“虽然不再是纸品盒 / 桶篮 / 托盘链，但又长成另一串卫浴小设备矩阵”的假修复；
  当前第四池已经更像浴室前台会自然共存的设备、布卷、袋包、清洁刷和少量陈设物，而不是一排功能相近的小机器；
  当前版本可以作为 `浴室+洗护` 的新基线继续使用。

## 9. 剩余风险与暂留理由

- 本轮已经解决的根因是“上一轮虽然摆脱了纸品盒 / 桶篮 / 托盘链，却又长成另一批 `xx机 / xx器 / xx钟 / xx盆` 的假修复矩阵”。
- 仍保留两类中低风险：
  `visible_small_items` 里的旅行装、小样和替换件单位仍然偏多，本轮没有动它；
  `scene_expansion_items` 里仍保留一定比例的设备、画册和盆栽，只是已经不再被单一的设备后缀或 `卷` 后缀主导。
- 这些残余风险本轮暂留的原因是：
  先优先修正第四池读感最差的假修复链，避免为了继续压密度，马上把第四池又改回另一串 `盒 / 筒 / 盘 / 桶 / 包` 的新模板；
  `浴室+洗护` 本身允许一定比例的吹风、香薰、蒸脸、暖风和清洁工具，完全压平反而容易失真。
- 如果下一轮继续细修，本类优先级建议是：
  先轻压第三池后半段的旅行装与替换件串联，再视需要微修第一池中后段的 `瓶 / 罐 / 管 / 卷` 密度；
  第四池不要回退到 `托盘 / 桶篮 / 布包 / 纸品盒`，也不要再长成新的纯设备矩阵，只做小幅平衡。
