# 浴室+洗护 审查文档

审查日期：2026-05-07

## 审查对象

- 类别：`浴室+洗护`
- 数据来源：`Game content extraction/data/blind_boxes.py` 中 `BLIND_BOX_ITEM_POOL_BUNDLES['浴室+洗护']`
- 审查范围：仅审查该类别 `core_items` / `support_items` / `visible_small_items` / `scene_expansion_items` 四池内容
- 本次操作：已同步修改 `blind_boxes.py` 中该类别数据，并重写本审查文档

## 总体结论

状态：`已修改并复审`

这类原先最明显的问题，不是场景错了，而是“看起来都对、实际都像同一种东西在换说明”：

- `core_items` 之前大量是瓶、管、罐换内容物，很多差异主要靠洗护品名称成立。
- `support_items` 之前混入了不少过细的小配件、洗衣后场物和排水清洁物，辅助池边界不稳。
- `visible_small_items` 之前明显在镜像核心池，只是把同类洗护容器改成 `一盒 / 一排 / 一组 / 一瓶`。
- `scene_expansion_items` 之前同时有壳体链和后场链：`盒 / 筒 / 包 / 篮 / 架 / 垫 / 托盘` 连续成片，外加清洁、洗衣、安装、防水、电器保护等偏后场内容。

## 本次修改方向

- `core_items`：保留浴室洗护最核心的口腔护理、洁面护肤、洗发沐浴、毛巾软物和剃须护理主线，但压掉一串“同瓶身换内容”的横向扩写，补入 `洁面仪`、`面膜刷`、`头皮按摩刷`、`按摩发梳`、`足部磨砂板` 这类轮廓更不同的主体物。
- `support_items`：删除洗衣链和一批极细小配件，改成更稳定的取用、沥水、清洁、修整、旅行分装和镜前辅助物，减少“零件清单感”。
- `visible_small_items`：保留可见单位规则，但压掉对核心池的机械镜像，改成牙线、棉签、面膜、旅行装、浴盐、发饰、剃须替换件、护足小件等更分散的小单位。
- `scene_expansion_items`：把第四池拉回到纸品容器、地面软垫、垃圾桶、毛巾篮、泡脚桶、小型前台设备、钟历、托盘、花盆、烛台和少量常见软包，不再让它主要由清洁维修物、洗衣后场物和连续壳体构成。

## 本次主要压掉的伪多样性

- `不同洗护内容 + 相同瓶 / 管 / 罐壳体`
- `不同前缀 + 相同盒 / 筒 / 包 / 篮 / 架后缀`
- `不同小样内容 + 相同旅行装模板`
- `不同功能词 + 相同第四池服务壳`
- `看似贴场景，实际属于洗衣 / 排水 / 安装 / 防护后场`

## 简要修改理由

- 把主要靠内容物说明成立的名称，换成轮廓更明确、普通用户更容易一眼识别的对象。
- 把过小、过薄、过依赖包装说明的小件，从核心池和第四池里压回到更合适的位置，或直接删除。
- 把第四池从“辅助壳体池”改回“前台中型单物池”，让多样性来自物体家族差异，而不是换前缀。

## 复审结果

- 四池均为 50 条，池内唯一。
- `scene_expansion_items` 与其他三池无精确重复。
- `scene_expansion_items` 已通过中型场景碎片边界自检，未命中超大物和微小信息面禁区。
- 当前版本已明显压低 `core_items` 的瓶罐链、`support_items` 的细小配件链、`visible_small_items` 的旅行装镜像链，以及第四池的后场清洁 / 洗衣 / 安装偏移。
- 第四池现在更像浴室洗护前台和洗漱区周边会自然出现的中型物，而不是壳体补位池或后场工具池。

## 保留风险

- `浴室+洗护` 天然仍会保留较高的瓶、管、罐密度，这是类别识别需要；后续补写时仍要继续拦截“只换内容物、不换物体本体”的扩写。
- `visible_small_items` 里仍会出现一定数量的旅行装和护理小样，这是类别自然组成；但后续若继续补写，优先补更不同的小单位，不要再回到连续小瓶小管链。
- `scene_expansion_items` 目前仍保留少量盒、盘、筒、包，这是可接受的真实前台物；但后续若继续补写，不能再滑回连续服务壳体路线。
