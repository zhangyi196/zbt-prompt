# 厨房+烘焙 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `厨房+烘焙` 的四池内容。
- 本轮结论：当前版本的第一池和第二池总体比前两个类别更稳，但第二池后半段已经出现明显的 `罐 / 瓶 / 喷罐` 扩写，第四池仍然没有通过，核心问题是它被写成了托盘 / 盒筒 / 包袋 / 垫板 / 后场小机器 / 泛家居摆件的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的厨房台面感基本成立，但 `罐 / 盒 / 袋 / 篮 / 壶` 家族已经偏密，部分条目更像食材包装或承载壳横向平移。
- `support_items` 整体方向比 `餐桌+茶歇` 更好，因为大部分仍是工具本体；但后半段开始明显依赖 `研磨罐 / 挤瓶 / 滴管瓶 / 喷罐` 这类分配容器来撑多样性。
- `visible_small_items` 格式合规，但纸杯托、塔壳胚和装饰糖家族偏密，读感已经有“烘焙耗材补给包”的倾向。
- `scene_expansion_items` 是当前最大问题：大量条目首先读成收纳壳、服务壳、后场辅助机或泛装饰物，而不是烘焙操作台前台自然出现的中型单物。

## 3. 按池问题与必换项

### core_items

- 本池没有出现必须整片清洗的问题，但存在明显的包装壳和平行容器扩写。
- 低优先级平衡项：`面粉罐`、`砂糖罐`、`香草精瓶`、`曲奇铁盒`、`马卡龙盒`、`蜂蜜瓶`、`橄榄油瓶`、`可可粉袋`、`杏仁粉袋`、`柠檬篮`、`浆果篮`、`面糊分配壶`。
- 这些条目不是立刻必换，而是说明本池已经在用“不同内容物 + 相似容器壳”维持密度；后续若要腾位，可优先从这组下手。
- 当前可作为锚点保留的方向：`台式厨师机`、`手持搅拌机`、`台面小烤箱`、`厨房电子秤`、`矩形烤盘`、`圆形蛋糕模`、`吐司模`、`马芬烤盘`、`搅拌盆`、`擀面杖`、`蛋糕转台`、`冷却网`、`揉面垫`、`面包发酵篮`、`巧克力熔锅`。

### support_items

- 本池不需要推倒重写，但后半段的分配容器和喷罐写法过密，优先替换：`坚果研磨罐`、`肉桂研磨罐`、`糖浆挤瓶`、`巧克力滴瓶`、`香草滴管瓶`、`喷雾油瓶`、`糖粉摇罐`、`蛋糕脱模喷罐`。
- 次级观察项：`奶油枪`、`面糊分配器`、`可调蛋糕圈` 这类条目虽然是具体工具，但已经靠近专业设备扩写边界，后续应注意不要继续朝这一方向堆叠。
- 当前更稳的辅助物方向仍应放在真正参与搅拌、切分、筛粉、压模、刮抹、夹取、去核、扎孔的工具本体，如 `量勺`、`打蛋器`、`硅胶刮刀`、`奶油抹刀`、`蛋清分离器`、`面包刀`、`饼干压模`、`糕点夹`、`派皮扎孔器`、`宽口漏斗`。

### visible_small_items

- 整体格式合规，暂不建议整池推倒。
- 低优先级压缩项：`一叠马芬纸杯`、`一包纸杯蛋糕托`、`一筒木质搅拌棒`、`一盒蛋挞壳`、`一盘水果挞壳`、`一盘迷你塔壳`、`一盒蛋白霜糖`、`一盒糖霜花`。
- 这些条目本身不违规，但它们和第二池、第四池的容器壳条目叠加后，会进一步放大“烘焙耗材补给区”的读感。

### scene_expansion_items

- 收纳壳和承载壳回流，必须优先替换：`烘焙工具篮`、`香料盒`、`鸡蛋托盘`、`小号称量盘`、`木质托盘`、`糕点托盘`、`蛋糕提盒`、`面包盒`、`纸杯叠放筒`、`搅拌棒筒`、`擦手巾筒`、`烤箱手套筒`、`围裙布包`、`木柄厨具筒`、`糖包木盒`、`茶包木盒`、`香料木盒`、`湿巾抽取盒`。
- 第二池工具和后场辅助机回流，必须优先替换：`台面刮板`、`软木锅垫`、`硅胶隔热垫`、`食谱支架`、`真空封口机`、`杏仁研磨机`、`奶油起泡器`、`冷饮搅拌器`、`手持撒粉器`、`柠檬榨汁器`、`果酱挤压器`、`迷你开瓶器`、`玻璃餐盘罩`。
- 泛家居摆件和弱相关氛围物偏多，必须整体压缩：`翻页台历`、`桌面月历`、`留声机`、`音乐盒`、`复古摆钟`、`桌面收音机`、`陶瓷小花盆`、`苔藓玻璃盆`、`干花花盆`、`鲜花花篮`、`香薰蜡烛座`、`客用留言本`。
- 当前较接近第四池方向、可作为保留候选的仅有少数：`厨房计时钟`、`沙漏计时器`、`木质案板`、`面包保温篮`、`水果浅篮`。

## 4. 多样性与模板密度

- 第一池最拥挤的家族是 `罐 / 盒 / 袋 / 篮 / 壶`，说明主体池已经出现食材包装和容器横向平移。
- 第二池最拥挤的尾词是 `罐 / 瓶 / 器 / 刀 / 模`，其中真正危险的是后半段 `挤瓶 / 滴瓶 / 喷罐 / 研磨罐` 形成的分配容器家族。
- 第三池格式没错，但 `纸杯托 / 塔壳胚 / 糖装饰` 家族已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `托盘 / 盒 / 筒 / 篮`、`垫 / 板 / 支架`、`后场小机器`、`台历 / 收音机 / 花盆 / 留言本` 这类泛家居摆件链。

## 5. 第四池专项

- 第四池当前的根因不是个别坏词，而是整池写法站在“工具备品和耗材收纳”视角，而不是“烘焙操作台前台场景”视角。
- 结果是四类问题同时叠加：收纳壳堆叠、服务壳堆叠、后场辅助机混入、泛家居摆件混入。
- 本类第四池应优先补“普通玩家一眼就能认出的厨房台面或烘焙操作台前台中型物、烘焙过程中自然外露的稳定单物”，而不是继续补盒、筒、托盘、垫、支架、抽取盒和小机器。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留稳定的设备、烤盘、模具、碗盆、成品点心和少量高频食材承载物，必要时压缩不同内容物共用相似容器壳的条目。
- 第二池：保持“烘焙工具本体池”方向，但压掉 `研磨罐 / 挤瓶 / 滴瓶 / 喷罐` 这类容器扩写。
- 第三池：继续保持 `一盒 / 一包 / 一盘 / 一排 / 一篮 / 一碗` 的可见单位格式，但避免纸品壳、塔壳胚和装饰糖继续膨胀。
- 第四池：按“操作台前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用工具壳、纸品壳、后场辅助机和泛装饰摆件作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `盒 / 筒 / 托盘 / 垫 / 支架 / 布包 / 抽取盒` 是否还成片出现。

## 7. 本次修改记录

- 已基于上一轮结果完成第二次重做，修改文件：`Game content extraction/data/blind_boxes.py`。
- 本轮重点不是继续沿用上一轮那套“换一批能过边界的设备词和拼盘词”，而是处理“旧坏家族被换成新坏家族”的假修复问题。
- `core_items`：
  轻修了第一池里最明显的内容物 + 包装壳写法，把 `曲奇铁盒`、`马卡龙盒`、`可可粉袋`、`杏仁粉袋`、`面糊分配壶` 改成更像独立点心或甜品主体物的 `玛德琳蛋糕`、`巧克力塔`、`焦糖布丁`、`芝士挞`、`烤布丁`。
- `support_items`：
  本轮未继续重写第二池，保留上一轮已经压下 `研磨罐 / 挤瓶 / 滴瓶 / 喷罐` 后的结构，避免同时把第二池和第四池一起重新打散。
- `visible_small_items`：
  本轮仍未动第三池；纸杯托、塔壳胚和糖装饰偏密的问题继续暂留，但不是这次重做的主矛盾。
- `scene_expansion_items`：
  整段撤掉了上一轮那批看起来通过、但读感又长成另一串弱模板的条目，如 `纸杯蛋糕架`、`马卡龙塔架`、`甜甜圈塔架`、`甜点拼盘`、`曲奇拼盘`、`水果拼盘`、`坚果拼盘`、`点心拼盘`、`胶囊咖啡机`、`酸奶机`、`果汁机`、`面包机` 等高密设备或内容词 + 盘 / 架 / 盆串联。
  改为更像真实烘焙台前中型物的组合：少量稳定计时与台面物，如 `厨房计时钟`、`厨房挂钟`、`复古座钟`、`翻页台历`；餐篮和冰桶类，如 `面包保温篮`、`藤编餐篮`、`小号冰桶`、`香槟冰桶`；真实台面盘盆类，如 `高脚甜点盘`、`长柄蛋糕盘`、`不锈钢打发盆`、`玻璃和面盆`；再辅以适量真实设备，如 `台式咖啡机`、`台式制冰机`、`桌面烤面包机`、`华夫饼机`、`巧克力喷泉机`、`热压三明治机`，以及少量稳定装饰物，如 `玻璃烛台座`、`银质烛台座`、`鲜花花篮`、`香草盆栽`。
- 本轮核心目标是把第四池从“拼盘 / 塔架 / 盆 / 小机器串”拉回更自然的厨房前台物，而不是继续用另一批边界友好词硬顶过去。

## 8. 修改后复审

- 已完成本类第二轮结构复查。
- `厨房+烘焙` 四池当前均为 50 条，且池内唯一。
- `scene_expansion_items` 与其他三池无机械精确重复。
- 按本类专项口径复查：
  `scene_expansion_items` 50 条全部通过中型边界检查；
  本类四池均未命中 blocked patterns；
  第四池与其他三池无跨池机械重复。
- 人审结论：
  相比上一轮，这次已经明确去掉了 `拼盘 / 塔架 / 盆 / 小机器串` 这一条假修复链；
  当前第四池虽然仍保留一定比例的设备、盘盆和篮桶，但整体已经更像烘焙操作台前台，而不是“换一批后缀继续过规则”；
  当前版本可以作为 `厨房+烘焙` 的新基线继续使用。

## 9. 剩余风险与暂留理由

- 本轮已经解决的根因是“上一轮虽然不再是托盘壳和抽取盒，但又长成了另一批 `拼盘 / 架 / 盆 / 机` 的弱模板串联”。
- 仍保留两类中低风险：
  `scene_expansion_items` 里的 `盘 / 盆 / 机` 家族仍有一定密度，只是已经比上一轮的 `拼盘 / 塔架 / 小机器串` 更自然；
  `visible_small_items` 后半段仍然偏纸杯托、塔壳胚和糖装饰单位，本轮暂未处理。
- 这些残余风险本轮暂留的原因是：
  先优先修正读感最差的假修复链条，避免为了继续压尾词又引入新的假自然词；
  `厨房+烘焙` 本身允许一定比例的设备、盘盆、篮桶和台面装饰物，完全压平反而容易失真。
- 如果下一轮继续细修，本类优先级建议是：
  先轻压第四池的 `盘 / 盆 / 机` 密度，再视需要微修第三池后半段的纸杯托与塔壳胚串联；
  第一池只在确认这两块稳定后，再做低优先级的容器横向平移压缩。
