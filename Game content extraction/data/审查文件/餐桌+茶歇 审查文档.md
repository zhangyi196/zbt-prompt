# 餐桌+茶歇 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `餐桌+茶歇` 的四池内容。
- 本轮结论：当前版本的第一池和第三池总体可用，但第二池已有明显的托、套、盒、盖密度问题，第四池仍然没有通过，核心问题是它被写成了托盘 / 罩子 / 立架 / 立筒 / 包袋 / 垫板和泛家居摆件的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 场景感基本成立，但 `壶 / 杯 / 罐 / 盅 / 盘` 家族偏密，部分条目更像容器横向平移而不是主体物扩展。
- `support_items` 的主要问题不是不属于茶歇，而是过度依赖 `托 / 套 / 盒 / 盖 / 罩 / 架` 这类服务壳体来制造多样性。
- `visible_small_items` 格式合规，但后半段明显偏向纸品、包装和一次性配套物，读感开始像补给清单。
- `scene_expansion_items` 是当前最大问题：大量条目首先读成“为别的物服务的壳体或承载器”，其次又混入了一批泛家居氛围物，已经偏离“餐桌前台自然单物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池没有出现必须整片清洗的问题，但存在明显的容器横向平移。
- 低优先级平衡项：`饮料分配器`、`方糖盅`、`果酱罐`、`蜂蜜罐`、`茶叶罐`、`饼干罐`、`曲奇铁盒`、`玻璃糖果罐`。
- 这些条目不是立刻必换，而是说明本池已经在用不同内容物 + 相似容器壳来撑密度；后续若要腾位，优先从这组里动。
- 当前可作为锚点保留的方向：`瓷茶壶`、`法压咖啡壶`、`热饮保温壶`、`蛋糕托台`、`玻璃蛋糕罩`、`双层点心架`、`三层蛋糕架`、`面包篮`、`水果盘`、`圆形奶油蛋糕`、`可颂面包`、`马卡龙`。

### support_items

- 服务壳体家族过密，优先替换：`茶包托`、`茶匙托`、`杯盖`、`隔热杯套`、`纸杯套`、`糖包盒`、`茶包盒`、`咖啡滤纸架`、`餐具托`、`刀叉托`、`点心纸托`、`马芬纸杯托`、`茶壶保温套`、`餐盘盖`、`食物罩`。
- 问题根因不是这些词完全不能出现，而是这一池已经过度依赖“托住谁 / 盖住谁 / 包住谁”的服务逻辑。
- 当前更稳的辅助物方向仍应放在真正参与取用和切分动作的器具本体，如 `茶匙`、`甜点叉`、`黄油刀`、`蛋糕刀`、`面包刀`、`果酱勺`、`点心夹`、`方糖夹`、`蛋糕铲`、`冰块夹`、`茶漏`、`滤茶球`。

### visible_small_items

- 整体格式合规，暂不建议整池推倒。
- 低优先级压缩项：`一叠餐巾纸`、`一叠杯垫`、`一束纸吸管`、`一筒搅拌棒`、`一叠点心纸托`、`一叠马芬纸杯`、`一包木质小叉`、`一包纸杯套`、`一卷杯套`、`一包湿巾`。
- 这些条目本身不违规，但它们和第二池、第四池的壳体条目叠加后，会进一步放大“茶歇耗材补给区”的读感。

### scene_expansion_items

- 服务壳体和承载器回流，必须优先替换：`餐巾架`、`桌面纸巾盒`、`纸吸管筒`、`搅拌棒筒`、`纸杯叠放筒`、`外带杯架`、`外带杯托盘`、`玻璃餐盘罩`、`折叠餐盘罩`、`木质托盘`、`杯垫立架`、`茶匙托盘`、`点心分隔盘`、`甜点分格篮`、`曲奇餐盘罩`、`起司餐盘罩`、`蜂蜜棒玻璃筒`、`奶精玻璃筒`。
- 包袋和收纳壳回流，必须优先替换：`保冷餐袋`、`便携餐具包`、`花纹餐巾包`、`桌布收纳包`。
- 第二池工具和后场辅助物回流，必须优先替换：`滤茶器`、`冷饮搅拌器`、`迷你开瓶器`、`甜点切分板`、`面包切片板`、`茶壶隔热垫`、`热饮壶垫`、`防烫桌垫`。
- 泛家居摆件和弱相关氛围物偏多，建议压缩：`香薰蜡烛座`、`餐桌烛台座`、`翻页台历`、`桌面月历`、`留声机`、`桌面收音机`、`音乐盒`、`陶瓷小花盆`、`苔藓玻璃盆`、`鲜花花篮`、`桌面相册`、`明信片相册`、`客用留言本`、`干花花盆`。
- 当前较接近第四池方向、可作为保留候选的仅有少数：`小号冰桶`、`面包保温篮`、`水果浅篮`、`沙漏计时器`、`桌面小钟`。

## 4. 多样性与模板密度

- 第一池最拥挤的家族是 `壶 / 杯 / 罐 / 盅 / 盘`，虽然仍在场景内，但已经出现容器横向平移信号。
- 第二池最拥挤的尾词是 `托 / 套 / 盒 / 盖 / 罩 / 架`，主要问题是“不同服务对象 + 相同壳体”的伪多样性。
- 第三池格式没错，但 `纸 / 包装 / 一次性配套物` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `筒 / 架 / 托 / 盘`、`罩 / 盖 / 垫`、`包 / 袋 / 收纳壳`、`台历 / 收音机 / 花盆 / 相册` 这类泛家居摆件链。

## 5. 第四池专项

- 第四池当前的根因不是几个坏词，而是整池写法站在“茶歇配件和补给物”视角，而不是“餐桌前台自然单物”视角。
- 结果是三类问题同时叠加：服务壳体堆叠、一次性耗材承载器堆叠、泛家居装饰物混入。
- 本类第四池应优先补“普通玩家一眼就能认出的餐桌前台中型物、茶歇服务场景中自然出现但不依赖说明的单物”，而不是继续补罩、盘、托、架、筒、垫、包。
- 这一池不适合微调修补；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留稳定的茶壶、咖啡壶、杯具、蛋糕架、甜点主体物，必要时压缩不同内容物共用同类容器壳的条目。
- 第二池：从“服务壳体池”拉回“辅助器具池”，优先补真正参与取食、搅拌、切分、夹取、过滤的工具本体。
- 第三池：继续保持 `一盒 / 一包 / 一排 / 一盘 / 一碗 / 一叠` 的可见单位格式，但避免纸品和一次性耗材继续膨胀。
- 第四池：按“前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用抽取器、托座、纸品壳体、后场辅助物和“服务动作 + 载体”的伪自然写法。
- 收尾时额外检查同后缀家族，尤其是 `托 / 盘 / 罩 / 架 / 筒 / 包 / 垫` 是否还成片出现。

## 7. 本次修改记录

- 已基于上一轮结果完成第二次重做，修改文件：`Game content extraction/data/blind_boxes.py`。
- 本轮重点不是继续在上一轮结果上补丁，而是处理“旧坏家族被换成新坏家族”的假修复问题。
- `core_items`：
  轻修了第一池里最明显的内容物 + 承载壳写法，把 `起司拼盘`、`巧克力拼盘`、`葡萄果盘`、`焦糖布丁杯`、`帕菲甜点杯` 改成更像独立甜点主体物的 `提拉米苏`、`千层蛋糕`、`巧克力慕斯`、`草莓奶油卷`、`水果奶冻`。
- `support_items`：
  本轮未继续重写第二池，保留上一轮已压下服务壳体后的结构，避免同时把第二池和第四池一起重新打散。
- `visible_small_items`：
  本轮仍未动第三池；它的纸品与一次性耗材偏密问题继续暂留，但不是这次重做的主矛盾。
- `scene_expansion_items`：
  整段撤掉了上一轮那批看起来过关、但读感已经长成另一串弱模板的条目，如 `纸杯蛋糕架`、`马卡龙塔架`、`甜甜圈塔架`、`三明治拼盘`、`水果拼盘`、`坚果拼盘`、`司康拼盘`、`可颂拼盘`、`菜单展示板`、`饮品菜单册`、`甜点菜单册`、`桌面菜单夹`。
  改为更像真实茶歇前台中型物的组合：少量真实餐篮和餐盘，如 `面包保温篮`、`水果浅篮`、`藤编餐篮`、`银质点心盘`、`高脚甜点盘`；少量稳定台面装饰物，如 `玻璃烛台座`、`银质烛台座`、`鲜花花篮`；饮品与甜点设备，如 `咖啡机`、`台式磨豆机`、`台式制冰机`、`巧克力喷泉机`、`华夫饼机`；再辅以少量桌面计时与氛围物，如 `桌面小钟`、`沙漏计时器`、`翻页台历`、`点唱机`、`音乐盒`、`香薰扩香器`。
- 本轮核心目标是把第四池从“拼盘 / 菜单壳 / 设备串”拉回更自然的餐桌前台物，而不是继续用边界友好后缀硬顶过去。

## 8. 修改后复审

- 已完成本类第二轮结构复查。
- `餐桌+茶歇` 四池当前均为 50 条，且池内唯一。
- `scene_expansion_items` 与其他三池无机械精确重复。
- 按本类专项口径复查：
  `scene_expansion_items` 50 条全部通过中型边界检查；
  本类四池均未命中 blocked patterns；
  第四池与其他三池无跨池机械重复。
- 人审结论：
  相比上一轮，这次已经明确去掉了 `拼盘 / 塔架 / 菜单壳` 这一条假修复链；
  当前第四池仍有设备类密度，但整体已经更像真实茶歇前台，而不是“换一批后缀继续过规则”；
  当前版本可以作为 `餐桌+茶歇` 的新基线继续使用。

## 9. 剩余风险与暂留理由

- 本轮已经解决的根因是“上一轮虽然不再是托盘壳和菜单壳，但又变成了另一批内容词 + 拼盘 / 架 / 册的弱模板串联”。
- 仍保留两类中低风险：
  `scene_expansion_items` 里的 `机 / 器` 家族仍有一定密度，只是已经比上一轮的 `拼盘 / 架 / 菜单壳` 更自然；
  `visible_small_items` 后半段仍偏纸品、一次性配套物和耗材单位，本轮暂未处理。
- 这些残余风险本轮暂留的原因是：
  先优先修正读感最差的假修复链条，避免为了继续压设备尾词又引入新的假自然词；
  `餐桌+茶歇` 本身允许一定比例的咖啡、制冰、甜点设备和桌面氛围物，完全压平反而容易失真。
- 如果下一轮继续细修，本类优先级建议是：
  先轻压第四池的 `机 / 器` 密度，再视需要微修第三池后半段的纸品与耗材串联；
  第一池只在确认这两块稳定后，再做低优先级的容器横向平移压缩。
