# 庭院+园艺 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `庭院+园艺` 的四池内容。
- 本轮结论：当前版本的第一池仍在场景内，但已经明显混入较多土料包装壳、育苗壳和园艺基建物；第二池被 `夹 / 带 / 盒 / 提篮 / 收纳篮 / 盘` 的工具壳扩写拉得过散；第四池仍然没有通过，核心问题是它被写成了园艺工作台、备料区和工具收纳区的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的庭院和园艺场景感还在，但 `袋 / 包 / 桶 / 罐 / 托盘 / 罩` 家族已经偏密，部分条目更像基材包装壳、育苗壳和作业设备横向平移。
- `support_items` 的主要问题不是不属于园艺，而是把真正的园艺小工具、绑扎件、分装壳、提篮和收纳壳混在了一池里。
- `visible_small_items` 格式合规，但育苗材料、小型配件、接头和绑夹已经开始放大“园艺耗材补给区”的读感。
- `scene_expansion_items` 是当前最大问题：本、手册、桶、篮、盒、托盘、工具包和备料器同时回流，已经偏离“庭院前台自然单物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池没有出现必须整片清洗的问题，但基材包装壳和育苗作业壳已经过密。
- 低优先级平衡项：`育苗托盘`、`育苗盆`、`育苗加热垫`、`番茄支撑笼`、`果实采摘篮`、`悬挂花篮`、`营养土袋`、`陶粒包`、`珍珠岩包`、`有机肥桶`、`缓释肥罐`、`覆盖木片包`、`堆肥桶`、`雨水收集桶`、`育苗温室罩`。
- 这些条目不是立刻必换，而是说明本池已经在用“作业壳 / 基材壳 / 育苗壳”维持密度；后续若要腾位，可优先从这组下手。
- 当前可作为锚点保留的方向：`陶土花盆`、`长条花槽`、`方形种植箱`、`园艺喷壶`、`金属浇水壶`、`花园软管`、`水管喷枪`、`多肉盆栽`、`蜂鸟喂食器`、`开花盆栽`、`蔬菜种植箱`、`窗台花箱`、`园艺铲`、`移栽铲`、`修枝剪`、`花剪`、`手耙`、`松土叉`、`小锄头`、`剪枝锯`、`杂草拔除器`、`落叶耙`、`庭院扫帚`、`防刺手套`、`园艺围裙`、`遮阳帽`、`陶瓷鸟浴盆`、`园艺跪凳`。

### support_items

- 工具壳、分装壳和提篮类条目过密，优先替换：`花盆托盘`、`植物绑带卷`、`麻绳卷`、`园艺铁丝卷`、`肥料分装盒`、`种球分装盒`、`育苗工具盒`、`工具腰包`、`花盆搬运带`、`土袋封口夹`、`育苗杯`、`育苗保温盖`、`软管挂带`、`嫁接胶带卷`、`嫁接夹盒`、`种子收纳盒`、`苗盆提篮`、`花盆滚轮盘`、`苗盘分隔片`、`园艺收纳篮`。
- 植物支撑和绑扎家族存在扩写风险，优先复查：`植物支撑杆`、`植物支撑环`、`番茄支撑笼`、`花枝固定夹`、`枝条软夹`、`枝条绑扎夹`。
- 问题根因不是这些词完全不能出现，而是这一池已经同时承担了“真正的小工具池”和“园艺备件与收纳池”两种职责。
- 当前更稳的辅助物方向仍应放在真正参与播种、移栽、拔草、修枝、压孔、混土和清洁动作的工具本体，如 `肥料量勺`、`土壤量杯`、`播种器`、`种子撒播器`、`移栽叉`、`根钩`、`拔草叉`、`花盆清洁刷`、`叶面清洁刷`、`护膝垫`、`防刺护臂`、`育苗压孔器`、`花土搅拌勺`、`清土刷`、`土块压实器`、`种球种植器`、`喷壶清洁刷`、`工具清洁刷`、`园艺鞋刷`、`育苗喷壶`、`喷壶补水漏斗`、`修枝剪磨刀器`。

### visible_small_items

- 整体格式合规，暂不建议整池推倒。
- 低优先级压缩项：`一套育苗杯`、`一排育苗盆`、`一盒肥料棒`、`一包缓释肥`、`一袋陶粒`、`一盒嫁接夹`、`一盒花枝夹`、`一卷植物绑带`、`一包珍珠岩`、`一盒滴灌头`、`一包蛭石`、`一卷嫁接胶带`、`一盒软扎带`、`一排喷雾小瓶`、`一盒育苗海绵`、`一套植物支撑环`、`一袋营养土小包`、`一排扦插管`、`一包树皮块`、`一盒枝条软夹`、`一卷软管修补带`、`一盒花盆绑夹`、`一包堆肥小袋`、`一盒土壤量勺`、`一盘育苗泥块`、`一盒迷你喷壶`、`一包植物绑扣`、`一盒喷枪喷头`、`一排肥料小瓶`、`一盒花盆脚垫`、`一袋椰糠砖`、`一筒滴灌软管扣`、`一盒陶土盆托`、`一盒修枝刀片`。
- 这些条目本身不违规，但它们和前两池、第四池的备料壳条目叠加后，会进一步放大“园艺耗材备料区”的读感。

### scene_expansion_items

- 本册、手册和记录物回流，必须优先替换：`园艺记录本`、`播种手册`、`育苗手册`。
- 桶、篮、盒、提袋和工具包回流，必须优先替换：`折叠水桶`、`修枝废料桶`、`收获篮`、`球根篮`、`种植筛篮`、`工具篮`、`种球收纳盒`、`灌溉喷头盒`、`收果提篮`、`工具提包`、`嫁接工具包`、`种球工具包`、`育苗保温盒`、`扦插盒`、`昆虫旅馆盒`、`收枝提袋`、`落叶收纳袋`。
- 作业器和备料器过密，必须优先替换：`堆肥翻拌器`、`泥土筛刷`、`枝叶护理刷`、`落叶夹器`、`手压喷雾筒`、`花盆搬运器`、`肥料撒播机`、`手摇撒播机`、`肥料搅拌器`、`土壤筛分器`。
- 托盘、分装壳和作业铺垫回流，必须优先替换：`园艺跪垫`、`雨靴托盘`、`种植托盘`、`移栽托盘`、`收获托盘`、`种苗分装盒`、`育苗暖垫`、`石质喂鸟盘`。
- 后场清理和地面整理物偏多，必须整体压缩：`清洗盆`、`靴子刷`、`泥土刷`、`鞋底刮泥板`、`户外脚垫`。
- 当前较接近第四池方向、可作为保留候选的仅有少数：`庭院小钟`、`沙漏计时器`、`翻页台历`、`桌面月历`、`喂鸟器`。

## 4. 多样性与模板密度

- 第一池最拥挤的家族是 `袋 / 包 / 桶 / 罐 / 托盘 / 罩`，说明主体池已经出现基材壳、育苗壳和作业容器横向平移。
- 第二池最拥挤的尾词是 `夹 / 带 / 盒 / 杯 / 提篮 / 收纳篮 / 盘`，主要问题是“不同园艺步骤 + 相同工具壳”的伪多样性。
- 第三池格式没错，但 `育苗材料 / 绑扎配件 / 滴灌接头 / 喷壶配件` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `本 / 手册 / 记录本`、`桶 / 篮 / 盒 / 提袋 / 工具包`、`托盘 / 跪垫 / 分装盒 / 暖垫`、`筛刷 / 搅拌器 / 撒播机 / 作业器` 这类工作区壳体链。

## 5. 第四池专项

- 第四池当前的根因不是单个坏词，而是整池写法站在“园艺工作台和备料流程”视角，而不是“庭院前台场景”视角。
- 结果是四类问题同时叠加：记录本和手册混入、工具篮和提包堆叠、托盘和分装壳堆叠、作业器和整理器混入。
- 本类第四池应优先补“普通玩家一眼就能认出的庭院地面、花盆区、浇水区或种植角前台中型物、自然外露的稳定场景补充物”，而不是继续补本、手册、桶、篮、盒、托盘和工作器。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留稳定的花盆、种植箱、浇水物、修枝工具和少量高频基材承载物，必要时压缩土料壳、肥料壳和育苗壳的横向平移。
- 第二池：从“园艺备件和收纳壳混池”拉回“真正辅助工具池”，优先保留能直接参与播种、移栽、修枝、混土、清洁和绑扎动作的工具本体。
- 第三池：继续保持 `一包 / 一盒 / 一套 / 一排 / 一卷 / 一筒` 的可见单位格式，但避免育苗材料、绑扎件、滴灌接头和喷壶配件继续膨胀。
- 第四池：按“庭院前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用记录本、手册、工具包、桶篮、托盘和作业器作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `夹 / 架 / 盒 / 包 / 筒 / 篮 / 托盘 / 手册 / 记录本` 是否还成片出现。

## 7. 本次修改记录

- 本轮实际修改了 `blind_boxes.py` 中 `庭院+园艺` 的 `support_items` 和 `scene_expansion_items`，`core_items` 与 `visible_small_items` 暂不大动。
- 第二池的修改方向是把“分装盒、提篮、收纳篮、工具包、托盘、绑带卷混池”拉回“真实播种、移栽、除草、嫁接、清洁、滴灌维护工具池”。
- 具体清掉了 `花盆托盘 / 植物绑带卷 / 麻绳卷 / 园艺铁丝卷 / 分装盒 / 工具盒 / 工具腰包 / 搬运带 / 育苗杯 / 育苗保温盖 / 软管挂带 / 嫁接胶带卷 / 嫁接夹盒 / 种子收纳盒 / 苗盆提篮 / 花盆滚轮盘 / 苗盘分隔片 / 园艺收纳篮` 这类弱本体工具壳。
- 回填时改用 `除草刀 / 除草钩 / 扦插锥 / 移苗镊子 / 授粉刷 / 花剪磨刀石 / 滴灌打孔器 / 滴灌疏通针 / 水管接头钳 / 采果剪 / 采种镊子 / 根系梳` 等更直接参与动作的工具本体。
- 第四池本轮按高风险家族整段清理，移除了 `记录本 / 手册 / 桶 / 篮 / 盒 / 提袋 / 工具包 / 托盘 / 跪垫 / 筛分器 / 撒播机 / 搅拌器` 这类工作台和备料区写法。
- 第四池回填改成“时间物 + 小型环境设备 + 鸟浴盆与装饰水盆 + 花盆盆栽 + 户外毯面席面 + 图册画册”结构，尽量让它更像庭院前台自然可见的中型场景补充物，而不是园艺备料区。
- 后续复看时又发现 `xx册` 仍然偏密，因此继续按“每种后缀最多保留两个”的口径微调，把 `花卉图册 / 鸟类图册 / 昆虫图册 / 自然图册 / 花鸟图册 / 旅行画册` 替换为 `多肉组合 / 花苗组合 / 喂鸟盘 / 叶纹席面 / 鹅卵石席面 / 陶盆组合`，把第四池从“读物串联”再拉回一点到庭院前台盆栽物、席面物和展示物的混合结构。

## 8. 修改后复审

- 当前四池数量均为 50，且池内唯一。
- 第四池与其余三池无重叠。
- 第四池本轮本地复查未命中禁词，且全部通过 `_is_medium_scene_fragment_item` 的中型场景边界校验。
- 这说明本轮已经先把“结构性错误”修掉了，尤其是第四池不再被 `本 / 手册 / 桶 / 篮 / 盒 / 提袋 / 托盘 / 作业器` 这类工作区家族主导。
- 从读感上看，第二池已经明显回到园艺工具池；第四池也从“园艺工作台和备料流程区”转回“庭院前台场景补充物”。
- 进一步按后缀统计复查后，第四池里的 `图册 / 画册 / 摄影册 / 写真册` 已分别压到 `2 / 2 / 1 / 1`，满足“每种后缀最多保留两个”的补充要求。
- 本节结论仅针对 `庭院+园艺` 的定点复查，不代表全仓所有类别已全量通过。

## 9. 剩余风险与暂留理由

- 第一池里仍保留了少量基材包装壳和育苗作业壳，例如 `育苗托盘`、`育苗盆`、`营养土袋`、`陶粒包`、`珍珠岩包`、`有机肥桶`、`缓释肥罐`、`堆肥桶`、`雨水收集桶`；这轮暂不动，是为了先稳住主体池，不把问题从第四池转移到第一池。
- 第三池仍保留了一些育苗材料和滴灌配件单位，例如 `一套育苗杯`、`一排育苗盆`、`一盒滴灌头`、`一卷嫁接胶带`、`一筒滴灌软管扣`；它们当前格式合规，但后续如果继续做第三池精修，可以再轻压一次。
- 第四池虽然已经脱离“备料壳链”，但现在仍有 `钟 / 历 / 机 / 盆 / 卷 / 册` 这些自然家族；后续若继续微调，重点应放在压模板密度，而不是重新引入 `盒 / 包 / 篮 / 托盘 / 手册`。
- 这轮已经把 `xx册` 压到较合理密度；后续若继续微调，优先关注 `卷 / 花盆 / 盆栽` 这类相邻家族，而不是重新把图册画册补回来。
- 当前版本已经可以作为后续同类修改的合格基线，下一步若继续优化，建议优先做“第一池基材壳轻压”和“第三池育苗配件轻压”，而不是再大改第二池和第四池。
