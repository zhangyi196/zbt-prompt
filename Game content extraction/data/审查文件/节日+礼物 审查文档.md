- 审查范围：仅审查该类别 `core_items` / `support_items` / `visible_small_items` / `scene_expansion_items` 四池内容

## 总体结论

状态：`已修改并复审`

这一类原先的主要问题，不是数量不够，而是太依赖包装壳体、服务工具壳体和板册链：

- `core_items` 过度集中在 `礼盒 / 礼袋 / 礼篮 / 罐 / 桶 / 卷` 的礼品包装变体，很多条目只是换了容器外形。
- `support_items` 明显滑向 `夹 / 托 / 模 / 卷 / 纸托 / 底座 / 护套` 的包装附件链和桌面服务链。
- `visible_small_items` 虽然做了可见单位升级，但甜食包装、纸托、糖果条和小装饰物仍容易堆成“不同内容物 + 相同单位壳体”。
- `scene_expansion_items` 之前整池都在用 `架 / 筒 / 盒 / 托盘 / 收纳篮 / 底座 / 板 / 册` 充数，更像包装工位和后场器具区，不像节日桌面或礼物展示区的自然中型单物。

## 本次修改方向

- `core_items`：保留礼品包装线，但降低同一包装壳体密度，补入礼物本体、装饰摆件、节庆桌面物和可赠送物。
- `support_items`：保留必要包装与摆盘工具，但不再沿着封口夹、托座、压模盒、护套件继续横向扩写。
- `visible_small_items`：让小物多样性来自糖果、烘焙纸品、装饰挂件、吊牌、蜡封颗粒、祝福卡片等真实差异，而不是全靠同类零食包装。
- `scene_expansion_items`：改成节日桌面 / 礼物展示区前台可见的中型单物，例如相框、相册、烛台、花瓶、提篮、靠垫、计时钟、展示台，而不是一串工具架、收纳壳和板册。

## 本次主要压掉的伪多样性

- `不同礼品内容 + 相同礼盒礼袋壳体`
- `不同桌面用途 + 相同托盘 / 架 / 筒 / 底座壳体`
- `不同甜点内容 + 相同纸托 / 夹 / 服务工具壳体`

## 复审结论

- 四池均为 50 条，池内唯一。
- 当前版本已明显压低礼盒礼袋礼篮横向平移、包装附件链、板册链和第四池器具壳体链。
- 第四池已经回到“节日桌面 / 礼物展示 / 祝福留言区”的前台自然中型物，不再像包装工位。
- 该类别允许保留一定包装属性，但多样性必须更多来自礼物本体、节庆摆件、展示物和桌面承载物，而不是继续换壳体。

## 保留风险

- `节日+礼物` 天然会包含包装和甜点，后续补写时仍要持续压住：
  - `礼盒 / 礼袋 / 礼篮 / 罐桶` 的壳体横移
  - `夹 / 架 / 筒 / 底座 / 托盘 / 收纳盒 / 板 / 册` 的第四池工具壳链
  - `不同糖果点心 + 相同小包装单位` 的可见小物伪多样性
