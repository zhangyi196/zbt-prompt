# 节日+礼物 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `节日+礼物` 的四池内容。
- 本轮结论：当前版本的第一池仍能读出“节庆礼物桌面区”的主体场景，但已经混入较多礼盒壳、礼袋壳、礼篮壳和点心容器壳；第二池被 `卷 / 轴 / 模具 / 内衬 / 衬垫 / 托 / 册` 的包装工作台补充壳拉散；第四池明显没有通过，已经被写成展示、留言、包装收纳和甜点摆盘系统的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的礼盒、礼篮、花环、烛台和节庆装饰主场景还在，但“礼赠包装目录”和“甜点容器陈列区”的读感已经开始抬头。
- `support_items` 的主要问题不是与礼物场景无关，而是把真实包装工具、包装耗材、甜点承载壳和装饰挂件辅具混在了一池里。
- `visible_small_items` 格式合规，但糖果、包装耗材、装饰耗材和派对小件成串后，已经很像“节庆补给包”。
- `scene_expansion_items` 是当前最大问题：板、册、本、架、台、盘、篮、盒、垫、筒集中回流，明显偏离“前台自然中型物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池最大问题不是整体失场，而是 `礼盒 / 礼袋 / 礼篮 / 礼筒 / 礼套 / 盘 / 桶 / 罐` 壳层已经开始挤占主体位。
- 低优先级平衡项：`方形礼盒`、`圆筒礼盒`、`抽屉礼盒`、`手提礼品袋`、`束口礼品袋`、`布面礼品袋`、`水果礼篮`、`零食礼篮`、`咖啡礼篮`、`沐浴礼篮`、`热可可礼筒`、`茶包礼盒`、`马克杯礼包`、`护手霜礼套`、`手工皂礼套`、`巧克力礼盘`、`黄油饼干桶`、`糖果礼瓶`、`软糖礼筒`、`棉花糖桶`、`曲奇礼桶`、`纸杯蛋糕塔`、`坚果拼盘`、`迷你派盘`、`可可粉罐`、`咖啡豆罐`、`糖果罐`、`曲奇罐`、`倒数日历盒`。
- 这些条目不一定要立刻整片清洗，但它们说明本池已经开始用“包装壳、礼赠壳和点心承载壳”维持密度；后续若要腾位，可优先从这组压缩。
- 当前可作为锚点保留的方向：`香薰蜡烛杯`、`香氛扩香瓶`、`姜饼屋模型`、`陶瓷糖果碗`、`桌面装饰树`、`桌面花环`、`节日桌旗`、`棒棒糖花束`、`干花束`、`纸质拉花球`、`彩色绒球串`、`丝带蝴蝶结`、`节庆挂饰串`、`节日相框`、`装饰烛台`、`星形摆件`、`毛绒抱枕`、`节庆音乐盒`、`陶瓷花瓶`、`玻璃糖罐`、`礼帽摆件`。

### support_items

- 本池被包装工作台壳、装饰辅具壳和甜点承载壳拉得过散，优先替换：`丝带分配器`、`缎带卷轴`、`麻绳卷轴`、`蝴蝶结模具`、`纸花模具`、`礼盒内衬`、`瓦楞内衬`、`彩色纸丝包`、`棉纸衬垫`、`蜂窝纸垫`、`礼篮衬布`、`甜点纸托`、`蛋糕纸杯`、`巧克力托盘`、`饼干隔纸`、`贺卡夹`、`桌旗夹`、`花束绑带`、`干花固定夹`、`花环挂钩`、`装饰球架`、`礼物压块`、`杯身护套`、`香薰杯垫`、`手工皂托`、`吊牌分配器`、`祝福贴纸册`。
- 本池里也存在“同流程不同对象 + 相同壳体”的伪多样性，优先复查：`双面胶带卷`、`纸胶带卷`、`礼卡打孔器`、`礼签打孔钳`、`封带压轮`。
- 问题根因不是这些词完全不能出现，而是这一池已经同时承担了“真实包装工具池”“包装耗材池”和“甜点摆台辅具池”三种职责。
- 当前更稳的辅助物方向仍应放在真正参与裁切、折叠、打孔、粘贴、封蜡、分食和修剪动作的工具本体，如 `包装剪刀`、`安全裁纸刀`、`圆角裁切器`、`手持打孔器`、`热熔胶枪`、`压痕板`、`折纸骨刀`、`包装刮板`、`蜡封加热勺`、`封蜡章`、`礼品测尺`、`糖果夹`、`点心夹`、`蛋糕铲`、`甜点叉`、`坚果勺`、`糖果勺`、`迷你花剪`。

### visible_small_items

- 低优先级压缩项：`一盒迷你巧克力`、`一包硬糖`、`一套礼物蝴蝶结`、`一排巧克力棒`、`一叠包装纸`、`一盒软糖`、`一包棉花糖`、`一套纸花装饰`、`一排薄荷糖筒`、`一叠彩色棉纸`、`一盒太妃糖`、`一包曲奇饼`、`一套绒球装饰`、`一排条纹棒糖`、`一叠餐巾纸`、`一套纸杯蛋糕托`、`一排纸吸管`、`一叠蛋糕纸杯`、`一筒彩色吸管`、`一筒纸花枝`、`一卷纸胶带`、`一盒礼品吊牌`、`一卷双面胶带`、`一组缎带卷`、`一组丝带卷`、`一包彩色纸丝`、`一组麻绳卷`、`一组装饰球`、`一组松果装饰`、`一包派对蜡烛`、`一盒祝福卡片`。
- 这些条目本身未必违规，但它们和前两池、第四池的包装、装饰和摆盘壳叠加后，会继续放大“节庆补给与包装工位”的读感。

### scene_expansion_items

- 板、挂牌和留言本家族回流，必须优先替换：`祝福留言板`、`礼物标签板`、`倒数挂牌`、`宾客留言本`、`祝福留言本`。
- 展示台、展示架、立架和托架家族过密，必须优先替换：`礼盒展示台`、`照片展示架`、`礼物展示台`、`贺卡托架`、`纸杯蛋糕架`、`贺卡立架`、`照片晾挂架`、`贺卡展示架`、`甜点展示架`、`礼卡展示架`。
- 相册、画册和册类回流，必须优先替换：`纪念相册`、`贺卡册`、`节日画册`、`礼品画册`、`相框展示册`。
- 盘、篮、盒、垫、筒和工具包家族过密，必须优先替换：`点心展示盘`、`糖果展示盘`、`曲奇展示盘`、`节庆托盘`、`礼物提篮`、`点心提篮`、`水果浅篮`、`纸杯提篮`、`丝带收纳篮`、`抽纸盒`、`湿巾盒`、`餐巾盒`、`蜡封收纳盒`、`糖果铁盒`、`曲奇铁盒`、`花纹餐垫`、`节庆餐垫`、`木叉筒`、`纸吸管筒`、`干花插筒`、`缎带收纳筒`、`蜡封工具包`。
- 装饰插盆和软装靠垫混入，必须整体压缩：`桌面靠垫`、`丝绒靠垫`、`纸花插盆`、`松果陶盆`。
- 当前较接近第四池方向、可作为保留候选的条目相对有限：`节庆倒数日历`、`节日转盘`、`小号冰桶`、`热饮保温桶`。

## 4. 多样性与模板密度

- 第一池当前最明显的压力不是“节庆主体物太多”，而是 `礼盒 / 礼袋 / 礼篮 / 礼筒 / 礼套 / 盘 / 桶 / 罐` 与点心承载壳一起抬高了场景杂音。
- 第二池最拥挤的尾词是 `卷 / 轴 / 模具 / 内衬 / 衬垫 / 托 / 册`，主要问题是“不同包装步骤 + 相同工作台壳”的伪多样性。
- 第三池格式没错，但 `糖果小件 / 包装耗材 / 装饰耗材 / 派对小物` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `留言板 / 标签板 / 挂牌`、`展示台 / 展示架 / 立架 / 托架`、`相册 / 画册 / 留言本 / 贺卡册`、`提篮 / 收纳篮 / 盒 / 筒 / 餐垫 / 工具包` 这几类节庆服务壳体链。

## 5. 第四池专项

- 第四池当前的根因不是个别坏词，而是整池写法站在“展示、留言、包装和摆盘服务系统”的视角，而不是“节庆前台场景补充物”的视角。
- 结果是四类问题同时叠加：展示台架成串、相册画册和留言本成串、提篮筒盒成串、餐垫和工具包等服务壳体混入。
- 本类第四池应优先补“普通玩家一眼就能认出的礼物桌、节庆装饰角、甜点摆放区或庆祝前台”里的自然中型单物，而不是继续补标签板、展示架、提篮、收纳筒和留言本。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留节庆装饰物、花束、烛台、音乐盒和少量稳定礼赠主物，必要时压缩礼盒壳、礼袋壳、礼篮壳和点心承载壳的横向平移。
- 第二池：从“包装工作台壳与甜点摆台壳混池”拉回“真正辅助工具池”，优先保留裁切、折叠、打孔、粘贴、封蜡、分食和修剪动作里会直接上手的物件本体。
- 第三池：继续保持 `一盒 / 一包 / 一套 / 一排 / 一叠 / 一卷 / 一组 / 一束 / 一盘 / 一筒` 的可见单位格式，但避免糖果小件、包装耗材和装饰耗材继续膨胀。
- 第四池：按“节庆前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用展示架、留言板、相册、提篮、收纳筒和工具包作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `板 / 架 / 台 / 册 / 本 / 盘 / 篮 / 盒 / 筒` 是否还成片出现。

## 7. 本次修改记录

- 本轮实际修改了 `blind_boxes.py` 中 `节日+礼物` 的 `support_items` 和 `scene_expansion_items`，`core_items` 与 `visible_small_items` 暂不大动。
- 第二池的修改方向是把“包装工作台壳、装饰辅具壳和甜点摆台壳混池”拉回“真实裁切、折叠、打孔、粘贴、封蜡、分食和修剪工具池”。

- 本轮明确不再沿用 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 画册图册` 这类容易跨类别复用的旧生成习惯，改为直接补 `花环摆座 / 烛台摆座 / 姜饼屋座 / 糖果塔座 / 礼物堆组合 / 雪花挂饰阵列 / 松果阵列 / 礼物包裹` 这类只属于节庆礼物语境的中型物。

## 8. 修改后复审

- 当前四池数量均为 50，且池内唯一。
- 第四池与其余三池无重叠。
- 第四池本轮本地复查未命中禁词，且全部通过 `_is_medium_scene_fragment_item` 的中型场景边界校验。
- 第四池中的 `地毯卷 / 毯面 / 席面 / 花盆 / 图册 / 画册` 当前实值分别为 `2 / 2 / 2 / 0 / 0 / 0`，已经不再出现同后缀连续堆叠压读感的问题。
- 这说明本轮已经先把“结构性错误”修掉了，尤其是第四池不再被 `板 / 架 / 台 / 册 / 本 / 盘 / 篮 / 盒 / 筒` 或 `地毯卷 / 花盆 / 图册 / 画册` 这类节庆服务和旧模板壳主导。
- 从读感上看，第二池已经明显回到包装与节庆辅助工具池；第四池也从“展示、留言、包装和摆盘服务系统”转回“节庆前台场景补充物”。
- 本节结论仅针对 `节日+礼物` 的定点复查，不代表全仓所有类别已全量通过。

## 9. 剩余风险与暂留理由

- 第一池里仍保留了少量礼赠壳和点心承载壳，例如 `方形礼盒`、`圆筒礼盒`、`抽屉礼盒`、`手提礼品袋`、`束口礼品袋`、`布面礼品袋`、`水果礼篮`、`零食礼篮`、`咖啡礼篮`、`沐浴礼篮`、`热可可礼筒`、`茶包礼盒`、`巧克力礼盘`、`黄油饼干桶`、`糖果礼瓶`、`软糖礼筒`、`棉花糖桶`、`曲奇礼桶`、`纸杯蛋糕塔`、`坚果拼盘`、`迷你派盘`；这轮暂不动，是为了先稳住主体池，不把问题从第四池转移到第一池。
- 第三池仍保留了较多糖果小件、包装耗材和装饰耗材单位，例如 `一盒迷你巧克力`、`一包硬糖`、`一套礼物蝴蝶结`、`一叠包装纸`、`一叠彩色棉纸`、`一卷纸胶带`、`一盒礼品吊牌`、`一卷双面胶带`、`一组缎带卷`、`一组丝带卷`、`一组麻绳卷`、`一盒祝福卡片`；它们当前格式合规，但后续如果继续做第三池精修，可以再轻压一次。
- 第四池虽然已经脱离“展示和包装壳链”，也清掉了 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 画册图册` 这条跨类别复用链，但现在仍有 `钟 / 历 / 机 / 卷 / 面 / 座 / 组合 / 阵列 / 分区` 这些自然家族；后续若继续微调，重点应放在继续压模板密度，而不是重新引入 `板 / 架 / 台 / 册 / 本 / 盘 / 篮 / 盒 / 筒`。
- 当前版本已经可以作为后续同类修改的合格基线，下一步若继续优化，建议优先做“第一池礼赠点心壳轻压”和“第三池糖果包装耗材轻压”，而不是再大改第二池和第四池。
