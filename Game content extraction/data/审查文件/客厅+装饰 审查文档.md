# 客厅+装饰 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `客厅+装饰` 的四池内容。
- 本轮结论：当前版本的第一池仍在场景内，但已经明显被花瓶、烛台、摆件、收纳筒和陈列壳体横向平移占满；第二池则被底托、护套、擦布、止滑件和防尘件接管；第四池仍然没有通过，核心问题是它被写成了篮筐 / 托盘 / 礼盒 / 香氛壳 / 摆件外设的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的客厅陈设感还在，但 `花瓶 / 花器 / 烛台 / 摆件 / 盒筒罐` 家族已经偏密，部分条目更像陈列壳和平行摆件横向扩写。
- `support_items` 的主要问题不是不属于客厅，而是过度依赖 `底托 / 护套 / 擦布 / 收纳盘 / 防尘罩 / 止滑片` 这类护理和保护壳来制造多样性。
- `visible_small_items` 格式合规，但已经明显混入太多香氛耗材、花艺耗材、装饰补件和展示配件，读感开始像装饰店补货清单。
- `scene_expansion_items` 是当前最大问题：纸品壳、篮筐、托盘、相册、礼盒、香片盒、装饰小花篮和氛围摆件同时回流，已经偏离“客厅前台自然单物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池没有出现必须整片清洗的问题，但陈列摆件和平行容器壳已经过密。
- 低优先级平衡项：`茶几托盘`、`陶瓷果盘`、`装饰碗`、`陶瓷花瓶`、`高颈花瓶`、`宽口花器`、`陶瓷小雕像`、`木雕摆件`、`石膏半身像`、`几何雕塑`、`动物陶偶`、`桌面音乐盒`、`陶瓷储物罐`、`木质首饰匣`、`编织收纳筒`、`玻璃花罩`、`雪花球摆件`、`玻璃糖果罐`、`浮雕摆盘`、`珠串摆饰`、`装饰棋盘`、`铜质铃铛`。
- 这些条目不是立刻必换，而是说明本池已经在用“同类摆件不同造型”“不同内容物共用容器壳”维持密度；后续若要腾位，可优先从这组下手。
- 当前可作为锚点保留的方向：`方形抱枕`、`圆筒靠枕`、`针织盖毯`、`装饰桌旗`、`盆栽绿植`、`多肉盆栽`、`桌面相框`、`装饰画框`、`咖啡桌书`、`摄影画册`、`桌面时钟`、`复古座钟`、`金属地球仪`、`香薰瓶`、`藤条扩香瓶`、`香薰蜡烛`。

### support_items

- 护理壳、保护壳和底托类条目过密，优先替换：`抱枕套`、`靠枕护套`、`毯子束带`、`桌旗压条`、`托盘衬布`、`杯垫座`、`花瓶底托`、`盆栽接水盘`、`干花绑带`、`香薰漏斗`、`相框撑脚`、`画框护角套`、`书挡止滑片`、`画册护套`、`杂志篮布套`、`桌面收纳盘`、`遥控器托`、`蜡烛防尘罩`、`花盆垫片`、`装饰碗底托`、`香薰炉底托`、`玻璃摆件底座`、`摆件底托`、`毛毯收纳带`、`室内拖鞋盘`。
- 清洁维护物偏多，优先压缩：`眼镜擦布`、`软毛除尘刷`、`除尘掸`、`清洁喷瓶`、`粘毛滚筒`、`首饰擦布`、`留声机擦布`、`唱片清洁刷`。
- 问题根因不是这些词完全不能出现，而是这一池已经过度依赖“摆件护理和保护配件”来撑多样性。
- 当前更稳的辅助物方向仍应放在真正参与客厅待客、花艺整理和香氛补充动作的工具本体，如 `杯垫`、`点心夹`、`糖果夹`、`果叉`、`坚果勺`、`小茶勺`、`浇水壶`、`喷雾瓶`、`花枝剪`、`花泥砖`、`花艺胶带`、`扩香藤条`、`烛芯剪`、`灭烛罩`。

### visible_small_items

- 整体格式合规，暂不建议整池推倒。
- 低优先级压缩项：`一盒香薰蜡块`、`一盒装饰贝壳`、`一包干花香片`、`一包香片`、`一筒扩香藤条`、`一瓶香薰补充液`、`一盒玻璃烛石`、`一盒茶灯烛杯`、`一盒装饰木珠`、`一盒迷你花瓶`、`一卷花艺胶带`、`一卷装饰丝带`、`一套书挡`、`一套抱枕套`、`一套花瓶护垫`、`一套藤条补充芯`、`一排香薰小瓶`、`一排蜡烛杯`、`一排迷你花瓶`、`一排装饰木珠`、`一包首饰擦布`、`一卷擦拭布`、`一包粘毛纸`、`一套相框撑脚`、`一筒装饰羽毛`、`一袋香薰沙`。
- 这些条目本身不违规，但它们和前两池、第四池的壳体条目叠加后，会进一步放大“装饰补货区”和“礼品陈列区”的读感。

### scene_expansion_items

- 纸品壳、篮筐和收纳壳回流，必须优先替换：`抽纸盒`、`湿巾包`、`棉柔巾包`、`杂志篮`、`毛毯篮`、`靠枕篮`、`室内拖鞋篮`、`围巾浅篮`、`毛毯布包`、`抱枕布包`、`客用纸巾盒`。
- 托盘、盒筒和礼盒家族回流，必须优先替换：`木质托盘`、`玻璃托盘`、`陶瓷托盘`、`装饰珠串盘`、`木质香片盒`、`蜡烛礼盒`、`干花筒`、`扩香藤条筒`、`香氛礼盒`、`点心盘`。
- 泛装饰摆件和氛围物过密，必须整体压缩：`留声机`、`音乐盒`、`桌面收音机`、`白噪音机`、`迷你音响机`、`香薰蜡烛座`、`玻璃烛台座`、`香薰石浅盘`、`陶瓷小花盆`、`苔藓玻璃盆`、`干花花盆`、`鲜花花篮`、`桌面相册`、`明信片相册`、`客用留言本`、`装饰小花篮`、`玻璃罩钟`。
- 设备和家居电器有一定场景合理性，但仍需压缩成少量补充项，不应再整段扩写：`空气净化器`、`除湿机`、`香氛雾化器`、`蜡烛暖器`。
- 当前较接近第四池方向、可作为保留候选的仅有少数：`桌面小钟`、`沙漏计时器`、`翻页台历`、`桌面月历`、`空气净化器`、`除湿机`。

## 4. 多样性与模板密度

- 第一池最拥挤的家族是 `花瓶 / 花器 / 烛台 / 摆件 / 罐 / 匣 / 筒`，说明主体池已经出现摆件造型平移和容器壳平移。
- 第二池最拥挤的尾词是 `底托 / 护套 / 擦布 / 布套 / 收纳盘 / 防尘罩`，主要问题是“不同陈列对象 + 相同维护配件”的伪多样性。
- 第三池格式没错，但 `香薰补件 / 花艺耗材 / 装饰补件 / 展示配件` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `篮 / 包 / 纸巾盒`、`托盘 / 盒 / 筒 / 礼盒`、`花盆 / 烛台 / 相册 / 摆件外设`、`香氛壳 / 蜡烛壳 / 展示壳` 这类陈列壳体链。

## 5. 第四池专项

- 第四池当前的根因不是单个坏词，而是整池写法站在“客厅陈列维护和礼品装饰”视角，而不是“客厅前台场景”视角。
- 结果是四类问题同时叠加：纸品壳和篮筐堆叠、托盘和礼盒堆叠、香氛壳和蜡烛壳堆叠、泛装饰摆件和外设混入。
- 本类第四池应优先补“普通玩家一眼就能认出的客厅茶几、柜面或展示角前台中型物、自然外露的稳定场景补充物”，而不是继续补托盘、香片盒、礼盒、纸巾盒和陈列壳。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留稳定的软装物、相框、书本、绿植、时钟和少量高频香氛主体物，必要时压缩花器、摆件和陈列壳的横向平移。
- 第二池：从“陈列护理配件池”拉回“辅助工具池”，优先补真正参与待客、花艺、香氛和日常整理动作的工具本体。
- 第三池：继续保持 `一盒 / 一包 / 一筒 / 一瓶 / 一排 / 一套` 的可见单位格式，但避免香氛补件、装饰补件和展示配件继续膨胀。
- 第四池：按“客厅前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用纸巾盒、托盘、礼盒、陈列壳和泛摆件外设作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `盘 / 托 / 盒 / 筒 / 篮 / 包 / 礼盒` 是否还成片出现。

## 7. 本次修改记录

- 已按本单完成第二轮落库修改，修改文件：`Game content extraction/data/blind_boxes.py`。
- `core_items`：
  这轮不再把第一池完全放着不动，而是轻压几处最明显的托盘、盒筒和陈列壳，把 `茶几托盘`、`陶瓷果盘`、`桌面音乐盒`、`陶瓷储物罐`、`编织收纳筒`、`唱片收纳盒` 调整为更像真实客厅主体物的 `装饰挂镜`、`玻璃醒酒壶`、`黑胶唱机`、`留声机`、`香薰炉`、`编织脚凳` 等，把上一轮仍偏“同类摆件横向平移”和“容器壳换内容物”的读感再往下压一层。
- `support_items`：
  本轮未继续改第二池，沿用上一轮已经完成的工具本体化结果，不再在这一池追加新的护理壳替换，避免把本轮重点打散。
- `visible_small_items`：
  本轮仍未改第三池；香氛补件、花艺耗材和展示配件偏密的问题继续记账，但不是这次“假修复重做”的主战场。
- `scene_expansion_items`：
  本轮把第四池按“二次重做”处理，针对上一轮新长出的 `机 / 器 / 盘 / 盆 / 册` 混合矩阵再次拆解，不再继续在 `点心拼盘 / 水果拼盘 / 坚果拼盘 / 奶酪拼盘 / 巧克力拼盘` 这类“不同内容物 + 相同壳体”链条上横向扩写，也不再让设备、画册和花盆各自成串挤在一起。
  回填方向改成真正属于客厅前台的不同物族混合：保留少量必要的时钟、空气设备、音响设备和香氛设备，同时补入 `国际象棋盘`、`跳棋盘` 这类待客与休闲物，`毛毯卷`、`针织毯卷`、`绒毯卷`、`羊毛毯卷`、`地毯卷` 这类稳定织物卷，`旅行画册`、`艺术画册`、`家居画册`、`建筑画册`、`自然图册`、`城市图册`、`植物图册` 这类桌面读物，以及 `陶瓷小花盆`、`苔藓玻璃盆`、`玻璃花盆`、`香草盆栽`、`多肉花盆`、`绿植花盆`、`鲜花花篮` 等少量绿植与花艺物，并把顺序打散，避免整段读成新的单一后缀矩阵。

  进一步复看时又发现 `xx卷` 仍偏多，因此再把 `毛毯卷`、`绒毯卷`、`羊毛毯卷` 三项替换为 `沙发靠垫组合`、`围棋棋盘`、`花艺组合`，把第四池从“卷状软装串联”继续拉回到客厅待客物、软装物和装饰物的混合结构。

## 8. 修改后复审

- 已完成本类第二轮结构复查。
- `客厅+装饰` 四池当前均为 50 条，且池内唯一。
- `scene_expansion_items` 与其他三池无机械精确重复。
- 按 `test_blind_box_content_model.py` 的同口径做了本类专项校验：
  `scene_expansion_items` 本类 50 条全部通过中型边界检查；
  本类四池均未命中 blocked patterns；
  第四池与前三池无机械精确重叠。
- 当前再次执行 `test_blind_box_content_model.py` 时，全量 14 项规则测试已通过；本类本轮微调没有引入新的共享规则回归。
- 人审结论：
  相比上一轮，这次真正压掉的是“虽然不再是篮筐 / 托盘 / 礼盒链，但又长成另一串 `机 / 器 / 盘 / 盆 / 册` 混合矩阵”的假修复；
  当前第四池已经更像客厅前台会自然共存的时间件、空气与音响设备、棋盘、毯卷、读物和绿植，而不是一段又一段同后缀家族；
  其中读物部分也进一步从 7 条压到 3 条，`画册 / 图册` 不再单独形成一整段可感知模板串；
  同时 `xx卷` 也从 6 条压到 3 条，不再单独形成“软装卷物串联”；
  当前版本可以作为 `客厅+装饰` 的新基线继续使用。

## 9. 剩余风险与暂留理由

- 本轮已经解决的根因是“上一轮虽然摆脱了篮筐 / 托盘 / 礼盒链，却又长成另一批 `xx机 / xx器 / xx盘 / xx盆 / xx册` 的假修复矩阵”。
- 仍保留两类中低风险：
  `visible_small_items` 里的香氛补件、花艺耗材和展示配件单位仍然偏多，本轮没有动它；
  `core_items` 里的花器、摆件和陈列物仍然偏密，只是已经比上一轮少了几处明显的托盘、盒筒和陈列壳。
- 这些残余风险本轮暂留的原因是：
  先优先修正第四池读感最差的假修复链，避免为了继续压密度，马上把第四池又改回另一串 `盒 / 筒 / 篮 / 托盘 / 礼盒` 的旧模板；
  `客厅+装饰` 本身允许一定比例的时钟、音响、香氛设备、读物和花艺物，完全压平反而容易失真。
- 如果下一轮继续细修，本类优先级建议是：
  先轻压第三池的香氛补件和装饰补件串联，再视需要微修第一池中后段的花器、摆件和容器壳密度；
  第四池不要回退到 `托盘 / 礼盒 / 篮筐 / 纸品盒`，也不要再长成新的 `画册 / 花盆 / 毯卷 / 小机器` 纯家族矩阵；当前读物和卷物都已压到较合理密度，后续只做小幅平衡。
