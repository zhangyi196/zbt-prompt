# 客厅+装饰 审查文档

- 数据来源：`Game content extraction/data/blind_boxes.py` 中 `BLIND_BOX_ITEM_POOL_BUNDLES['客厅+装饰']`
- 审查范围：仅审查该类别 `core_items` / `support_items` / `visible_small_items` / `scene_expansion_items` 四池内容
- 本次操作：已同步修改 `blind_boxes.py` 中该类别数据，并重写本审查文档

## 总体结论

状态：`已修改并复审`

这类原先最明显的问题，不是客厅感不够，而是“很多东西都像装饰，但差异主要停留在题材和壳体说明上”：

- `core_items` 容易在 `花瓶 / 花器 / 盆栽 / 摆件 / 书册 / 盘碗` 这些高频装饰族里继续横向扩写。
- `support_items` 容易滑成一串 `底托 / 护套 / 擦布 / 固定件 / 花艺维护件`，看起来像保养和安装清单。
- `visible_small_items` 容易回到 `一盒 / 一盘 / 一排 / 一套` 套同一批糖果、香薰和装饰小物。
- `scene_expansion_items` 最容易掉进 `盒 / 篮 / 包 / 托盘` 的模板池，或者滑向办公数码、线缆理线、电子外设和后场清洁维护。

## 本次修改方向

- `core_items`：压掉一部分只换题材不换轮廓的旧堆叠，保留客厅装饰主线，同时补入 `玻璃糖果罐`、`装饰棋盘`、`铜质铃铛`、`枝形烛台` 这类轮廓更不同的立体本体。
- `support_items`：保留少量真实常见的承托、花艺、香薰和清洁辅助物，但把过细、过像安装件和后场维护件的内容压掉，减少“同一块布 / 同一块托 / 同一块底托”反复扩写。
- `visible_small_items`：保留可见单位，但不再继续扩写书刊、遥控器、电池和办公纸品，改成成套杯垫、餐巾圈、干花、烛杯、迷你花瓶、装饰木珠等更稳的客厅小单位。
- `scene_expansion_items`：把第四池拉回纸品容器、篮筐、音响香薰设备、钟历、花盆、少量托盘和礼盒，不再让它主要由数码配件、办公用品、理线物和连续游戏包壳体构成。

## 本次主要压掉的伪多样性

- `不同装饰题材 + 相同摆件壳体`
- `不同花材 / 植物内容 + 相同花瓶花盆壳体`
- `不同用途说明 + 相同盒 / 篮 / 包 / 托盘后缀`
- `不同客厅杂物 + 相同服务型承托壳`
- `看似家居，实际更像办公桌 / 数码外设 / 后场维护`

## 简要修改理由

- 把主要靠“内容主题变化”成立的条目，换成普通用户一眼就能识别、轮廓更独立的真实装饰物。
- 把办公、理线、数码配件和后场维护链从第四池里撤出去，避免客厅装饰场景被带偏。
- 把第四池的多样性重新建立在不同物体家族上，而不是继续堆 `盒 / 篮 / 包 / 托盘` 这种服务壳体。

## 复审结果

- 四池均为 50 条，池内唯一。
- `scene_expansion_items` 与其他三池无精确重复。
- `scene_expansion_items` 已通过中型场景碎片边界自检，未命中超大物和微小信息面禁区。
- 当前版本已明显压低办公数码错位、游戏包壳体补位、第四池连续承托后缀，以及核心池里只换题材不换本体的旧问题。
- 第四池现在更像客厅前台和茶几周边、沙发周边、角落陈设区自然会出现的中型单物，而不是承托模板池或后场维护池。

## 保留风险

- `客厅+装饰` 天然仍会保留一定数量的花瓶、花盆、摆件、烛台、托盘和书册，这是类别识别需要；后续补写时仍要继续拦截“题材变了、物体本体没变”的扩写。
- `visible_small_items` 里仍保留少量零食、香薰和装饰小件，这是客厅茶几自然会有的单位；但后续若继续补写，不要再回到连续食品盒盘链。
- `scene_expansion_items` 目前仍有一定数量的盒、篮、盘、包，这是客厅前台真实存在的中型承载物；但后续若继续补写，不能再滑回连续游戏包、连续托盘或连续纸品盒链。
