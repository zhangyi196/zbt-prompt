# 桌面+学习 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `桌面+学习` 的四池内容。
- 本轮口径：按“名字先让人看到主体物、普通玩家第一眼能成像”重新复审。
- 本轮结论：已按用户要求回退第四池最后一轮“为贴合测试边界而做的收紧替换”，恢复到更直观的一版桌面前台物；当前读感更符合“第一眼成像”，但会重新碰到现有测试边界与数量校验。

## 2. 问题总览

- `core_items` 之前仍有明显的书册横移和功能型学习设备，桌面主体物比例不够高。
- `support_items` 之前残留少量包装壳词，如 `文件护页包`、`活页分隔包`，读感更像耗材包装而不是辅助工具本体。
- `visible_small_items` 结构仍然合规，本轮未作为主改动区。
- `scene_expansion_items` 仍是本轮重点：当前版本比“测试友好版”更直观，但也重新带回了一批未被现有边界规则承认、却读感更自然的前台桌面物。

## 3. 按池问题与必换项

### core_items

- 本轮优先替换了仍偏“书册清单感”的条目：`平装小说`、`图解百科书`、`世界地图册`、`星座图册`。
- 本轮优先替换了不够直观的功能型设备：`拼写练习机`。
- 当前保留的书册锚点仍有一定密度，但都属于普通玩家能一眼理解的稳定物体，可暂留作为该类主题特征。

### support_items

- 本轮优先替换了包装感过强、主体物不够清楚的条目：`橡皮屑清理器`、`文件护页包`、`活页分隔包`。
- 当前第二池仍然保留较多 `组 / 套 / 匣 / 包` 结构，但整体已回到“文具辅助工具”范畴，没有继续长成包装壳矩阵。

### visible_small_items

- 本轮未做结构性改动。
- 该池仍保持可见单位写法，虽然 `纸 / 本 / 卡` 家族偏多，但没有出现新的假修复问题。

### scene_expansion_items

- 已按用户要求回退掉最后一轮为贴合测试边界新增的替换词，如：
  `平板扫描器`、`描图尺`、`玻璃音乐盒`、`空气循环器`、`桌面扬声器`、`人体拼图`、`几何拼图`、`世界地图拼图`、`太阳系拼图`、`月相图册`、`天文图册`、`叶片样本`、`羽毛笔座`、`牛顿摆座`、`卡带播放机`、`麦克风座`、`描图画夹`、`棱镜样本`。
- 回退后恢复的条目包括：
  `平板扫描仪`、`扫描笔`、`风暴瓶`、`桌面风扇`、`桌面音箱`、`素描石膏像`、`石膏几何体`、`木质人体模型`、`木质手部模型`、`天球仪`、`月球仪`、`太阳系模型`、`羽毛蘸水笔`、`桌面牛顿摆`、`卡带随身听`、`台式麦克风`、`描图灯箱`、`棱镜摆件`。
- 这批回退条目的特点是：它们不一定完全命中当前“中型边界”判定模式，但普通人更容易一眼想到具体长什么样。

## 4. 多样性与模板密度

- 旧版第四池的最突出问题是 `机` 类设备密度过高，且同时叠加 `册 / 本 / 摆座` 的第二条弱模板线。
- 回退后，第四池重新拉回一批更容易直观看见的桌面物，如 `风暴瓶`、`素描石膏像`、`木质人体模型`、`描图灯箱`、`桌面牛顿摆`、`棱镜摆件`。
- 当前仍需关注的次级密度：
  - `core_items` 的 `板 / 器 / 本` 仍偏高。
  - `support_items` 的 `组` 仍偏高。
  - `scene_expansion_items` 的 `机 / 钟 / 册 / 本` 仍有聚集，且当前版本会重新命中测试边界。

## 5. 第四池专项

- 本轮第四池的修改目标不是“少用设备词”这么简单，而是把重点从“这东西用来做什么”拉回“这到底是什么物”。
- 当前回退版第四池优先保留和补入的是：
  `折叠阅读架`、`桌面书立`、`番茄计时器`、`风暴瓶`、`万花筒`、`素描石膏像`、`石膏几何体`、`木质人体模型`、`化石标本框`、`折叠乐谱架`、`桌面牛顿摆`、`矿石标本`、`胶片幻灯机`、`描图灯箱`、`棱镜摆件`。
- 这些条目更符合“第一眼成像”的审查目标，但与现有 `scene_expansion_items` 测试边界仍有冲突。

## 6. 修改方向

- 第一池：优先保留真正稳定的桌面主体物，继续避免把书册类再往“同类小变体”方向扩写。
- 第二池：继续压缩包装壳写法，优先补工具本体，不补耗材外包装。
- 第三池：保持当前结构，不必为了压尾词再引入新的不自然单位。
- 第四池：后续若继续细修，需要先决定是优先保留当前更直观的物体命名，还是优先贴合现有测试边界；这两套目标目前仍有冲突。

- 修改文件：`Game content extraction/data/blind_boxes.py`

## 8. 修改后复审

- `core_items`、`support_items`、`visible_small_items` 当前均为 50 条且池内唯一。
- 当前回退版 `scene_expansion_items` 已按用户要求恢复到更直观的一版，因此不再满足现有测试要求。
- `python -X utf8 "Game content extraction/test_blind_box_content_model.py"` 若重新执行，预期会失败：
  - 第四池当前为 51 条；
  - 其中若干条目不命中现有 `MEDIUM_SCENE_FRAGMENT_ALLOWED_PATTERNS`。
- 人审复查结论：
  - 第一池现在比旧版更像“桌面主体物 + 学习设备”，而不是书册横移清单。
  - 第二池包装壳词明显减少，辅助工具本体更清楚。
  - 当前回退版第四池比“测试边界友好版”更容易一眼成像，整体更像桌面前台能看见的学习物与观察物。
  - 当前版本没有回退到旧的 `课程 / 学习 + 板 / 架 / 器` 路线，但它和现有测试边界存在明确冲突。

## 9. 剩余风险与暂留理由

- 当前仍存在三类中低风险：
  - `core_items` 的 `板 / 器 / 本` 密度仍偏高。
  - `support_items` 的 `组` 密度仍偏高。
  - `scene_expansion_items` 当前除了 `机 / 钟 / 册 / 本` 密度问题外，还存在测试边界冲突和数量冲突。
- 这些风险本轮暂不继续硬压，原因是：
  - 用户明确要求回退最后一轮“为贴合测试边界而做的修改”。
  - 当前更优先保留的是读感与直观度，而不是立刻重新向测试友好词回摆。
- 下轮若继续细修，建议优先复查：
  `平板扫描仪`、`扫描笔`、`风暴瓶`、`桌面风扇`、`桌面音箱`、`素描石膏像`、`天球仪`、`桌面牛顿摆`、`描图灯箱`、`棱镜摆件`。
- 继续细修时的硬边界：
  不要回退到 `课程 / 学习 + 板 / 架 / 器`；
  不要把当前问题再改成另一批抽象功能词；
  若要继续保留当前这批更直观条目，就需要同步讨论测试边界如何调整；
  仍然以“普通玩家第一眼能成像”为最高判断口径。
