# 儿童房+玩具 审查文档

审查日期：2026-05-07

## 审查对象

- 类别：`儿童房+玩具`
- 数据来源：`Game content extraction/data/blind_boxes.py` 中 `BLIND_BOX_ITEM_POOL_BUNDLES['儿童房+玩具']`
- 审查范围：仅审查该类别 `core_items` / `support_items` / `visible_small_items` / `scene_expansion_items` 四池内容
- 本次操作：已同步修改 `blind_boxes.py` 中该类别数据，并重写本审查文档

## 总体结论

状态：`已修改并复审`

这类原先最明显的问题，不是玩具不够，而是“同一类玩具被反复用不同包装、不同小件、不同后缀凑成一长串”：

- `core_items` 之前积木、拼图、绘本、娃娃、车辆、角色扮演和轨道玩具同族过密。
- `support_items` 之前大量是剪、夹、刀、板、轨道碎件、角色小工具和过薄的小配件。
- `visible_small_items` 之前很多条目只是把核心玩具拆成更小的盒装、排装、片装、薄片和集合件。
- `scene_expansion_items` 之前明显滑向生活后勤、电子办公、安全防护和墙面固定物，已经不是玩具场景的前台中型单物。

## 本次修改方向

- `core_items`：保留儿童房最核心的积木、拼插、音乐、美术、绘本、娃娃、过家家、交通、动物、运动和感统玩具主线，同时压掉一部分只换题材不换轮廓的同族堆叠。
- `support_items`：保留真正给玩具和手工服务的辅助物，但尽量用更厚实、轮廓更清楚的辅助件，减少薄片、细棒、细牌和纯后场件。
- `visible_small_items`：保留清楚可见的小单位，但压掉对轨道碎片、薄片、小牌和集合件的机械扩写，改成更容易一眼识别的成套单位。
- `scene_expansion_items`：把第四池收回到玩具房前台自然会出现的中型物，如绘本篮、积木桶、拼图盒、音乐机、角色道具包、玩具收纳和玩具操作辅助物，避免继续向生活后勤、办公电子和墙面固定物偏移。

## 本次主要压掉的伪多样性

- `不同玩具主题 + 相同本体壳`
- `不同角色扮演内容 + 相同帽子 / 包 / 盒后缀`
- `不同小玩具内容 + 相同一盒 / 一排 / 一筒模板`
- `不同辅助功能 + 相同夹 / 刀 / 板 / 盘 / 盒后缀`
- `看似儿童房，实际更像生活收纳、办公学习或安全防护`

## 简要修改理由

- 把差异主要靠内容说明成立的条目，换成轮廓更清楚、普通用户一眼能识别的真实玩具或玩具辅助物。
- 把第四池从后勤化、电子化、墙面化拉回玩具前台，确保它还是“中型单物补充池”，不是杂物池。
- 让多样性更多来自玩具家族和轮廓差异，而不是继续堆包装单位和同尾词。

## 复审结果

- 四池均为 50 条，池内唯一。
- `scene_expansion_items` 与其他三池无精确重复。
- `scene_expansion_items` 已通过中型场景碎片边界自检，未命中超大物和微小信息面禁区。
- 当前版本已明显压低积木拼图过密、书册与轨道碎件重复、第四池生活后勤化和模板化问题。
- 第四池现在更像儿童房前台和玩具角周边能自然出现的中型单物，而不是办公、收纳或安全防护后场。

## 保留风险

- `儿童房+玩具` 天然仍会保留一些积木、绘本、娃娃、车轨和手工物，这是类别识别需要；后续补写时仍要继续拦截“只换主题、不换本体”的扩写。
- `visible_small_items` 里仍会保留少量餐厨过家家、轨道和手工单位，这是玩具房自然组成；但后续不要再回到大量薄片、碎件和集合件。
- `scene_expansion_items` 目前仍有一定数量的盒、桶、筒、包、盘、册，这是可接受的玩具房中型物；但后续若继续补写，不能再滑回生活后勤或办公电子链。
