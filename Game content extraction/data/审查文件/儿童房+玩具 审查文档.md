# 儿童房+玩具 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `儿童房+玩具` 的四池内容。
- 本轮结论：当前版本的第一池仍在场景内，但已经明显被 `套装 / 盒 / 桶 / 组` 横向平移占满；第二池被玩法辅助件和收纳壳混写；第四池仍然没有通过，核心问题是它被写成了玩具收纳区、手工物料区和儿童整理区的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的儿童房和游戏区场景感还在，但 `套装 / 盒 / 桶 / 组 / 箱` 家族已经偏密，部分条目更像玩具包装壳和主题套件横向扩写。
- `support_items` 的主要问题不是不属于玩具，而是把真正的玩法辅助件、收纳壳、展示壳和场景底板混在了一池里。
- `visible_small_items` 格式合规，但卡牌、手工耗材、贴片和组件盒已经有明显“补给清单感”。
- `scene_expansion_items` 是当前最大问题：篮、桶、盒、包、册、本、盘、清洁盆和手工纸包同时回流，已经偏离“儿童房前台自然单物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池没有出现必须整片清洗的问题，但包装壳和主题套件已经过密。
- 低优先级平衡项：`木质积木桶`、`磁力片套装`、`形状分类盒`、`水彩颜料盒`、`橡皮泥套装`、`玩具医生箱`、`玩具工具箱`、`火车轨道套装`、`动物农场套装`、`迷你保龄球组`、`穿绳珠珠盒`、`手偶剧场盒`。
- 这些条目不是立刻必换，而是说明本池已经在用“主题套件 + 包装壳”维持密度；后续若要腾位，可优先从这组下手。
- 当前可作为锚点保留的方向：`彩虹套叠塔`、`套叠杯玩具`、`轨道滚球塔`、`弹珠滚塔`、`儿童算盘`、`敲击台`、`木琴`、`小鼓`、`玩具键盘`、`磁性画板`、`桌面画架`、`硬页绘本`、`立体翻翻书`、`触摸布书`、`毛绒小熊`、`布娃娃`、`娃娃屋`、`娃娃推车`、`娃娃摇篮`、`模型小汽车`、`玩具消防车`、`木质火车头`、`玩具飞机`、`恐龙模型`、`机器人玩具`。

### support_items

- 收纳壳、展示壳和保护壳过密，优先替换：`大号印章盒`、`拼图托盘`、`拼图分类碟`、`绘本书立`、`彩笔收纳筒`、`贴纸收纳盒`、`蜡笔收纳盒`、`画纸收纳夹`、`角色道具架`、`积木收纳袋`、`轨道收纳盒`、`恐龙展示盘`、`公仔收纳篮`、`球拍收纳袋`、`鼓槌套`。
- 底板和场景辅件存在越界风险，优先复查：`黏土垫板`、`积木底板`、`车库斜坡板`、`火车站台`、`轨道桥墩`、`停车标志牌`。
- 问题根因不是这些词完全不能出现，而是这一池已经同时承担了“玩法工具池”和“配套收纳池”两种职责。
- 当前更稳的辅助物方向仍应放在真正参与涂画、手工、拼搭、角色扮演和轨道玩法的工具本体，如 `安全剪刀`、`儿童胶棒`、`可洗胶水瓶`、`美术夹板`、`画笔清洗桶`、`水彩调色碟`、`颜料海绵棒`、`手指画滚轮`、`黏土滚轮`、`黏土压模`、`积木拆件器`、`娃娃奶瓶`、`娃娃背带`、`娃娃梳子`、`玩具围裙`、`玩具锅铲`、`玩具砧板`、`玩具听诊器`、`玩具扳手`、`玩具螺丝刀`、`玩具锤子`、`泡泡液瓶`、`游戏计时器`。

### visible_small_items

- 整体格式合规，暂不建议整池推倒。
- 低优先级压缩项：`一叠拼图卡`、`一套记忆卡牌`、`一包手工彩纸`、`一束扭扭棒`、`一盒海绵印章`、`一盒黏土罐`、`一盘大颗粒珠子`、`一盒串绳木珠`、`一盅游戏骰子`、`一筒拼豆板`、`一盒闪粉贴片`、`一套木偶服装`。
- 这些条目本身不违规，但它们和前两池、第四池的物料壳条目叠加后，会进一步放大“手工教具补给区”的读感。

### scene_expansion_items

- 收纳壳和包袋回流，必须优先替换：`绘本篮`、`积木桶`、`拼图盒`、`画笔筒`、`蜡笔筒`、`木偶剧盒`、`手偶包`、`角色披风包`、`角色面具盒`、`消防头盔包`、`恐龙桶`、`轨道桶`、`玩具车桶`、`小鼓包`、`沙锤筒`、`铃鼓包`、`公仔提包`、`软球袋`、`飞盘包`。
- 册、本、盘和纸包家族回流，必须优先替换：`奖牌册`、`手工册`、`涂鸦本`、`涂色册`、`成长相册`、`贴画册`、`奖励册`、`积木盘`、`串珠盘`、`黏土盘`、`印章盘`、`拼豆盘`、`手工纸包`、`剪纸本`。
- 后场清洁和整理物回流，必须优先替换：`纸巾盒`、`湿巾盒`、`桌面垃圾桶`、`玩具清洁盆`、`洗笔桶`、`布偶洗澡盆`、`泡泡盘`。
- 篮和桶类收纳物过密，必须整体压缩：`毛绒玩偶篮`、`球类篮`、`套圈桶`。
- 当前较接近第四池方向、可作为保留候选的仅有少数：`儿童闹钟`、`故事机`、`白噪音机`、`泡泡机`、`星星投影机`。

## 4. 多样性与模板密度

- 第一池最拥挤的家族是 `套装 / 盒 / 桶 / 组 / 箱`，说明主体池已经出现主题玩具套件和包装壳横向平移。
- 第二池最拥挤的尾词是 `盒 / 筒 / 盘 / 架 / 袋 / 板`，主要问题是“不同玩法对象 + 相同收纳壳”与“不同玩法对象 + 相同底板辅件”的双重伪多样性。
- 第三池格式没错，但 `卡牌 / 手工耗材 / 小组件 / 贴片` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `篮 / 桶 / 盒 / 筒`、`包 / 提包 / 披风包 / 软球袋`、`册 / 本 / 相册 / 奖励册`、`盘 / 盆 / 纸包 / 清洁盆` 这类收纳和物料壳体链。

## 5. 第四池专项

- 第四池当前的根因不是单个坏词，而是整池写法站在“玩具收纳和手工物料管理”视角，而不是“儿童房前台游戏场景”视角。
- 结果是四类问题同时叠加：收纳桶篮堆叠、角色扮演包袋堆叠、册本和盘类物料壳堆叠、后场清洁整理物混入。
- 本类第四池应优先补“普通玩家一眼就能认出的儿童房地面或游戏区前台中型物、自然外露的稳定玩乐场景补充物”，而不是继续补桶、盒、包、册、本、盘和清洁盆。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留稳定的大玩具、绘本、玩偶、轨道玩具和角色玩具，必要时压缩 `套装 / 盒 / 桶 / 箱` 的横向平移。
- 第二池：从“玩法辅助件 + 收纳壳混池”拉回“真正辅助工具池”，优先保留能直接参与绘画、手工、拼搭、角色扮演和游戏动作的工具本体。
- 第三池：继续保持 `一盒 / 一套 / 一排 / 一桶 / 一筒 / 一包` 的可见单位格式，但避免手工物料、卡牌组件和贴片继续膨胀。
- 第四池：按“儿童房前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用收纳桶篮、角色包袋、册本纸包和清洁整理物作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `桶 / 盒 / 包 / 筒 / 盘 / 册 / 本 / 篮` 是否还成片出现。

## 8. 修改后复审

- 已完成本类一轮结构复查。
- `儿童房+玩具` 四池当前均为 50 条，且池内唯一。
- `scene_expansion_items` 与其他三池无机械精确重复。
- 按 `test_blind_box_content_model.py` 的同口径做了本类专项校验：
  `scene_expansion_items` 本类 50 条全部通过中型边界检查；
  本类四池均未命中 blocked patterns；
  第四池与前三池无机械精确重叠。
- 进一步按后缀统计复查后，第四池里的 `卷 / 画册 / 图册` 已分别压到 2 条，满足“每种后缀最多保留两个”的补充要求。
- 当前再次执行 `test_blind_box_content_model.py` 时，全量 14 项规则测试已通过；本类本轮微调没有引入新的共享规则回归。
- 人审结论：
  这轮已经明显压下了第二池的收纳壳 / 展示壳扩写，以及第四池的 `桶 / 盒 / 包 / 册 / 盘 / 清洁盆` 回流；
  当前第四池已经更像儿童房前台会自然共存的计时件、故事设备、棋盘、睡袋、少量读物和软装物，而不是玩具整理区和手工物料区；
  其中 `卷 / 画册 / 图册` 都已压到最多 2 条，不再单独形成可感知的后缀模板串；
  当前版本可以作为 `儿童房+玩具` 的新基线继续使用。

## 9. 剩余风险与暂留理由

- 本轮已经解决的根因是“第四池站在玩具收纳和手工物料管理视角补词，而不是站在儿童房前台场景视角补词”。
- 仍保留两类中低风险：
  `visible_small_items` 里的卡牌、手工耗材、贴片和组件盒仍然偏多，本轮没有动它；
  `scene_expansion_items` 里仍保留一定比例的 `机 / 睡袋 / 花盆 / 画架`，只是这些条目已经不再是同一类收纳壳或物料壳的横向平移，而是不同物族混合。
- 这些残余风险本轮暂留的原因是：
  先优先修正第二池和第四池读感最差的收纳壳链，避免为了继续压尾词密度，马上把第四池又改回另一串 `箱 / 桶 / 盒 / 包 / 册` 的旧模板；
  `儿童房+玩具` 本身允许一定比例的故事设备、棋盘、图册、毯卷和睡袋，完全压平反而容易失真。
- 如果下一轮继续细修，本类优先级建议是：
  先轻压第三池的卡牌、手工耗材和贴片组件串联，再视需要微修第一池中后段的 `套装 / 盒 / 桶 / 箱` 密度；
  第四池不要回退到 `桶 / 盒 / 包 / 册 / 盘 / 清洁盆`，也不要再长成新的纯 `机 / 卷 / 画册 / 图册 / 袋` 家族；当前相关后缀已压到较合理密度，后续只做小幅平衡。
