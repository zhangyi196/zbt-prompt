# 卧室+梳妆 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `卧室+梳妆` 的四池内容。
- 本轮结论：当前版本的第一池和第三池基本仍在场景内，但第一池已经出现明显的瓶罐壳、盒壳和软包堆叠，第二池几乎被 `盒 / 筒 / 包 / 卷 / 护套` 接管，第四池仍然没有通过，核心问题是它被写成了收纳壳、展示盘、挂架、礼袋和氛围摆件的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的梳妆台场景感还在，但 `瓶 / 罐 / 盒 / 包 / 架` 家族已经偏密，部分条目更像包装壳或收纳壳横向平移。
- `support_items` 的主要问题不是不属于梳妆，而是过度依赖 `盒 / 筒 / 包 / 卷 / 护套 / 保护壳` 这类容器和保护壳来制造多样性。
- `visible_small_items` 格式合规，但大量 `一盒 / 一包 / 一筒` 的化妆耗材、护肤耗材和饰品包装单位已经开始放大“补给清单感”。
- `scene_expansion_items` 是当前最大问题：纸品壳、托盘、展示盘、相册、挂架、浅篮、礼袋和泛装饰物同时回流，已经偏离“卧室梳妆前台自然单物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池没有出现必须整片清洗的问题，但瓶罐壳和收纳壳已经过密。
- 低优先级平衡项：`化妆刷筒`、`彩妆刷卷包`、`散粉罐`、`粉底液瓶`、`彩妆铅笔筒`、`口红立架`、`润唇膏罐`、`首饰盒`、`假睫毛盒`、`手表盒`、`化妆包`、`旅行洗漱包`、`化妆棉罐`、`护肤软包`。
- 这些条目不是立刻必换，而是说明本池已经在用“内容物 + 壳体”维持密度；后续若要腾位，可优先从这组下手。
- 当前可作为锚点保留的方向：`台式化妆镜`、`手持化妆镜`、`电热卷发棒`、`直发夹`、`迷你吹风机`、`热风造型刷`、`宽齿梳`、`气垫梳`、`洁面仪`、`美容滚轮`、`香水瓶`、`指甲油瓶`。

### support_items

- 容器壳和保护壳过密，优先替换：`修眉刀盒`、`眉剪盒`、`美容镊盒`、`指甲剪盒`、`指甲锉盒`、`美甲锉块盒`、`护肤勺盒`、`面膜刷盒`、`粉扑盒`、`海绵蛋盒`、`粉底海绵盒`、`粉扑清洁皂盒`、`发卷筒`、`缎面发圈包`、`小发爪盒`、`刘海夹盒`、`发蜡棒筒`、`旅行空瓶包`、`美甲分趾器盒`、`洗脸腕带包`、`刷具保护筒`、`美妆蛋保护盒`、`眉形模板盒`、`首饰小镊盒`。
- 问题根因不是这些词完全不能出现，而是这一池已经过度依赖“工具 + 壳体”或“耗材 + 保护壳”写法来撑多样性。
- 当前更稳的辅助物方向仍应放在真正参与梳妆、修整、清洁、分装和护理动作的工具本体，如 `睫毛夹`、`硅胶面膜铲`、`化妆刷清洁垫`、`化妆刷清洁杯`、`乳霜刮刀`、`化妆调色板`、`梳齿清洁耙`、`香水分装漏斗`、`分装瓶清洁刷`。

### visible_small_items

- 整体格式合规，暂不建议整池推倒。
- 低优先级压缩项：`一包化妆棉`、`一包卸妆棉`、`一包洗脸巾`、`一筒棉签`、`一罐棉片`、`一卷棉柔巾`、`一盒粉扑`、`一托海绵蛋`、`一套化妆海绵`、`一盒迷你香水`、`一套旅行空瓶`、`一包护肤小样`、`一盒耳环`、`一盒项链`。
- 这些条目本身不违规，但它们和前两池、第四池的壳体条目叠加后，会进一步放大“梳妆耗材补给区”的读感。

### scene_expansion_items

- 纸品壳和软包回流，必须优先替换：`抽纸盒`、`湿巾包`、`棉柔巾包`、`首饰卷包`、`旅行首饰卷包`、`化妆工具包`、`梳妆布包`、`丝巾礼袋`。
- 托盘、展示盘和承载壳回流，必须优先替换：`香薰石浅盘`、`木质托盘`、`玻璃托盘`、`陶瓷托盘`、`护肤瓶托盘`、`首饰展示盘`、`发饰展示盘`。
- 挂架、浅篮、筒盒家族回流，必须优先替换：`围巾挂架`、`发带挂架`、`发饰浅篮`、`围巾浅篮`、`洗脸腕带篮`、`鲨鱼夹筒`、`发卷玻璃筒`、`眼镜硬壳盒`、`备用毛巾架`。
- 泛家居摆件和礼赠物偏多，必须整体压缩：`翻页台历`、`桌面月历`、`留声机`、`音乐盒`、`陶瓷小花盆`、`苔藓玻璃盆`、`干花花盆`、`鲜花花篮`、`香薰蜡烛座`、`玻璃烛台座`、`桌面相册`、`照片相册`、`明信片相册`、`客用留言本`、`香氛礼盒`。
- 当前较接近第四池方向、可作为保留候选的仅有少数：`桌面小钟`、`沙漏计时器`、`白噪音机`、`香薰机`、`桌面加湿器`、`蒸脸器`、`美容喷雾器`、`美甲烘干机`、`首饰清洁机`。

## 4. 多样性与模板密度

- 第一池最拥挤的家族是 `瓶 / 罐 / 盒 / 包 / 架`，说明主体池已经出现化妆品壳体和收纳壳横向平移。
- 第二池最拥挤的尾词是 `盒 / 筒 / 包 / 卷 / 护套 / 泵`，主要问题是“不同小工具 + 相同保护壳”的伪多样性。
- 第三池格式没错，但 `一盒 / 一包 / 一筒` 的耗材和分装单位已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `托盘 / 展示盘 / 浅盘`、`卷包 / 工具包 / 礼袋`、`挂架 / 浅篮 / 筒盒`、`相册 / 花盆 / 烛台 / 礼盒` 这类氛围摆件链。

## 5. 第四池专项

- 第四池当前的根因不是单个坏词，而是整池写法站在“收纳展示和礼赠氛围”视角，而不是“卧室梳妆前台场景”视角。
- 结果是四类问题同时叠加：纸品壳和软包堆叠、展示盘堆叠、挂架浅篮堆叠、泛装饰摆件和礼赠物混入。
- 本类第四池应优先补“普通玩家一眼就能认出的梳妆台或床边护理前台中型物、自然外露的稳定设备或场景补充物”，而不是继续补卷包、托盘、挂架、礼袋、展示盘和礼盒。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留稳定的镜子、梳具、造型电器、少量高频彩妆和护肤主体物，必要时压缩不同内容物共用相似壳体的条目。
- 第二池：从“工具保护壳池”拉回“辅助工具池”，优先补真正参与梳妆、修整、清洁、分装和护理动作的工具本体。
- 第三池：继续保持 `一盒 / 一包 / 一筒 / 一套 / 一卷 / 一盘` 的可见单位格式，但避免耗材包装和分装单位继续膨胀。
- 第四池：按“梳妆前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用托盘、展示盘、挂架、礼袋、礼盒和纯收纳壳作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `盒 / 筒 / 包 / 架 / 盘 / 礼袋 / 礼盒` 是否还成片出现。

## 7. 本次修改记录

  本轮仍未改第三池；它的 `一盒 / 一包 / 一筒` 耗材单位偏密问题继续记账，但不是这次“假修复重做”的主战场。

  回填方向改成真正属于卧室梳妆前台的不同物族混合：保留少量必要的时钟、音响、香薰和护理电器，同时补入 `斜挎包`、`手提包`、`单肩包`、`晚宴手包`、`编织手袋` 这类真实包袋，本体更稳的 `领带卷`、`腰带卷`、`丝巾卷`、`床尾毯卷`、`针织毯卷`、`绒毯卷`，以及 `软毛衣刷`、`衣物除尘刷`、`除静电刷`、`丝绒衣刷`、`家居画册`、`摄影画册`、`时尚画册`、`珠宝图册`、`鲜花花篮` 等，让第四池从“美容设备陈列串”退回到更像卧室梳妆区会自然出现的中型单物混合池。

## 8. 修改后复审

- 已完成本类第二轮结构复查。
- `卧室+梳妆` 四池当前均为 50 条，且池内唯一。
- `scene_expansion_items` 与其他三池无机械精确重复。
- 按 `test_blind_box_content_model.py` 的同口径做了本类专项校验：
  `scene_expansion_items` 本类 50 条全部通过中型边界检查；
  本类四池均未命中 blocked patterns；
  第四池与前三池无机械精确重叠。
- 共享规则测试目前仍会被其他尚未改完的类别拖失败，这一轮不再把“全量红灯”误判成本类失败信号；本类本轮新增内容经专项脚本复查已通过。
- 人审结论：
  相比上一轮，这次真正压掉的是“虽然不再是托盘礼袋链，但又长成另一串护理小电器矩阵”的假修复；
  当前第四池已经更像床边与梳妆前台会自然共存的时钟、音响、香薰、包袋、衣物卷、衣刷和少量画册，而不是一排功能相近的小机器；
  当前版本可以作为 `卧室+梳妆` 的新基线继续使用。

## 9. 剩余风险与暂留理由

- 本轮已经解决的根因是“上一轮虽然摆脱了托盘 / 挂架 / 礼袋链，却又长成另一批 `xx机 / xx器 / xx钟 / xx盆` 的假修复矩阵”。
- 仍保留两类中低风险：
  `core_items` 里的 `瓶 / 罐 / 盒` 仍然偏多，只是已经比上一轮少了几处明显的 `架 / 包 / 盒` 弱壳体；
  `visible_small_items` 里的 `一盒 / 一包 / 一筒` 耗材包装单位仍偏密，本轮没有动它。
- 这些残余风险本轮暂留的原因是：
  先优先修正第四池读感最差的假修复链，避免为了继续压第一池和第三池密度，马上把第四池又改回另一串 `盒 / 篮 / 本 / 册` 的新模板；
  `卧室+梳妆` 本身允许一定比例的香薰、音响、护理电器、包袋和织物卷，完全压平反而容易失真。
- 如果下一轮继续细修，本类优先级建议是：
  先轻压第一池中后段的 `瓶 / 罐 / 盒` 密度，再视需要微修第三池后半段的耗材包装串联；
  第四池不要回退到 `支架 / 托盘 / 礼袋`，也不要再长成新的 `包 / 卷 / 册` 纯家族矩阵，只做小幅平衡。
