# 手作+编织 审查文档

## 1. 类别与范围

- 审查对象：`Game content extraction/data/blind_boxes.py` 中 `手作+编织` 的四池内容。
- 本轮结论：当前版本的第一池仍能读出“编织工位”的主体场景，但已经混入较多图样册、样本册、半成品和织片目录；第二池被 `盒 / 筒 / 套 / 夹 / 包 / 册 / 袋` 的配件壳和项目壳拉散；第四池明显没有通过，已经被写成收纳、洗护、样本资料和氛围设备区的混合池。
- 修改优先级：`scene_expansion_items` > `support_items` > `core_items` > `visible_small_items`。

## 2. 问题总览

- `core_items` 的毛线、棒针、钩针、编织框和编织成品主场景还在，但“材料目录”和“样片目录”的读感已经开始抬头。
- `support_items` 的主要问题不是与编织无关，而是把真实辅助工具、保护壳、项目夹、样片夹和收纳壳混在了一池里。
- `visible_small_items` 格式合规，但记号扣、样片、短针、小线材和保护帽成串后，已经很像“编织配件目录”。
- `scene_expansion_items` 是当前最大问题：提篮、布袋、工具包、收纳盒、洗护盆、卷筒、样本件和拼贴件集中回流，明显偏离“前台自然中型物”的第四池职责。

## 3. 按池问题与必换项

### core_items

- 本池最大问题不是整体失场，而是 `册 / 半成品 / 织片 / 样本 / 收纳桶 / 纱卷` 家族已经开始挤占主体位。
- 低优先级平衡项：`毛线收纳桶`、`图样册`、`针法练习本`、`配色样卡册`、`织片样本册`、`线色参考册`、`编织图鉴`、`围巾半成品`、`毛帽半成品`、`袜子半成品`、`手套半成品`、`披肩半成品`、`毛毯半成品`、`针织杯套`、`针织坐垫`、`祖母方格片`、`罗纹织片`、`麻花织片`、`钩针圆片`、`针法样片`、`玩偶钩织体`。
- 这些条目不一定要立刻整片清洗，但它们说明本池已经开始用“图样册、样片和半成品”维持密度；后续若要腾位，可优先从这组压缩。
- 当前可作为锚点保留的方向：`羊毛线团`、`棉线线轴`、`毛线蛋糕`、`亚克力毛线饼`、`手染毛线绞`、`粗纱线束`、`马海毛线球`、`羊驼毛绞`、`雪尼尔线团`、`T恤纱卷`、`竹制棒针`、`木制棒针`、`金属棒针`、`环形棒针`、`双头棒针`、`粗号钩针`、`细号钩针`、`人体工学钩针`、`阿富汗钩针`、`手摇绕线器`、`方形编织框`、`圆形织帽器`、`纱线绕线架`、`毛线碗`、`编织篮`、`针织开衫`、`针织背心`、`针织围脖`、`针织帽`。

### support_items

- 本池被配件壳、项目壳和保护壳拉得过散，优先替换：`安全剪刀套`、`缝合针盒`、`毛线针筒`、`环针连接线包`、`环针收纳夹`、`棒针防护套`、`针尖保护帽`、`钩针握柄套`、`钩针束带`、`棒针束带`、`图样磁条`、`图样夹条`、`样片活页夹`、`样片夹`、`线团网袋`、`可拆针头盘`、`针织项目册`、`备用针筒`、`线头收集篮`。
- 本池里也存在“同对象不同补位 + 相同壳体”的伪多样性，优先复查：`织片固定夹`、`花样夹`、`作品夹`、`织物夹`、`线圈夹`。
- 问题根因不是这些词完全不能出现，而是这一池已经同时承担了“真实辅助工具池”“项目管理池”和“保护收纳池”三种职责。
- 当前更稳的辅助物方向仍应放在真正参与计数、测量、剪线、缝合、理线、定型和护理动作的工具本体，如 `行数计数器`、`伸缩卷尺`、`针号尺`、`密度尺`、`编织量规`、`纱线剪刀`、`弹簧线剪`、`针目保持器`、`理线梳`、`分线盘`、`线团转盘`、`纱线导引环`、`纱线张力器`、`毛球制作器`、`绒球修剪器`、`绒球量规`、`定型钢丝`、`定型梳`、`定型梳针`、`泡沫定型块`、`喷雾瓶`、`织物护理刷`、`护腕套`、`工作布`、`织片压条`、`编织参考尺`。

### visible_small_items

- 低优先级压缩项：`一盒开口记号扣`、`一盒彩色记号扣`、`一盒闭口记号环`、`一盒锁针记号扣`、`一包棒针防脱塞`、`一包针尖保护套`、`一套环针连接线`、`一套可换针头帽`、`一套环针锁紧钥匙`、`一组图样磁条`、`一组图样翻页夹`、`一组样片夹`、`一包线端夹`、`一包织片固定夹`、`一包理线夹`、`一包停针别针`、`一叠针法样片`、`一叠罗纹织片`、`一叠麻花织片`、`一叠祖母方格片`、`一叠钩针圆片`、`一册针法卡`、`一册配色样卡`、`一盒织片样片`、`一盒配色样本`、`一册编织图谱`、`一包毛线束带`、`一束配色短线`、`一束毛线尾线`、`一卷缝合线`、`一卷废线绳`、`一排短钩针`、`一排针帽`、`一组针号尺`、`一组密度尺`、`一筒短双头针`、`一包钩针握柄套`。
- 这些条目本身未必违规，但它们和前两池、第四池的样片壳与收纳壳叠加后，会继续放大“编织配件与样本目录”的读感。

### scene_expansion_items

- 氛围设备和护理设备回流，必须优先替换：`便携收音机`、`白噪音机`、`桌面小钟`、`沙漏计时器`、`翻页月历`、`编织录音笔`、`香薰机`、`桌面加湿器`、`便携暖风机`、`电动除毛器`、`手持吸尘器`、`蒸汽顺毛刷`、`针织除屑刷`。
- 提篮、布袋、工具包和收纳壳家族过密，必须优先替换：`毛线提篮`、`线团提篮`、`编织收纳篮`、`羊毛收纳篮`、`项目布袋`、`围巾布包`、`针织工具包`、`成品防尘袋`、`钩针笔袋`、`钩针收纳袋`、`配件收纳袋`、`羊毛收纳盒`。
- 卷筒、线筒和洗护容器家族过密，必须优先替换：`毛线卷筒`、`缝合线卷`、`配色线筒`、`羊毛洗护盆`、`冷水浸泡盆`、`折叠水桶`、`清洗水桶`、`毛线铁盒`、`茶杯托盘`、`护理托盘`、`线头收集桶`、`废线收集桶`。
- 资料、样本和拼贴家族回流，必须优先替换：`图样词典`、`绒球套组`、`配色样本`、`毛线拼贴`、`织片拼贴`、`钩织玩偶半成品`、`拼色围巾半成品`、`祖母方格毯面`、`编织布面`、`拼色针织布面`。
- 当前较接近第四池方向、可作为保留候选的条目相对有限：`桌面小钟`、`沙漏计时器`、`香薰机`、`桌面加湿器`、`便携暖风机`。

## 4. 多样性与模板密度

- 第一池当前最明显的压力不是“编织主体物太多”，而是 `册 / 半成品 / 织片 / 样本 / 卷 / 桶` 与材料目录家族一起抬高了场景杂音。
- 第二池最拥挤的尾词是 `盒 / 筒 / 套 / 夹 / 包 / 册 / 袋`，主要问题是“不同配件或项目对象 + 相同保护壳/项目壳”的伪多样性。
- 第三池格式没错，但 `记号扣 / 样片 / 短针 / 小线材 / 保护帽` 已经成串，后续只宜轻压，不宜继续扩写。
- 第四池当前最危险的四条串联家族是 `提篮 / 收纳篮 / 布袋 / 工具包`、`洗护盆 / 水桶 / 托盘 / 收集桶`、`图样词典 / 配色样本 / 拼贴 / 半成品`、`卷筒 / 线筒 / 收纳盒 / 笔袋` 这几类编织工位服务壳体链。

## 5. 第四池专项

- 第四池当前的根因不是个别坏词，而是整池写法站在“收纳、洗护、样本资料和氛围设备系统”的视角，而不是“手作编织前台工位”的视角。
- 结果是四类问题同时叠加：氛围设备和护理设备成串、提篮工具包成串、样本拼贴和半成品成串、洗护容器与卷筒成串。
- 本类第四池应优先补“普通玩家一眼就能认出的编织桌面、放线区、作品展示位或手边工作角”里的自然中型单物，而不是继续补工具包、收纳袋、洗护盆和样本拼贴。
- 这一池不适合沿现有词表微调；建议修改阶段按高风险家族整段清理，再回填更自然的前台物。

## 6. 修改方向

- 第一池：保留毛线、棒针、钩针、编织框、成衣和少量稳定样片主物，必要时压缩图样册、样本册、半成品和织片目录的横向平移。
- 第二池：从“项目管理壳与保护收纳壳混池”拉回“真正辅助工具池”，优先保留计数、测量、剪线、缝合、理线、定型和护理动作里会直接上手的物件本体。
- 第三池：继续保持 `一盒 / 一包 / 一套 / 一组 / 一排 / 一叠 / 一卷 / 一筒 / 一束 / 一册` 的可见单位格式，但避免记号扣、样片和小线材继续膨胀。
- 第四池：按“手作编织前台自然单物 > 场景一眼可读 > 中型边界”重建，禁用提篮、工具包、收纳袋、洗护盆、样本件和卷筒作为主要补位手段。
- 收尾时额外检查同后缀家族，尤其是 `篮 / 袋 / 包 / 盒 / 筒 / 卷 / 册 / 盆 / 桶` 是否还成片出现。

## 7. 本次修改记录

- 本轮实际修改了 `blind_boxes.py` 中 `手作+编织` 的 `support_items` 和 `scene_expansion_items`，`core_items` 与 `visible_small_items` 暂不大动。
- 第二池的修改方向是把“项目管理壳、保护壳和收纳壳混池”拉回“真实编织辅助工具池”。
- 具体清掉了 `安全剪刀套 / 缝合针盒 / 毛线针筒 / 环针连接线包 / 环针收纳夹 / 棒针防护套 / 针尖保护帽 / 钩针束带 / 图样磁条 / 样片活页夹 / 线团网袋 / 可拆针头盘 / 针织项目册 / 备用针筒 / 线头收集篮` 这类壳层、项目层和保护层写法。
- 回填时改用 `缝合针 / 毛线针 / 曲针缝针 / 挑针钩 / 收针钩 / 漏针修补钩 / 线圈拾取针 / 绞线展开器 / 配色转盘 / 环针钥匙 / 针绳连接器 / 针头转换头` 等更直接参与计数、测量、挑针、并针、理线和收尾动作的工具本体。
- 第四池本轮按高风险家族整段清理，移除了 `提篮 / 布袋 / 工具包 / 收纳袋 / 收纳盒 / 洗护盆 / 水桶 / 卷筒 / 词典 / 样本 / 拼贴 / 半成品` 这类后场收纳、洗护和资料系统写法，并额外清掉了 `花盆 / 盆栽 / 玻璃盆 / 画册 / 图册` 这几组最容易模板化、也最容易把编织工位写回旧词链的内容。
- 第四池回填改成“时间物 + 少量环境设备 + 毯面席面 + 工位分区 + 针线阵列 + 花样拼贴 + 操作面与少量半成品”这一路线，尽量让它更像编织前台工作角或休闲编织位里自然可见的中型场景补充物。
- 本轮明确不再沿用 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 画册图册` 这类容易跨类别复用的旧生成习惯，改为直接补 `绕线分区 / 配色分区 / 线团阵列 / 花样拼贴 / 引返操作面 / 并针操作面 / 围巾套组 / 披肩展示面` 这类只属于编织工位语境的中型物。

## 8. 修改后复审

- 当前四池数量均为 50，且池内唯一。
- 第四池与其余三池无重叠。
- 第四池本轮本地复查未命中禁词，且全部通过 `_is_medium_scene_fragment_item` 的中型场景边界校验。
- 第四池中的 `地毯卷 / 毯面 / 席面 / 花盆 / 图册 / 画册` 当前实值分别为 `0 / 2 / 2 / 0 / 0 / 0`，已经不再出现同后缀连续堆叠压读感的问题。
- 这说明本轮已经先把“结构性错误”修掉了，尤其是第四池不再被 `篮 / 袋 / 包 / 盒 / 筒 / 卷 / 册 / 盆 / 桶` 或 `花盆 / 盆栽 / 画册 / 图册` 这类后场服务和旧模板壳连片主导。
- 从读感上看，第二池已经明显回到编织辅助工具池；第四池也从“收纳、洗护、样本资料和氛围设备系统”转回“编织前台场景补充物”。
- 本节结论仅针对 `手作+编织` 的定点复查，不代表全仓所有类别已全量通过。

## 9. 剩余风险与暂留理由

- 第一池里仍保留了较多 `册 / 半成品 / 织片 / 样本 / 收纳桶 / 纱卷` 家族，例如 `毛线收纳桶`、`图样册`、`针法练习本`、`配色样卡册`、`织片样本册`、`线色参考册`、`编织图鉴`、`围巾半成品`、`毛帽半成品`、`袜子半成品`、`手套半成品`、`披肩半成品`、`毛毯半成品`、`祖母方格片`、`罗纹织片`、`麻花织片`、`钩针圆片`、`针法样片`；这轮暂不动，是为了先稳住主体池，不把问题从第二池和第四池转移到第一池。
- 第三池仍保留了较多记号扣、样片、小线材和保护帽单位，例如 `一盒开口记号扣`、`一盒彩色记号扣`、`一盒闭口记号环`、`一盒锁针记号扣`、`一包棒针防脱塞`、`一包针尖保护套`、`一组样片夹`、`一叠针法样片`、`一叠罗纹织片`、`一叠麻花织片`、`一叠祖母方格片`、`一叠钩针圆片`、`一盒织片样片`、`一盒配色样本`、`一册编织图谱`；它们当前格式合规，但后续如果继续做第三池精修，可以再轻压一次。
- 第四池虽然已经脱离“收纳和洗护壳链”，也清掉了 `绿植花盆 / 苔藓玻璃盆 / 空气凤梨盆 / 画册图册` 这条跨类别复用链，但当前仍有 `毯面 / 席面 / 分区 / 阵列 / 拼贴 / 操作面 / 半成品 / 展示面` 这些自然家族；后续若继续微调，重点应放在继续压模板密度，而不是重新引入 `篮 / 袋 / 包 / 盒 / 筒 / 卷 / 册 / 桶`。
- 当前版本已经可以作为后续同类修改的合格基线，下一步若继续优化，建议优先做“第一池样册半成品轻压”和“第三池记号扣样片单位轻压”，而不是再把第二池和第四池拉回壳体补位。
