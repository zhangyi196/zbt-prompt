# 全局不合理命名清单

用途：记录 `Game content extraction/data/blind_boxes.py` 中跨类别反复出现、但未必重名的“不一眼成像”条目，方便后续统一改写。

## 纳入条件

- 读完名字后，普通人脑中没有稳定画面，只知道它“用来干什么”。
- 名字重点落在功能、流程、壳体、被服务对象或抽象分类，而不是主体物。
- 去掉修饰后，稳定不变的仍是通用壳体或解释型骨架。
- 原名更适合改成“可见内容 + 主体物”，说明当前命名过于功能化、壳体化或抽象化。

## 记录字段

- `原词`
- `类别 / 池名 / blind_boxes.py:行号`
- `问题类型`
- `不直观原因`
- `改写方向`
- `示例改名`
- `状态`：`待修改`、`已修改`、`已复查`

## 问题类型建议

- `功能先于主体`
- `壳体先于本体`
- `解释型命名`
- `流程化命名`
- `前缀缝合伪直观`

## 位置写法

- `类别 / 池名 / 条目 / blind_boxes.py:行号`
