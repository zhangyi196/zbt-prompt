# Data Agents Guide

本目录只存放抽取工具的数据事实源，不加入 UI、抽样流程、历史读写或打包逻辑。

## 文件职责

- `blind_boxes.py`：20 个盲盒 `场景+用途` 入口、四池物品库、四栏兼容映射和风险排除配置。
- `审查规则.md`：盲盒物品库内容质检规则，用于发现合理性、实质重复、模板雷同和四池边界问题。
- `审查文件.md`：审查文档索引和命名规则说明。
- `审查文件/`：按类别保存盲盒内容审查结果；子文件命名为 `{类别} 审查文档.md`，如 `桌面+学习 审查文档.md`。修改阶段必须读取对应类别文档并按问题清单执行。
- `修改规则.md`：审查完成后的替换落地规则；先按通用框架检查场景语境、视觉本体、载体模板、四池角色和可玩性，再按类别附加边界修正内容。
- `animals.py`：动物类型、动物用品和动物痕迹数据。
- `item_states.py`：物品状态词；透明、反光、高光语义视为风险。
- `__init__.py`：数据包标记文件。

修改盲盒内容时，先读本文件、`审查规则.md`、`审查文件.md`、目标类别的 `审查文件/{类别} 审查文档.md` 和 `修改规则.md`，再改 `blind_boxes.py`。修改动物或状态词时，仍需遵守上级 `../agents.md` 和根目录 `../../agents.md` 的通用禁区。

## 盲盒写库目标

盲盒物品库服务找不同游戏，不是场景装饰库。每个候选物必须真实可见、边界清楚、尺度合适、符合场景，并能稳定形成一个单一、可圈选的差异点。

审查和修改是两个步骤：先用 `审查规则.md` 找出问题并写入 `审查文件/{类别} 审查文档.md`，再用 `修改规则.md` 读取对应类别审查文档后替换内容。替换时先按通用根因标签判断 `场景错位 / 视觉本体重复 / 载体模板化 / 跨池扩写 / 可玩性弱`，再看类别附加边界；不能只把一个不合格名称换成另一个同视觉本体、同用途或同模板尾词的名称。即使条目通过自动测试，只要场景常识或自然度不稳，也必须继续替换。

## 市场适配

物品概念默认面向欧美 / 国际化找不同用户。中文名称只是输出语言，物品本身应是欧美日常场景也能自然识别的常见物。

除非类别明确要求中国文化、中文学习或特定地域节庆，否则不要写强中式、强地域、强课程体系绑定的物品。例如 `古诗词读本`、`新华字典`、`汉语词典`、`成语词典`、`拼音转盘`、`学生算盘`、`毛笔字帖` 这类内容，应替换为更国际化的同功能物品，如普通读本、双语词典、练习册、闪卡盒、地球仪、计算器、阅读器、笔记本。

## 四池契约

`BLIND_BOX_ITEM_POOL_BUNDLES` 的每个类别固定包含四个列表，每池 50 条且池内唯一：

```python
{
    "core_items": [],
    "support_items": [],
    "visible_small_items": [],
    "scene_expansion_items": [],
}
```

字段名必须保持不变，以兼容运行时映射：

- `core_items` 和 `scene_expansion_items` 映射到大型物品。
- `support_items` 加前 6 个 `core_items` 映射到中型物品。
- `visible_small_items` 映射到散落小型物品。
- `hanging` 当前为空兼容桶。

`scene_expansion_items` 是历史字段名，实际语义为 `placeable_items`：可放置中型单物，不再写组合。

## 四池分工

一句话口诀：代表类别进核心，服务使用进配套，小物成单位，去核心化后进第四池。

| 池 | 写什么 | 不写什么 |
| --- | --- | --- |
| `core_items` | 能代表类别的主题主体物、主承载物、主容器、主体设备 | 微小配件、大型家具、只靠文字识别的纸张标签、辅助物冒充核心 |
| `support_items` | 服务核心物使用的工具、辅助件、护套、承托件、替换件 | 单个针钉扣环签贴帽芯、把核心物换小名重写一遍 |
| `visible_small_items` | 原本偏小物品升级后的可见单位，如一盒、一包、一筒、一盘、一套、一卷、一束、一叠、一册、一排 | 单粒、单片、单根、单张、碎屑、小点、细线、污点、标签、贴纸、价格签 |
| `scene_expansion_items` | 去核心化后的辅助性中型单物，适合新增、删除、替换，放在边角或已有承载面也合理 | 组合、核心物同义变体、主视觉物、大型家具、过小配件 |

同一基础对象通常只能选择一个主归属；不要为了凑数同时写进多个池。若确实需要跨池出现，必须满足角色和尺度明显不同。

## 全局硬禁区

1. 禁止抽象词、用途词、区域词、动作词和概括词。
2. 禁止依赖文字内容、细线、小点、刻痕、碎屑、污渍、贴签、价格签作为识别点。
3. 禁止依赖光影、明暗、发光、高光、反光、透明、投影、滤镜、镜面成像变化。
4. 禁止单个微小物：针、钉、扣、环、珠、粒、芯、帽、签、贴、线头、碎片、小卡片。
5. 微小物必须升级为可见单位：`一盒`、`一包`、`一筒`、`一盘`、`一套`、`一卷`、`一束`、`一叠`、`一册`。
6. 禁止大型家具或大面积主体：书桌、柜子、货架、推车、工作台、操作台、大椅子、大帐篷。
7. 禁止同一基础对象在同一类别四池机械重复，例如 `单词卡册` 不得同时出现在 `core_items` 和 `support_items`。
8. 禁止靠同质尾词凑数：不要批量堆叠 `收纳盒`、`整理篮`、`展示架`、`托座`、`分装盒`、`组合`。
9. 宠物类别只写宠物用品，不写动物本体；海底潜水类别只写装备和可见器物，不写海洋动物本体。
10. 默认避免强中式、强地域或强课程体系绑定物品；优先写欧美用户也熟悉的通用物品。
11. 中文名称保持简洁，优先 2-10 个汉字；必要时可到 14 个汉字，但不得写成长句。

## 反模板与尾词配额

生成物品时先找真实物品族，再命名具体物品；禁止先写功能词，再套 `板`、`盘`、`盒`、`包`、`垫`、`册`、`卷` 等尾词造词。

禁止造词方式：

- `功能词 + 板`：如 `错题整理板`、`单词记忆板`、`作文构思板`。
- `功能词 + 盘`：如 `文具分类盘`、`纸张整理盘`、`便签分格盘`。
- `场景词 + 包/盒/册/卷`：如 `阅读工具包`、`练字工具包`、`课程分区册`。
- 同一尾词连续铺开，例如 10 条以上都是 `...板`，再接 10 条 `...盘`。

允许的尾词必须对应现实中稳定存在的物品名，而不是临时功能容器。比如 `桌面白板`、`软木记事板`、`文件托盘`、`文具盒`、`笔袋` 可以写；`阅读摘录板`、`重点归纳板`、`草稿整理垫` 不写。

### 雷同密度审查

尾词不超额仍然可能雷同。生成后必须再检查“同一物品族 / 同一模板骨架”的密度，避免只替换前缀来凑数。

视为同一模板的情况：

- `对象词 + 同一品类名`：如 `模具清洁刷`、`烤盘清洁刷`、`粉筛清洁刷`、`缝隙清洁刷`。
- `对象词 + 同一容器名`：如 `面粉密封袋`、`糖粉密封袋`、`可可夹链袋`、`香草糖袋`。
- `对象词 + 同一承托名`：如 `量勺挂架`、`防烫手套挂架`、`茶巾挂架`。
- `对象词 + 同一动作名`：如 `杯盖晾干架`、`滤杯晾干架`、`裱花嘴晾干座`。
- 同一功能词反复出现：如 `清洁`、`密封`、`分隔`、`晾干`、`搁置`、`抽取`、`防滑`、`隔热` 成片堆叠。

审查配额：

- 同一“完整品类名”每池原则上不超过 2 条，例如 `清洁刷`、`密封袋`、`晾干架`。
- 同一“功能词 + 器物族”每池原则上不超过 3 条，例如各种 `清洁 + 刷`、各种 `分隔 + 盒/筒`。
- 第四池若同一辅助物品族超过 3 条，必须替换为其他真实物品族，而不是继续换前缀。
- `袋`、`筒`、`刷`、`架`、`座`、`器` 虽不全部属于硬尾词配额，也必须参与雷同密度审查。

替换方向：若出现一串 `xxx清洁刷`，改用刮板、擦拭布卷、沥水篮、隔热垫、计时器、转移板、抽取盒等不同物品族；若出现一串 `xxx袋`，改用罐、盒、筒、篮、夹、卷、托盘、抽取器等不同形态。

尾词配额：

- 单一高频尾词原则上不超过 5 条。
- `scene_expansion_items` 中 `板`、`盘`、`垫`、`盒`、`包`、`册`、`卷` 各不超过 4 条。
- 同一尾词不得连续出现超过 3 条。
- 每池尽量覆盖至少 5 个真实物品族；第四池必须覆盖至少 5 个辅助性物品族。
- 若某一尾词超额，改用不同真实物品族，不要换近义功能词。

## 第四池专项规则

`scene_expansion_items` 必须先去组合，再去核心化。

第四池允许：

- 现实中稳定存在的中型单物。
- 比 `visible_small_items` 更清楚，比 `core_items` 更弱主角。
- 辅助取用、收纳承托、保护垫放、清洁整理、分装外带、轻量操作的小设备。
- 不需要大面积空位，不改变人物姿态、支撑关系或主体结构。

第四池禁止：

- 任何 `组合` 结尾条目，包括 `A+B组合`、`用品组合`、`小物组合`、`包装组合`。
- 第一池核心物的同义变体、材质变体、形状变体、层数变体、大小变体。
- 能独立成为当前场景中心主角的主承载物、主容器、主设备。
- 推车、抽屉柜、陈列架、货架、书桌、柜台等大型家具或存储主体。
- `功能词 + 板/盘/垫/盒/包/册/卷` 的临时造词。

生成第四池时，先列出该类别第一池的核心物家族，主动避开这些家族的同义变体，再从辅助性物品族补足 50 条：

- 取用操作类：夹、勺、托、座、筒、刷、固定夹、切分器、搅拌器。
- 收纳承托类：小型收纳筒、分隔盒、文件托盘、杯垫座、资料立架、餐具筒。
- 保护垫放类：杯垫、餐垫、书写垫、隔热垫、防滑垫。
- 分装外带类：纸袋、外带袋、密封袋、抽取盒、便携包。
- 清洁整理类：清洁刷、擦拭器、刮水片、碎屑清理器。
- 轻量设备类：计时器、计算器、起泡器、磨豆机、小钟、录音笔。

判断口诀：能当场景主角的进第一池；只能辅助场景成立、适合新增删除替换的进第四池。

## 生成流程

1. 先列出该场景的 8-12 个真实物品族，例如书本文具、餐具饮品、洗护瓶罐、工具耗材，并剔除强中式或地域专属物品族。
2. 生成 `core_items`：只放代表性主体。
3. 生成 `support_items`：服务核心物，并排除已在第一池出现的基础对象。
4. 生成 `visible_small_items`：把小物统一提升为成盒、成包、成筒、成盘、成套等可见单位。
5. 生成 `scene_expansion_items`：先排除第一池核心物家族的同义变体，再从辅助性中型单物中选物。
6. 检查每池 50 条、池内唯一、四池之间尽量不重复基础对象。
7. 检查尾词配额、连续尾词、真实物品族覆盖和雷同密度。
8. 删除所有抽象、过小、过大、组合、尾词凑数、功能造词、同模板凑数、核心同义变体和场景不匹配条目。
9. 若是对已写库内容做替换，必须先完成 `审查规则.md` 的问题归类并写入 `审查文件/{类别} 审查文档.md`，再按 `修改规则.md` 读取对应类别审查文档、选择不同视觉本体族的替换项，并进行修改后复审。

## 最终验收清单

- 每池是否正好 50 条且池内唯一？
- 条目是否真实可见、第一眼能识别、能单独圈选？
- 条目是否能作为一个单点差异？
- 是否存在抽象词、用途词、区域词、动作词或概括词？
- 是否存在微小单物、碎屑、小点、细线、污点、标签、贴纸、价格签？
- 是否依赖文字、光影、反光、高光、透明或发光效果？
- 是否默认适配欧美 / 国际化用户，避免了强中式、强地域或强课程体系绑定物品？
- 是否有大型家具、大面积主体或需要改变画面结构才能放置的物品？
- 四池之间是否机械重复同一基础对象？
- `scene_expansion_items` 是否完全没有 `组合`，并且已去核心化？
- 是否存在 `功能词 + 板/盘/垫/盒/包/册/卷` 的临时造词、尾词过量或尾词连续成片？
- 是否存在同模板凑数，例如多条 `xxx清洁刷`、`xxx密封袋`、`xxx晾干架`、`xxx搁置座`、`xxx分隔盒`？
- 是否只是把前缀从 `烤盘`、`模具`、`粉筛`、`裱花嘴` 换掉，但物品本体仍然是同一种？

## 示例边界

- 桌面学习：第一池可写课本、练习册、双语词典、文具盒；第四池优先写计时器、阅读架、文件托盘、笔筒、清洁刷、电子词典等辅助性中型单物；默认不写 `古诗词读本`、`新华字典`、`汉语词典`、`成语词典`、`拼音转盘`、`学生算盘`。
- 餐桌茶歇：第一池可写茶壶、咖啡杯、点心架、甜点盘、面包篮、饮品壶、饼干盒、果酱罐；第四池优先写茶包分隔盒、杯垫收纳座、搅拌棒筒、吸管收纳筒、咖啡滤纸盒、糖包抽取盒、餐巾固定夹、茶匙托、奶精球碗、滤茶器、牛奶起泡器、咖啡机清洁刷。
- 厨房烘焙：第四池可以出现少量烘焙纸抽取盒、隔热垫、晾干架、清洁刷、密封袋，但不能成串写 `模具清洁刷`、`烤盘清洁刷`、`粉筛清洁刷`、`缝隙清洁刷`，也不能成串写 `面粉密封袋`、`糖粉密封袋`、`可可夹链袋`、`香草糖袋`；应混入计时器、撒粉器、转移板、刮板、沥水篮、保护卷、工具包等不同物品族。

若任一验收项不通过，必须替换为更清楚、更稳定、更适合找不同的物品。
