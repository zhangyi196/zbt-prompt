# Data Agents Guide

本目录只存放抽取工具的数据事实源，不加入 UI、抽样流程、历史读写或打包逻辑。

## 文件职责

- `blind_boxes.py`：20 个盲盒 `场景+用途` 入口、四池物品库、四栏兼容映射和风险排除配置。
- `animals.py`：动物类型、动物用品和动物痕迹数据。
- `item_states.py`：物品状态词；透明、反光、高光语义视为风险。
- `__init__.py`：数据包标记文件。

修改盲盒内容时，先读本文件，再改 `blind_boxes.py`。修改动物或状态词时，仍需遵守上级 `../agents.md` 和根目录 `../../agents.md` 的通用禁区。

## 盲盒物品目标

盲盒物品库服务找不同游戏，不是场景装饰库。每个候选物必须能稳定形成一个清楚、单一、可圈选的差异点。

每个物品条目必须满足：

- 真实可见：现实中能看到的具体物品。
- 单点差异：一个条目默认只形成一个变化点。
- 可圈选：边界清楚，玩家能一眼圈出。
- 尺度合适：不能太小、太碎、太大或需要大面积改图。
- 场景匹配：符合当前盲盒类别的场景和用途。

## 四池结构

`BLIND_BOX_ITEM_POOL_BUNDLES` 的每个类别固定包含四个列表：

```python
{
    "core_items": [],
    "support_items": [],
    "visible_small_items": [],
    "scene_expansion_items": [],
}
```

字段名必须保持不变，以兼容运行时映射：

- `core_items` 和 `scene_expansion_items` 映射到大型物品。
- `support_items` 加前 6 个 `core_items` 映射到中型物品。
- `visible_small_items` 映射到散落小型物品。
- `hanging` 当前为空兼容桶。

`scene_expansion_items` 是历史字段名，实际语义为 `placeable_items`：可放置中型单物。这里不再写组合。

## 四池一体化判定

四池不是四套独立素材库，而是同一批场景物品按找不同用途分流。判定顺序如下：

1. 能代表类别、边界清楚、体量适中的主体物，放入 `core_items`。
2. 服务主体物使用的工具、辅助件、护套、承托件和替换件，放入 `support_items`。
3. 单个太小但成盒、成包、成盘、成套后清楚可见的小物，放入 `visible_small_items`。
4. 最适合做新增、删除、替换差异点的中型单物，放入 `scene_expansion_items`。

一句话口诀：代表类别进核心，服务使用进配套，小物成单位，最好落图进第四池。

同一基础对象通常只能选择一个主归属；不要为了凑数同时写进多个池。若确实需要跨池出现，必须满足角色和尺度明显不同，例如 `棉签筒` 可作核心物，但单个 `棉签` 不得作配套物，若需要只能写成 `一盒棉签` 或 `棉签筒` 这类可见单位。

第四池优先承接“看起来像一个完整物品、又比核心物更容易放进画面”的候选。它不是组合池，也不是扩写池；若一个条目必须靠多个物品同时出现才成立，应改写为其中最清楚、最适合圈选的单个主体物。

## 通用写库规则

1. 每池固定 50 条，池内唯一。
2. 条目必须是真实可见物或成单位的可见物品集合。
3. 禁止抽象词、用途词、区域词、动作词和概括词。
4. 禁止依赖文字内容、细线、小点、刻痕、碎屑、污渍、贴签、价格签作为识别点。
5. 禁止依赖光影、明暗、发光、高光、反光、透明、投影、滤镜、镜面成像变化。
6. 禁止单个微小物：针、钉、扣、环、珠、粒、芯、帽、签、贴、线头、碎片、小卡片。
7. 微小物必须升级为可见单位：`一盒`、`一包`、`一筒`、`一盘`、`一套`、`一卷`、`一束`、`一叠`、`一册`。
8. 禁止大型家具或大面积主体：书桌、柜子、货架、推车、工作台、操作台、大椅子、大帐篷。
9. 禁止同一基础对象在同一类别四池机械重复；例如 `单词卡册` 不得同时出现在 `core_items` 和 `support_items`。
10. 禁止靠同质尾词凑数：不要批量堆叠 `收纳盒`、`整理篮`、`展示架`、`托座`、`分装盒`、`组合`。
11. 宠物类别只写宠物用品，不写动物本体；海底潜水类别只写装备和可见器物，不写海洋动物本体。
12. 中文名称保持简洁，优先 2-10 个汉字；必要时可到 14 个汉字，但不得写成长句。

## 反模板与尾词配额

生成物品时先找真实物品族，再命名具体物品；禁止先写功能词，再套 `板`、`盘`、`盒`、`包`、`垫`、`册`、`卷` 等尾词造词。

禁止的造词方式：

- `功能词 + 板`：如 `错题整理板`、`单词记忆板`、`作文构思板`、`习题演算板`。
- `功能词 + 盘`：如 `文具分类盘`、`纸张整理盘`、`便签分格盘`。
- `场景词 + 包/盒/册/卷`：如 `阅读工具包`、`练字工具包`、`课程分区册`。
- 同一尾词连续铺开：如 10 条以上都是 `...板`，再接 10 条 `...盘`。

允许的尾词必须对应现实中稳定存在的物品名，而不是临时拼出来的功能容器。比如 `桌面白板`、`软木记事板`、`文件托盘`、`文具盒`、`笔袋` 可以写；`阅读摘录板`、`重点归纳板`、`草稿整理垫` 这类功能造词不写。

每池 50 条的尾词配额：

- 单一高频尾词原则上不超过 5 条；`scene_expansion_items` 中 `板`、`盘`、`垫`、`盒`、`包`、`册`、`卷` 各不超过 4 条。
- 同一尾词不得连续出现超过 3 条。
- 容器类、板面类、纸册类、工具类、电子小设备、书本文档、支架承托物至少覆盖 5 个物品族。
- 若发现某一尾词超额，必须改用不同真实物品族，而不是换一个近义功能词。

优先替换方向：

- `...板` 过多时，改为台历、计时器、计算器、词典、书立、笔筒、文件夹、资料册、夹板、阅读架。
- `...盘` 过多时，改为文件托盘、笔筒、文具盒、笔袋、收纳筒、书立、夹板、资料立架。
- `...盒/包` 过多时，改为具体工具、册本、支架、垫板、计时器、台历、卷筒或夹具。
- `...册/本` 过多时，改为文件夹、活页夹、夹板、资料立架、书立、台历或工具。

## 四池分工

### `core_items`

写当前场景最有代表性的主体物。它们比其他池更能代表类别，但仍必须适合单独圈选。

允许：

- 场景核心器物。
- 中等体量主体。
- 边界清楚的盒、包、罐、瓶、盘、篮、垫、板、册、卷、工具包。
- 能一眼判断类别归属的物品。

禁止：

- 单个微小配件。
- 大型家具和大面积结构。
- 只能靠文字内容识别的纸张、标签、卡片。
- 与 `support_items` 或 `scene_expansion_items` 只是尾词不同的同一物。

### `support_items`

写服务核心物使用的工具、辅助器具、替换件、承托件和操作配件。它可以比核心物小，但不能小到难以圈选。

允许：

- 工具类：剪刀、订书机、打孔器、量勺、刮刀、刷子。
- 操作辅助类：胶带座、清洁刷、杯垫座、束带、护套。
- 成单位配件：回形针盒、图钉盒、球针盒、珠针盘、棉签筒。
- 有明确形体的替换件或配件包。

禁止：

- 单个针、钉、扣、环、签、贴、帽、芯。
- 单个发夹、棉签、牙线棒、钥匙扣、球针、珠针、大头针。
- 把 `core_items` 换小名重写一遍。
- 大量 `夹盒`、`分装盒`、`托座` 等同质尾词。

### `visible_small_items`

写原本偏小物品的成组、成盒、成套、成叠单位。重点不是数量多，而是整体清楚可圈选。

优先格式：

- `一盒...`
- `一包...`
- `一筒...`
- `一盘...`
- `一套...`
- `一卷...`
- `一束...`
- `一叠...`
- `一册...`
- `一排...`

限制：

- `一组` 只用于体量已经足够的物品；微小物优先改成 `一盒`、`一包`、`一盘`。
- `两只`、`三只` 只允许用于杯、瓶、盒、碗、玩具球等本身体量足够的物品。
- 禁止 `两枚`、`三枚` 微小物。
- 禁止单粒、单片、单根、单张、碎屑、小点、刻痕、细线、污点、标签、贴纸、价格签和文字卡。

### `scene_expansion_items`

保留历史字段名，实际按 `placeable_items` 维护：写最适合找不同新增、删除、替换的中型单物。

允许：

- 现实中稳定存在的中型单物。
- 比 `visible_small_items` 更清楚。
- 比 `core_items` 更灵活、更容易放进画面。
- 不需要大面积空位。
- 不改变人物姿态、支撑关系或主体结构。
- 可新增、可删除、可替换，第一眼能看出差异。
- 尾词和物品族分布均衡，不出现 `...板`、`...盘`、`...盒` 成片堆叠。

禁止：

- 任何 `组合` 结尾条目。
- `A+B组合`、`A+B+C组合`。
- `用品组合`、`小物组合`、`包装组合`、`日常用品组合`。
- `功能词 + 板/盘/垫/盒/包/册/卷` 的临时造词。
- 过小配件。
- 大型家具。
- 需要改变画面结构才能放置的物品。

生成第四池时，先从以下物品族各取若干真实物品，再补足 50 条：

- 书本文档类：课本、练习册、资料册、文件夹、夹板、台历。
- 文具工具类：文具盒、笔袋、笔筒、订书机、打孔器、修正带盒。
- 支架承托类：书立、阅读架、平板支架、文件托盘、资料立架。
- 桌面设备类：计时器、计算器、电子词典、录音笔、桌面小钟。
- 纸册卡片类：便签本、索引册、单词卡盒、错题本、草稿本。
- 清洁整理类：桌面清洁刷、橡皮屑清理器、文件收纳包。

## 生成顺序

1. 先列出该场景的 8-12 个物品族，例如书本文具、餐具饮品、洗护瓶罐、工具耗材。
2. 每池从不同物品族中取材，避免一池被同一族占满。
3. 先生成 `core_items`，只放代表性主体。
4. 再生成 `support_items`，排除已在 `core_items` 中出现的基础对象。
5. 再生成 `visible_small_items`，把小物统一提升为成盒、成包、成筒、成盘、成套。
6. 最后生成 `scene_expansion_items`，只写可放置中型单物，禁止组合，且先按物品族分布选物，再检查尾词配额。
7. 检查每池 50 条、池内唯一、四池之间尽量不重复基础对象。
8. 检查每池尾词分布：单一高频尾词是否超 5 条，第四池 `板/盘/垫/盒/包/册/卷` 是否各超 4 条。
9. 删除所有抽象、过小、过大、组合、尾词凑数、功能造词和场景不匹配条目。

## 自检清单

每个条目都必须逐项通过：

- 是否是真实可见物？
- 是否第一眼能识别？
- 是否能单独圈选？
- 是否能作为一个单点差异？
- 是否太小、太碎或太细？
- 是否需要太大空位？
- 是否是抽象概括词？
- 是否依赖文字、标签、光影、反光或透明感？
- 是否与同类别其他池重复同一基础对象？
- `scene_expansion_items` 是否完全没有 `组合`？
- 是否存在 `功能词 + 板/盘/垫/盒/包/册/卷` 的临时造词？
- 同一尾词是否过量或连续成片？
- 第四池是否覆盖至少 5 个真实物品族？

若任一项不通过，必须替换为更清楚、更稳定、更适合找不同的物品。
