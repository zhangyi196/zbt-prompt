# 游戏内容抽取工具

这是一个基于 `tkinter` 的本地中文桌面工具，用于快速生成游戏主图 / 场景相关的抽取结果。当前界面采用单主窗口双工作区：

- `盲盒物品/动物抽取`：按盲盒编号抽取物品，并可按动物类型抽取动物内容。
- `人物表情抽取`：把表情组文本中的 `具体表情` 自动补全为眉 / 眼 / 嘴描述。

核心抽取功能不依赖 Web、数据库或服务端；核心数据保存在本目录和上级提示词文件中。右上角 `检查更新` 会联网读取 GitHub Releases。

## 运行

在仓库根目录运行：

```powershell
python 'Game content extraction\内容抽取.py'
```

或进入本目录后运行：

```powershell
python '内容抽取.py'
```

## 界面概览

主窗口顶部有两个切换按钮：

- `盲盒物品/动物抽取`：默认显示。
- `人物表情抽取`：在同一主窗口内切换显示，不再默认弹出独立窗口。

主窗口右上角有 `检查更新` 按钮，用于检查 GitHub Releases 是否有新版。

UI 使用浅灰蓝背景、白色内容区、蓝色主按钮和胶囊式切换按钮。第一版使用原生 `tkinter/ttk` 做轻量风格转译，不引入第三方 UI 依赖。

## 盲盒物品/动物抽取

输入框使用逗号分隔的单行语法：

```text
盲盒编号[,盲盒编号2...][,动物类型][,子类指令1][,子类指令2...]
```

规则：

- 至少输入一个盲盒编号，编号必须是数字。
- 可一次输入多个盲盒编号，程序会随机选中一个盲盒。
- 动物类型可选且最多一个：`无动物`、`地面动物`、`空中动物`、`水中动物`。
- 未填写动物类型时默认 `无动物`。
- 禁用某类内容：`无` + 类别标签，例如 `无大型物品`、`无动物用品`。
- 调整某类数量：`类别标签+数字` 或 `类别标签-数字`，例如 `中型物品+1`、`动物本体-1`。
- 输入覆盖只影响本次抽取，不修改界面默认勾选和数量，也不清空历史。

示例：

```text
1
1,5
1,地面动物
1,5,地面动物,无大型物品,中型物品+1
```

可抽取内容：

- 物品：大型物品、中型物品、散落小型物品、悬挂物品。
- 动物：动物本体、动物用品、动物痕迹。
- 物品状态：勾选 `启用物品状态` 后，会输出“状态词 + 物品名”的组合。

历史按钮：

- `重置物品历史`：只清空物品抽取历史。
- `重置动物历史`：只清空动物抽取历史。
- `重置全部历史`：同时清空物品和动物抽取历史。

## 人物表情抽取

人物表情工作区用于处理 `组图 23 表情前置.md` 输出的表情组文本。工具会读取：

- `极性`
- `具体表情`
- `单人/多人`

然后从 `../组图 23 表情库.md` 中查找对应模板，并只把眉 / 眼 / 嘴内容追加到 `具体表情:` 字段后，其他字段尽量保持原样。

核心规则：

- `组图 23 表情库.md` 是表情模板的单一事实源。
- 每个表情类别有 8 条模板。
- `1-4` 为单人模板。
- `5-8` 为多人模板。
- 默认使用 `随机模板`。
- 可切换为 `指定模板编号`，编号 `1-4` 对应单人，`5-8` 对应多人。
- 支持同一输入中的多组 `极性:` 逐组回填。
- 重复增强时会替换已有眉 / 眼 / 嘴模板，不会堆叠多份。
- 保留 `[目标物]`、`[证据物]`、`[对方人物]`、`[剧情食物]`、`[剧情小物]` 等占位符，不自动替换。
- 不写入 `draw_history.json`，不影响盲盒/动物历史。

示例输入：

```text
极性: 负向 剧情: 手中茶杯 → 突然变成粉色甜甜圈 → 感到不解与迟疑 → 手端甜甜圈发愣 [备选情绪反差方案] 单人/多人: 单人 具体表情: 困惑 人物定位: 画面中央穿粉色裙子的女士 表情功能: 困惑 适配提示: 眼神向下回指手中突然出现的粉色甜甜圈，嘴部表现出不解的微张或抿嘴，不可改变手部端托的动作。 禁用区域: 若采用该人物面部表情，该人物衣领/领带/颈部区、头发/头饰区、耳饰/耳侧饰品区不再建议修改；身体与姿态不变，头部朝向不变，四肢动作不变。
```

模板编号为 `4` 时，`具体表情` 会增强为类似：

```text
具体表情: 困惑，眉：一侧眉尾抬起，另一侧眉尾压平；眼：一侧眼撑开看着[目标物]，另一侧眼半垂；嘴：一侧嘴巴闭住下压，另一侧嘴角收紧。
```

## 数据与文件

- `内容抽取.py`：主程序；包含 UI、输入解析、抽取逻辑、历史读写、人物表情增强逻辑。
- `data/blind_boxes.py`：盲盒物品数据。
- `data/animals.py`：动物内容数据。
- `data/item_states.py`：物品状态词与权重。
- `draw_history.json`：抽取历史。
- `../组图 23 表情库.md`：人物表情模板资料源。
- `内容抽取.spec`：PyInstaller 打包配置。

## 历史机制

抽取历史保存在：

```text
draw_history.json
```

历史结构分为：

- `item_pools`：物品池历史。
- `animal_pools`：动物池历史。

抽中过的内容会临时降权，一轮基本跑完后自动进入下一轮。输入新内容不会自动清空历史；只有界面上的重置按钮会清空对应历史。

## 检查更新

右上角 `检查更新` 按钮会后台访问：

```text
https://api.github.com/repos/zhangyi196/zbt-prompt/releases/latest
```

当前程序版本写在 `内容抽取.py` 的 `APP_VERSION` 常量中，本次发布版本为 `0.1.1`。检查逻辑只读取 GitHub 最新 Release、比较版本并打开发布页，不会自动下载、覆盖或重启程序。

当前仓库如果还没有 GitHub Release，程序会提示“当前 GitHub Releases 还没有发布文件”，并可打开发布页：

```text
https://github.com/zhangyi196/zbt-prompt/releases
```

发布新版时建议：

- 在 GitHub Releases 创建新版本，tag 使用 `v0.1.2` 或 `0.1.2` 这类高于当前版本的数字版本。
- 上传打包好的新版文件。
- 下一次发版前同步更新 `APP_VERSION`。

## 打包

本项目包含 PyInstaller 配置：

```text
内容抽取.spec
```

在 `Game content extraction` 目录中可使用类似命令打包：

```powershell
pyinstaller '内容抽取.spec'
```

打包配置已包含 `../组图 23 表情库.md`，程序运行时会按源码目录、程序相邻目录和 `_MEIPASS` 查找表情库文件。

## 维护注意

- 保持中文界面和中文输出风格。
- 不要破坏盲盒/动物的逗号分隔输入语法。
- 不要修改 `draw_history.json` 的历史语义。
- 不要把人物表情抽取接入物品/动物历史机制。
- 不要让检查更新写入 `draw_history.json`。
- 检查更新只做提示和打开 GitHub Releases，不自动替换正在运行的 exe。
- 不要把 `data/` 拆得过碎；数据文件中不要加入 UI、抽样逻辑或历史保存逻辑。
- 修改表情抽取逻辑后，必须运行人物表情回归验证脚本。
- 修改 UI 后，至少运行 `py_compile`；环境允许时手动打开窗口检查两个工作区切换。
