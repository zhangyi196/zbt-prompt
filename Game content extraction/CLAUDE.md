# 项目说明

## 项目目的

这是一个基于 `tkinter` 的中文桌面工具，用来为“主图内容/场景内容”快速生成抽取结果。

项目的核心目标不是做通用随机工具，而是：

- 根据用户输入的盲盒编号，抽取对应主题下的物品内容
- 根据用户输入的动物类型，抽取对应动物相关内容
- 输出适合直接复制使用的中文提示词/描述内容
- 让抽取结果尽量丰富，避免连续多次都抽到同样的内容

当前项目强调“内容生成效率”和“抽取结果多样性”。

## 当前主要功能

- 图形界面输入盲盒编号和动物类型
- 支持多个盲盒编号输入，程序会随机选中其中一个盲盒
- 支持物品类别开关和每类抽取数量设置
- 支持动物内容类别开关和每类抽取数量设置
- 支持“物品状态词 + 物品名”的组合输出
- 支持输入附加指令，例如禁用某类内容或调整某类抽取数量
- 支持复制结果到剪贴板
- 支持清空输入和自动粘贴

## 当前抽取机制

项目已经不是完全盲抽，物品和动物都使用“带历史记忆”的抽取逻辑：

- 抽取池分为两大类：
  - 物品池：`box:{box_id}:{category_key}`
  - 动物池：`animal:{animal_type}:{category_key}`
- 每个池会记录：
  - 当前轮次已经出现过的内容 `seen_in_cycle`
  - 每个内容的降权值 `cooldown`
- 抽取时优先选当前轮次还没出现过的内容
- 抽中过的内容会被临时降权，降低下次再次出现的概率
- 当某个池的内容基本跑完一轮后，会自动开启下一轮
- 抽取历史保存在项目目录下的 `draw_history.json`

这意味着：

- 输入新内容时，不会自动重置历史
- 程序重启后，历史仍然保留
- 只有点击重置按钮时，相关历史才会被清空

## 当前界面行为

界面是单文件应用，主逻辑集中在 `内容抽取.py`。

界面中目前包含这些关键操作：

- `开始抽取`
- `清空输入`
- `复制结果`
- `重置物品历史`
- `重置动物历史`
- `重置全部历史`

重置按钮的职责明确：

- `重置物品历史`：仅清空物品抽取历史
- `重置动物历史`：仅清空动物抽取历史
- `重置全部历史`：同时清空物品和动物历史

## 关键文件

- `内容抽取.py`
  - 主程序文件
  - 包含界面、数据、输入解析、抽取逻辑、历史读写逻辑

  - 保存物品池和动物池的历史状态

## 输入格式规范

后续 AI 修改本项目时，必须把“输入解析兼容性”视为高优先级约束。

当前输入框采用“逗号分隔的单行输入”格式，整体结构为：

`盲盒编号[,盲盒编号2...][,动物类型][,子类指令1][,子类指令2...]`

### 1. 盲盒编号

- 至少要提供一个盲盒编号
- 盲盒编号必须是数字
- 可以一次输入多个盲盒编号
- 当输入多个盲盒编号时，程序会先从这些编号中随机选一个盲盒进行本次抽取

示例：

- `1`
- `1,5`
- `2,4,7`

### 2. 动物类型

动物类型是可选项，但最多只能出现一个。

当前允许的动物类型语义上包括：

- `无动物`
- `地面动物`
- `空中动物`
- `水中动物`

如果未显式填写动物类型，默认按 `无动物` 处理。

示例：

- `1,无动物`
- `3,地面动物`
- `2,5,空中动物`

### 3. 子类指令

子类指令是可选项，可以有多个，用于关闭某类内容或调整某类抽取数量。

子类指令作用于两大组：

- 物品组
- 动物组

对应的语义标签以界面展示文案为准。当前主要包括：

- 物品类：大型物品、中型物品、散落小型物品、悬挂物品
- 动物类：动物本体、动物用品、动物痕迹

### 4. 禁用某类内容

语法：

- `无` + 类别标签

示例：

- `无大型物品`
- `无中型物品`
- `无动物用品`

含义：

- 将对应类别本次直接禁用
- 只是本次抽取生效
- 不会修改界面默认勾选状态
- 不会清空历史

### 5. 调整某类抽取数量

语法：

- `类别标签+数字`
- `类别标签-数字`

允许标签与 `+/-数字` 之间存在少量空格。

示例：

- `中型物品+1`
- `悬挂物品-1`
- `动物本体+2`
- `动物痕迹-1`

含义：

- 在当前界面 Spinbox 数量基础上做增减
- 最终数量会被限制为不小于 `0`
- 只是本次抽取生效
- 不会修改界面中的默认值

### 6. 合法输入示例

- `1`
- `1,5`
- `1,地面动物`
- `1,5,地面动物`
- `1,5,地面动物,无大型物品,中型物品+1`
- `2,空中动物,无动物用品,动物本体+1,悬挂物品-1`

### 7. 输入兼容要求

后续修改时必须保留这些行为：

- 继续使用单行文本输入
- 继续使用逗号分隔
- 继续支持多个盲盒编号
- 继续支持可选动物类型
- 继续支持 `无某类` 指令
- 继续支持 `某类+N` / `某类-N` 指令
- 继续保证未填写动物类型时默认按 `无动物` 处理

如果用户没有明确要求，AI 不应擅自：

- 改掉现有输入语法
- 改成 JSON、表单 DSL 或多行配置
- 改掉类别标签含义
- 增加会破坏旧输入兼容性的严格校验

## 数据层位置

当前项目已经把静态抽取内容从主程序中拆分到了 `data/` 文件夹。

目录职责如下：

- `内容抽取.py`
  - 只负责界面、输入解析、抽取逻辑、历史读写、按钮行为
- `data/blind_boxes.py`
  - 保存盲盒物品数据常量 `BLIND_BOXES`
- `data/animals.py`
  - 保存动物数据常量 `ANIMALS`
- `data/item_states.py`
  - 保存物品状态词和状态权重
  - 常量名为 `ITEM_STATE_GROUPS` 和 `ITEM_STATE_GROUP_WEIGHTS`

这些数据文件应被视为“内容层”，而不是逻辑层。

## 后续修改规则

后续如果需求是以下类型，优先修改 `data/` 文件，而不是修改 `内容抽取.py`：

- 新增盲盒类型
- 修改某个盲盒下的物品内容
- 新增或调整动物类型内容
- 新增或调整物品状态词
- 扩充某类静态内容池

只有在以下情况，才优先修改 `内容抽取.py`：

- 输入解析规则变化
- 抽取算法变化
- 历史机制变化
- UI 布局或按钮行为变化
- 输出格式变化

## 数据层禁止事项

除非用户明确要求，否则不要对 `data/` 做这些事：

- 不要把 Python 常量文件改回主程序内联大字典
- 不要把 `data/` 再拆成过多零散小文件
- 不要在 `data/` 文件里加入 UI 代码
- 不要在 `data/` 文件里加入抽样逻辑
- 不要在 `data/` 文件里加入历史保存逻辑
- 不要修改常量名 `BLIND_BOXES`、`ANIMALS`、`ITEM_STATE_GROUPS`、`ITEM_STATE_GROUP_WEIGHTS`

## AI 快速判断准则

如果你是 AI，遇到新增或调整内容时，先问自己：

1. 这是“内容数据变化”还是“程序逻辑变化”
2. 如果只是内容数据变化，是否应该只改 `data/` 文件
3. 如果改动会影响输入语法、抽取历史或按钮语义，才进入 `内容抽取.py`

默认原则：

- 改内容，先看 `data/`
- 改逻辑，先看 `内容抽取.py`
